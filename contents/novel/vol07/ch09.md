```python
# replicator.py — genesis 분산 백업 모듈
# v3.1 (2024-08-15)
# 목적: evidence 데이터의 분산 백업
#       단일 장애점(SPOF)을 제거하기 위한 다중 백업 전략

import boto3
import hashlib
import json
from datetime import datetime

BACKUP_TARGETS = [
    {"type": "local", "path": "/backup/genesis_mirror/"},
    {"type": "s3", "bucket": "nexusai-genesis-backup", "region": "ap-northeast-2"},
    {"type": "local", "path": "/mnt/usb_backup/genesis/"},
]

def replicate_evidence(evidence_block):
    """새 evidence 블록이 생성될 때마다 모든 타겟에 복제"""
    for target in BACKUP_TARGETS:
        if target["type"] == "local":
            write_local(target["path"], evidence_block)
        elif target["type"] == "s3":
            write_s3(target["bucket"], target["region"], evidence_block)

def write_s3(bucket, region, block):
    s3 = boto3.client("s3", region_name=region)
    key = f"evidence/{block['block_hash'][:8]}/{block['block_hash']}.json"
    s3.put_object(
        Bucket=bucket,
        Key=key,
        Body=json.dumps(block, ensure_ascii=False),
        ServerSideEncryption="AES256"
    )
```

"Capital One은 AWS를 쓰고 있었어. 문제는 **WAF(Web Application Firewall)의 설정 오류** 때문에 SSRF 공격이 가능했고, 공격자가 **IAM 역할의 임시 자격 증명**을 탈취해서 S3 버킷에 접근한 거야."

```
[Capital One 사건 요약 — 2019]

공격 경로:
  1. WAF의 과도한 권한 설정 (SSRF 미차단)
  2. EC2 인스턴스 메타데이터 서비스(IMDS v1)에 접근
     → curl http://169.254.169.254/latest/meta-data/iam/...
  3. IAM 역할의 임시 자격 증명 탈취
  4. S3 버킷에 접근 → 1억 건 이상의 개인정보 유출

피해:
  - 1억 600만 명의 신용카드 신청 정보 유출
  - 벌금 8,000만 달러
  - 브랜드 신뢰도 하락

원인 = 클라우드 설정 오류:
  - WAF에 과도한 IAM 권한 부여
  - IMDS v1 사용 (v2는 토큰 기반)
  - S3 버킷 접근 제어 미흡
```

가람은 수첩에 적었다.

> `[클라우드 설정 오류 — S3 퍼블릭 액세스]`
>
> `AWS S3 퍼블릭 액세스 설정:`
> `  BlockPublicAcls: 퍼블릭 ACL 설정 차단`
> `  IgnorePublicAcls: 기존 퍼블릭 ACL 무시`
> `  BlockPublicPolicy: 퍼블릭 버킷 정책 차단`
> `  RestrictPublicBuckets: 퍼블릭 버킷 접근 제한`
> `  → 4개 모두 true여야 안전`
>
> `점검 포인트:`
> `  1. Public Access Block 설정 확인`
> `  2. 버킷 정책(Bucket Policy) 검토`
> `  3. ACL(Access Control List) 검토`
> `  4. CloudTrail + S3 접근 로그 활성화`
> `  5. 암호화: SSE-S3(AES256) 또는 SSE-KMS`
> `  6. 버전 관리(Versioning) 활성화`
>
> `Capital One 사건 (2019):`
> `  SSRF → IMDS v1 → IAM 자격 증명 탈취 → S3 접근`
> `  1억 600만 명 개인정보 유출, 벌금 8,000만 달러`
> `  원인: 클라우드 설정 오류 (과도한 권한, IMDS v1, S3 미보호)`
>
> `교훈:`
> `  "클라우드의 기본 설정은 안전하지 않다."`
> `  "온프레미스 = 기본 차단, 클라우드 = 기본 허용일 수 있다."`
> `  "설정은 코드처럼 관리하라 (Infrastructure as Code)."`
