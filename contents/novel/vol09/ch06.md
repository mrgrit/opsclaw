# Chapter 6: 감사원이 온다

---

월요일 오전 9시. 넥서스AI 사옥 1층 로비.

SOC 2 외부 감사가 시작되는 날. 관제실은 새벽부터 긴장감이 돌았다.

"감사원 몇 명이 오는 거예요?" 민재가 물었다.

"**3명.** 리드 감사원 1명, 기술 감사원 1명, 주니어 1명." 강혜윤이 말했다. "빅4 감사법인 소속이야."

"빅4라면..." 도윤이 삼켰다.

"감사법인 이름만 들어도 위가 아프다." 진호가 말했다.

추박하는 이례적으로 깔끔한 셔츠를 입고 있었다. 넥타이까지.

"추팀장님이 또 정장이시네요." 가람이 말했다.

"감사원 앞에서 만화책 읽을 수는 없잖아." 추박하가 말했다.

### SOC 2 감사 — 개요

강혜윤이 아침 브리핑을 했다.

```
[SOC 2 Type II 감사 — 개요]

SOC 2란?
  AICPA(미국공인회계사협회)의 Trust Services Criteria 기반
  클라우드/SaaS 기업의 보안 통제 수준을 평가
  Type I: 특정 시점의 통제 설계 평가
  Type II: 일정 기간(보통 6~12개월)의 통제 운영 효과 평가

5가지 Trust Services Criteria:
  1. Security (보안) — 필수
     시스템이 무단 접근으로부터 보호되는가?
  2. Availability (가용성)
     시스템이 약속된 수준으로 사용 가능한가?
  3. Processing Integrity (처리 무결성)
     시스템 처리가 완전하고 정확한가?
  4. Confidentiality (기밀성)
     기밀 정보가 적절히 보호되는가?
  5. Privacy (프라이버시)
     개인정보가 관련 법규에 따라 처리되는가?

넥서스AI 감사 범위:
  - Nexus-1 API 서비스
  - SOC 관제 체계
  - 고객 데이터 처리 프로세스
  - 감사 기간: 2025.12 ~ 2026.06 (6개월)

감사 일정:
  Day 1 (오늘): 킥오프 미팅 + 문서 검토
  Day 2: 기술 통제 검증 (접근제어, 로그, 모니터링)
  Day 3: 인터뷰 (경영진, SOC팀, 개발팀)
  Day 4: 현장 검증 + 증적 확인
  Day 5: 마감 미팅 + 초안 보고
```

"핵심은 **증적(Evidence)**이에요." 강혜윤이 말했다. "모든 통제가 실제로 작동하고 있다는 증거를 제시해야 합니다."

"OpsClaw의 evidence가 여기서 쓰이는 건가요?" 가람이 물었다.

"일부는. **자동화된 모니터링 로그, 인시던트 대응 기록, PoW 체인의 무결성 검증** 등이 SOC 2 증적으로 사용될 수 있어." 김영훈이 말했다.

### Day 1: 킥오프

오전 10시. 대회의실.

리드 감사원 박선호. 50대 초반. 안경 너머의 눈이 날카로웠다. 악수할 때 손이 차가웠다.

기술 감사원 이지은. 30대 후반. 노트북에 뭔가를 계속 타이핑했다. 질문이 예리했다.

주니어 감사원 한승준. 20대 후반. 체크리스트를 들고 조용히 따라다녔다.

"넥서스AI의 SOC 2 감사를 시작하겠습니다." 박선호가 말했다. "먼저 몇 가지 확인하겠습니다. 넥서스AI는 최근 **개인정보보호위원회에 자진 신고**를 하셨죠?"

회의실이 긴장했다.

"맞습니다." 강혜윤이 침착하게 말했다. "Nexus-1 학습 데이터 관련 건입니다. SOC 2 감사와는 별개의 건이지만, 투명하게 공유드립니다."

"감사합니다. 자진 신고 자체는 SOC 2 감사 범위 밖이지만, **관련 통제의 유효성**은 평가 대상입니다." 박선호가 말했다.

추박하가 강혜윤 옆에 앉아 있었다. 표정이 없었다. 20년간 연습한 무표정.

---

### Day 2: 기술 통제 검증

이지은 감사원이 SOC 관제실을 방문했다.

"접근 제어 정책을 보여주세요." 이지은이 말했다.

김영훈이 화면을 보여주었다.

```
[SOC 2 기술 통제 — 접근 제어]

1. 물리적 접근
   - SOC 관제실: 카드키 + 생체인증 (지문)
   - 서버실: 카드키 + 비밀번호 + CCTV
   - 방문자: 에스코트 필수

2. 논리적 접근
   - RBAC (Role-Based Access Control)
   - MFA (다중 인증) 전사 적용
   - 최소 권한 원칙
   - 세션 타임아웃: 15분

3. 네트워크 접근
   - nftables Zone 기반 정책
   - VPN 필수 (원격 접근)
   - 네트워크 세분화 (관제, 개발, 운영, DMZ)

4. 모니터링
   - Wazuh SIEM: 실시간 로그 수집 및 분석
   - OpsClaw Agent Daemon: 24시간 자동 관제
   - 모든 관리자 활동 로그 보존 (1년)
```

"OpsClaw Agent Daemon이요?" 이지은이 물었다. "이건 자체 개발 도구인가요?"

"네." 김영훈이 말했다. "보안 자동화 관제 플랫폼입니다."

"자체 개발 도구의 **보안 통제 자체**는 어떻게 보장합니까?" 이지은이 날카롭게 물었다.

김영훈이 대답했다. "OpsClaw는 **PoW(Proof-of-Work) 체인**으로 모든 작업을 기록합니다. 체인의 무결성은 자동 검증되며, 변조가 불가능합니다."

"PoW 체인이요?" 이지은이 흥미를 보였다. "블록체인 기반인가요?"

"개념은 유사합니다. 각 작업의 해시가 이전 블록에 연결되어 있어서, **중간의 어떤 기록도 수정하면 체인 전체가 깨집니다.**"

```bash
# PoW 체인 무결성 검증 (감사원 시연용)
curl -s -H "X-API-Key: $OPSCLAW_API_KEY" \
  "http://localhost:8000/pow/verify?agent_id=http://localhost:8002"
```

```json
{
  "valid": true,
  "blocks": 15847,
  "orphans": 0,
  "tampered": []
}
```

"15,847개의 블록이 **모두 유효**합니다." 김영훈이 말했다.

이지은이 고개를 끄덕이며 노트북에 타이핑했다. "인상적이네요. 이 수준의 감사 추적(Audit Trail)은 드물어요."

추박하가 구석에서 듣고 있었다. 표정이 미세하게 변했다 — 쓴웃음.

"15,847개"는 **공식 OpsClaw의 기록**이었다. genesis의 40,132건은 — 아직 감사원에게 공개되지 않았다.

### Day 3: 인터뷰

박선호 리드 감사원이 추박하와 1:1 인터뷰를 했다.

"추 팀장님. SOC팀을 몇 년간 이끌고 계시죠?"

"**8년**입니다."

"보안 인시던트 발생 시 대응 프로세스를 설명해 주시겠습니까?"

추박하가 설명했다. NIST IR 6단계, OpsClaw 자동화, 에스컬레이션 체인. 완벽했다.

"학습 데이터 관련 자진 신고 건은 어떻게 발견되었습니까?"

추박하가 잠시 멈추었다. 0.5초. 감사원은 눈치채지 못했지만, 밖에서 모니터링하던 김영훈은 눈치챘다.

"**내부 감사 과정에서** 데이터 흐름의 이상을 발견했고, SOC팀과 법무실이 공동으로 조사한 결과 문제를 확인했습니다."

사실이었다. 하지만 전부는 아니었다. genesis가 빠져 있었다.

"감사합니다." 박선호가 말했다. "넥서스AI의 보안 체계는 상당히 체계적이라는 인상을 받았습니다."

추박하가 고개를 숙였다. "감사합니다."

회의실을 나오며 추박하가 김영훈에게 말했다. "**반쪽짜리 진실.**"

"알고 있습니다." 김영훈이 말했다.

"감사원에게 genesis를 말해야 하는 거 아닌가?"

"SOC 2 감사 범위에 genesis는 포함되지 않습니다. **지금은.**" 김영훈이 말했다. "하지만 검찰 조사가 시작되면, 감사 보고서에 **추가 사항**으로 기재될 수 있어요."

"**거짓말은 하지 않았어.** 하지만 **전부를 말하지도 않았지.**" 추박하가 말했다. "그 차이가 나를 괴롭히는 거야."

---

### Day 4: 서문기

예상치 못한 일이 벌어진 것은 Day 4였다.

감사원들이 인사기록을 검토하던 중, 박선호가 질문했다.

"서문기 상무님과도 인터뷰를 요청드립니다. 데이터 관리 총괄이셨다고 되어 있는데요."

강혜윤이 말했다. "서 상무님은 현재 **직위 해제 상태**입니다."

"사유가 무엇입니까?"

"내부 조사 진행 중입니다. 상세 내용은 공유가 어렵습니다."

박선호의 안경 너머 눈빛이 변했다. 메모를 했다.

그날 오후. 서문기가 감사원에게 **개인적으로 연락을 시도했다**는 보고가 들어왔다.

"서문기가 뭘 한 거예요?" 민재가 놀랐다.

"감사원에게 직접 전화를 걸어서 **자신의 입장을 해명하려 했어.**" 강혜윤이 말했다. 목소리가 차가웠다. "감사 중 **비공식 접촉**은 감사 방해로 간주될 수 있어."

"막을 수 있어요?" 김영훈이 물었다.

"이미 막았어요. 법무팀이 서 상무에게 **접근 금지 통보**를 했고, 감사법인에도 공식 사과를 전달했습니다."

추박하가 말했다. "서문기는 **아직 포기하지 않았어.** 감사 결과에 자기 입장을 넣으려는 거야."
