### 시나리오 브리핑

```
[SOC 종합 IR 훈련 — 시나리오]
설계: 추박하
시간: 09:00 ~ 17:00 (8시간)
참가: SOC팀 전원

Phase 1 (09:00-10:30): 초기 탐지
  - Suricata에서 외부 IP의 포트 스캔 탐지
  - 동시에 Wazuh에서 내부 서버의 비정상 프로세스 알림

Phase 2 (10:30-12:00): 침투 확인
  - 웹 서버에서 SQLi + 파일 업로드 공격 흔적 발견

  - 내부 횡적 이동(Lateral Movement) 흔적
  - 다수 서버에서 비정상 SSH 접속
  - C2 통신 의심 DNS 쿼리
  - 질문: 공격 범위는 어디까지인가?

Phase 4 (14:30-16:00): 억제 & 제거
  - 감염 서버 격리
  - 악성코드 제거
  - 백도어 제거 확인
  - 질문: 공격자의 모든 거점을 제거했는가?

Phase 5 (16:00-17:00): 복구 & 보고
  - 서비스 복구
  - 인시던트 보고서 작성
  - Lessons Learned
  - 질문: 다음에 같은 공격이 오면 무엇을 바꿀 것인가?

역할 배정:
  인시던트 커맨더: 김영훈
  네트워크 분석: 윤정규 + 한도윤
  호스트 분석: 박지후 + 장가람
  CTI/OSINT: 송진호
  커뮤니케이션: 최민재
  OpsClaw 운영: 윤정규 (겸임)
```

"NIST 인시던트 대응 6단계를 **실전으로** 돌리는 거야." 김영훈이 말했다.

```
[NIST IR 6단계 — 복습]

1. 준비 (Preparation)
   - IR 정책, 도구, 팀 구성
   - 이미 완료 (오늘은 이 위에서 시작)

2. 탐지 및 분석 (Detection & Analysis)
   - 이벤트 → 인시던트 판별
   - Phase 1-2

3. 억제 (Containment)
   - 단기 억제: 네트워크 격리, IP 차단
   - 장기 억제: 시스템 패치, 설정 변경
   - Phase 4 전반

4. 제거 (Eradication)
   - 악성코드 제거, 백도어 삭제
   - Phase 4 후반

5. 복구 (Recovery)
   - 서비스 정상화, 모니터링 강화
   - Phase 5 전반

6. 교훈 (Lessons Learned)
   - 사후 분석, 프로세스 개선
   - Phase 5 후반
```

### Phase 1: 초기 탐지 (09:00)

추박하가 시나리오를 발동시켰다.

```bash
# 추팀장이 준비한 시뮬레이션 — Agent Daemon 경유
curl -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $OPSCLAW_API_KEY" \
  -d '{
    "name": "ir-drill-20260611",
    "request_text": "SOC 종합 IR 훈련 — 다중 벡터 동시 공격 시나리오",
    "master_mode": "external"
  }'
```

Suricata alert가 터졌다. 동시에 Wazuh alert가 울렸다.

"Suricata — **외부 IP 103.45.67.89에서 포트 스캔** 탐지." 정규가 보고했다. "TCP SYN scan, 상위 1000개 포트."

"Wazuh — **내부 서버 web-prod-01에서 비정상 프로세스**." 지후가 보고했다. "프로세스명: /tmp/.hidden/agent. UID=www-data."

김영훈이 인시던트 커맨더로서 지시했다. "정규, 해당 외부 IP의 GeoIP와 과거 IOC 조회. 지후, 해당 프로세스의 파일 해시와 네트워크 연결 확인. 진호, 103.45.67.89를 OpenCTI에서 검색."

"민재, Slack에 인시던트 타임라인 채널 생성. 모든 발견 사항을 시간순으로 기록."

"네!" 민재가 대답했다. 손이 떨렸지만 키보드는 정확했다.

### Lessons Learned

"잘한 점." 김영훈이 말했다.

"Agent Daemon의 **초기 탐지가 빨랐어요.**" 정규가 말했다. "사람보다 먼저 alert를 잡았어."

"execute-plan으로 **4개 서버를 동시에 분석**한 것도." 가람이 말했다.

"개선할 점." 김영훈이 말했다.

"Phase 3에서 **횡적 이동 범위를 파악하는 데 시간이 걸렸어요.** SSH 로그만 봤는데, WinRM이나 다른 프로토콜도 확인했어야 했어요." 도윤이 말했다.

"커뮤니케이션이 **중간에 끊긴 적이 있었어요.** Slack 채널이 두 개로 나뉘면서 정보가 분산됐어요." 민재가 반성했다.

추박하가 말했다. "**점수를 매기면 75점.** 훌륭하지는 않지만 합격이야."

"75점이요?" 민재가 실망했다.

"실전에서 75점이면 **회사가 살아남는 거야.** 100점은 교과서에만 있어." 추박하가 말했다.
