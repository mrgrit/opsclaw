### 자동화 관제 — Agent Daemon 아키텍처

```
[OpsClaw Agent Daemon — 24시간 자동 관제 체계]

구성 요소:
  1. Watcher (감시자)
     - 이벤트 소스를 주기적으로 폴링
     - Suricata alert, Wazuh alert, syslog, 웹 로그
     - 임계값 초과 시 Project 자동 생성

  2. Scheduler (스케줄러)
     - cron 기반 반복 작업 관리
     - 일일 취약점 스캔, 주간 보안 점검, 월간 감사
     - 각 작업을 OpsClaw Project로 실행

  3. Agent Daemon (데몬)
     - Watcher + Scheduler의 상위 프로세스
     - SubAgent에 작업 배분
     - 결과를 Manager API에 보고
     - PoW 체인에 모든 activity 기록

  4. Escalation Chain (에스컬레이션)
     - Low: 자동 처리 + Slack 알림
     - Medium: 자동 처리 + 담당자 알림 + 30분 내 확인 요구
     - High: 자동 차단 + 팀장 알림 + 즉시 대응
     - Critical: 자동 차단 + 전원 알림 + dry_run 강제

아키텍처:
  ┌──────────────────────────────────────┐
  │ OpsClaw Manager API (:8000)          │
  │  ├─ Watcher Worker                   │
  │  ├─ Scheduler Worker                 │
  │  └─ Project/Evidence/PoW Engine      │
  └───────────┬──────────────────────────┘
              │ dispatch / execute-plan
  ┌───────────┼──────────────────────────┐
  │           ↓                          │
  │  ┌─────────────┐  ┌─────────────┐   │
  │  │ SubAgent    │  │ SubAgent    │   │
  │  │ secu:8002   │  │ web:8002    │   │
  │  └─────────────┘  └─────────────┘   │
  │  ┌─────────────┐  ┌─────────────┐   │
  │  │ SubAgent    │  │ SubAgent    │   │
  │  │ siem:8002   │  │ local:8002  │   │
  │  └─────────────┘  └─────────────┘   │
  └──────────────────────────────────────┘
```

정규가 화면에 구성 파일을 보여주었다.

```yaml
# /etc/opsclaw/watcher-config.yaml

  - name: suricata_alert_burst
    source: suricata
    query: "alert.severity <= 2"

      create_project: true

      subagent_url: "http://192.168.208.150:8002"

  - name: wazuh_critical_alert
    source: wazuh
    query: "rule.level >= 12"
    interval: 10s
    threshold: 1
    window: 1m
    action:
      create_project: true
      risk_level: critical
      playbook: incident_response_host
      subagent_url: "http://192.168.208.152:8002"

  - name: web_attack_pattern
    source: bunkerweb
    query: "status_code IN (403, 429) AND path LIKE '/api/%'"
    interval: 60s
    threshold: 20
    window: 10m
    action:
      create_project: true
      risk_level: medium
      playbook: web_attack_response
      subagent_url: "http://192.168.208.151:8002"

    source: auth_log

    threshold: 1

      create_project: true
      risk_level: medium
```

"**4개의 Watcher**가 상시 돌아가고 있어." 정규가 말했다. "Suricata, Wazuh, BunkerWeb, Auth 로그를 각각 모니터링하면서, 임계값을 넘으면 **자동으로 Project를 생성하고 Playbook을 실행**해."

### Scheduler 설정

```yaml
# /etc/opsclaw/scheduler-config.yaml
schedules:
  - name: daily_vuln_scan
    cron: "0 2 * * *"  # 매일 새벽 2시
    playbook: vulnerability_scan
    subagent_url: "http://localhost:8002"
    risk_level: low

  - name: weekly_security_check
    cron: "0 3 * * 1"  # 매주 월요일 새벽 3시
    playbook: security_health_check
    subagent_url: "http://localhost:8002"
    risk_level: low

  - name: monthly_compliance_audit
    cron: "0 4 1 * *"  # 매월 1일 새벽 4시
    playbook: compliance_check
    subagent_url: "http://localhost:8002"
    risk_level: medium

  - name: genesis_integrity_check
    cron: "*/30 * * * *"  # 30분마다
    playbook: genesis_pow_verify
    subagent_url: "http://localhost:8002"
    risk_level: low
```

"genesis_integrity_check가 **30분마다** 돌아가요?" 가람이 놀랐다.

"genesis의 PoW 체인 무결성을 **자동 검증**하는 거야." 정규가 말했다. "누군가 evidence를 변조하면 즉시 탐지돼."

"추팀장님 지시인가요?" 민재가 물었다.

추박하가 만화책 — 아니, 커피잔 너머로 고개를 끄덕였다. "genesis는 **스스로를 감시해야 해.** 감시자도 감시받아야 하니까."

### 야간 무인 운영

"이제 **야간에 사람이 없어도** 기본 대응은 가능해." 정규가 말했다.

"야간 교대 폐지하는 건가요?" 도윤이 기대에 찬 눈으로 물었다.

"폐지가 아니라 **축소.**" 김영훈이 말했다. "야간 교대 인원을 2명에서 1명으로 줄이고, Agent Daemon이 1차 대응을 담당해. 사람은 **에스컬레이션된 건만** 처리하면 돼."

"야간에 1명이면 **누가** 하는 건데요..." 민재가 불안한 눈으로 물었다.

"로테이션이야." 김영훈이 말했다. 민재의 불안이 해소되지는 않았다.

추박하가 말했다. "Agent Daemon은 **내가 20년간 하고 싶었던 것**이야. 사람이 자지 않고 감시할 수는 없어. 하지만 시스템은 잘 수 있어 — **필요할 때 깨어나면 되니까.**"

---

가람은 수첩에 적었다.

> `[자동화 관제 — Agent Daemon]`
>
> `구성: Watcher(감시) + Scheduler(반복) + Daemon(통합) + Escalation(단계별 대응)`
>
> `Watcher 4종:`
> `  suricata_alert_burst: 30초 주기, 5건/5분 임계값`
> `  wazuh_critical_alert: 10초 주기, 1건/1분`
> `  web_attack_pattern: 60초 주기, 20건/10분`
> `  login_anomaly: 60초 주기, 10회 실패/3개 IP`
>
> `Scheduler 4종:`
> `  일일 취약점 스캔, 주간 보안 점검, 월간 감사, genesis 무결성(30분마다)`
>
> `성과:`
> `  alert → 대응: 30분 → 50초`
> `  false positive: 15% → 4.2% (RL 학습)`
> `  야간 교대: 2명 → 1명으로 축소`
>
> `핵심 원칙:`
> `  "자동화는 속도를 제공하고, 사람은 판단을 제공한다"`
> `  → 완전 자동화가 아닌 Human-in-the-Loop`
>
> `추팀장의 말:`
> `  "시스템은 잘 수 있어 — 필요할 때 깨어나면 되니까."`
