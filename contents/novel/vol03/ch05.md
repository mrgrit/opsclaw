# Chapter 5: 눈먼 파수꾼

---

목요일 오전. 외부 보안 업체 "시큐리온(Securion)"의 전문가 2명이 도착했다. 포렌식 전문가와 위협 헌팅 전문가.

김영훈이 현재까지의 상황을 브리핑했다. 시큐리온 측은 고개를 끄덕이며 들었고, 한 마디로 요약했다.

"APT급이네요. 킬체인이 완전하고, SIEM까지 무력화했으면 **상당히 숙련된 그룹**입니다."

"그룹이라고 보시나요? 개인이 아니라?" 김영훈이 물었다.

"5개 IP 동시 공격, 다단계 인프라(IAB + VPS + 유사 도메인), SIEM 무력화까지. 이건 **팀 단위 작전**이에요."

---

시큐리온 팀이 가장 먼저 한 일은 **Wazuh의 무결성을 확인**하는 것이었다.

"SIEM이 10일간 부분적으로 무력화된 상태였으니, 이 기간의 **Wazuh 경보가 누락되었을 수 있어요**." 포렌식 전문가가 말했다.

```bash
# Wazuh 경보 시간대별 건수 확인 — 누락 기간 식별
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"echo 1 | sudo -S cat /var/ossec/logs/alerts/alerts.json 2>/dev/null | python3 -c \"import sys,json\nfrom collections import Counter\ndates=Counter()\nfor l in sys.stdin:\n try:\n  a=json.loads(l)\n  d=a.get(\\\"timestamp\\\",\\\"\\\")[:10]\n  dates[d]+=1\n except: pass\nfor d in sorted(dates): print(f\\\"  {d}: {dates[d]}건\\\")\"","subagent_url":"http://10.20.30.100:8002"}'
```

"4월 3~5일, 3일간 경보가 **70% 감소**했어요." 포렌식 전문가가 말했다. "이 기간에 발생한 이벤트 중 상당수가 **SIEM에 전달되지 않았을** 가능성이 높아요."

"3일간의 사각지대." 김영훈이 중얼거렸다.

"**파수꾼이 눈을 감은 3일**이요." 가람이 수첩에 적었다.

---

"그 3일간 공격자가 뭘 했는지를 **Suricata 쪽 로그**에서 복원할 수 있어요." 위협 헌팅 전문가가 말했다. "Suricata는 secu 서버에서 동작하니까, web이나 siem의 무력화에 영향을 안 받아요."

"단, Suricata는 **패킷 수준**이라 어플리케이션 레벨 행동은 안 보여요. 하지만 **네트워크 흐름**은 볼 수 있어요."

```bash
# 4월 3~5일 Suricata 네트워크 흐름 (web 서버 관련)
sshpass -p1 ssh -o StrictHostKeyChecking=no secu@10.20.30.1 \
  "echo 1 | sudo -S cat /var/log/suricata/eve.json 2>/dev/null | \
   python3 -c \"
import sys, json
for l in sys.stdin:
    try:
        e = json.loads(l)
        ts = e.get('timestamp','')
        if '2026-04-03' <= ts[:10] <= '2026-04-05' and e.get('src_ip') == '10.20.30.80':
            if e.get('event_type') in ('flow','http','dns'):
                dst = e.get('dest_ip','?')
                if dst not in ('10.20.30.1','10.20.30.201'):
                    print(f'{ts[:19]} → {dst}:{e.get(\"dest_port\",\"?\")} [{e.get(\"event_type\")}]')
    except: pass
\" 2>/dev/null | head -15"
```

"4월 5일에 **443 포트(HTTPS)**로 통신이 있어요." 위협 헌팅 전문가가 가리켰다. "C2가 80번(HTTP)에서 **443(HTTPS)으로 전환**한 것 같아요. 암호화 채널이면 **내용을 볼 수 없어요**."

"ICMP 터널링에 이어 HTTPS로도 통신 채널을 열었구나." 진호가 말했다. "**다중 채널 C2**야."
