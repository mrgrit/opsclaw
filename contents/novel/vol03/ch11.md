# Chapter 11: agent_id: internal-ops

---

수요일. 가람은 증적 정리 업무 중에 OpsClaw의 PoW 블록 데이터를 분석하고 있었다.

일상적인 작업이었다 — 각 프로젝트의 블록 수, 보상 합계, 에이전트별 통계를 엑셀로 정리하는 것.

그런데 가람은 **leaderboard**에서 이상한 점을 발견했다.

```bash
curl -s http://localhost:8000/pow/leaderboard \
  -H "X-API-Key: opsclaw-api-key-2026" | python3 -c "
import sys,json
d = json.load(sys.stdin)
for a in d.get('leaderboard',[]):
    print(f'  {a[\"agent_id\"]:40s} balance={a[\"balance\"]:10.1f} tasks={a[\"total_tasks\"]:6d}')
"
```

```
  http://localhost:8002                    balance=     198.6 tasks=   187
```

에이전트가 **하나**밖에 없었다. `http://localhost:8002`.

하지만 가람은 이전에 추박하의 화면에서 다른 agent_id를 본 적이 있었다.

`internal-ops`

가람은 직접 확인해보기로 했다.

```bash
curl -s "http://localhost:8000/pow/verify?agent_id=internal-ops" \
  -H "X-API-Key: opsclaw-api-key-2026"
```

```json
{
  "status": "error",
  "detail": "Access denied"
}
```

접근 거부. genesis 프로젝트와 마찬가지로.

가람은 다른 방법을 시도했다.

```bash
# DB에서 직접 확인할 수는 없지만, PoW blocks API에 agent_id 필터가 있으면...
curl -s "http://localhost:8000/pow/blocks?agent_id=internal-ops" \
  -H "X-API-Key: opsclaw-api-key-2026"
```

```json
{
  "status": "error",
  "detail": "Unauthorized"
}
```

역시 접근 불가.

---

가람은 잠시 생각했다. **일반 API Key로는 접근할 수 없는 agent_id.**

하지만 PoW 블록은 DB에 저장되어 있다. 만약 DB에 직접 접근할 수 있다면...

*안 돼. 그건 권한을 넘는 거야.*

가람은 수첩에 적었다.

> `agent_id: internal-ops`
> `일반 API Key로 접근 불가. genesis와 같은 보호 수준.`
> `leaderboard에도 나타나지 않음.`
> `추팀장이 별도 API Key를 사용하는 것과 관련?`

---

가람은 다른 접근을 시도했다. PoW 블록의 **총 수**를 확인하는 것.

```bash
# 전체 PoW 블록 수 (verify API)
curl -s "http://localhost:8000/pow/verify?agent_id=http://localhost:8002" \
  -H "X-API-Key: opsclaw-api-key-2026"
```

```json
{
  "result": {
    "agent_id": "http://localhost:8002",
    "valid": true,
    "blocks": 187,
    "orphans": 0,
    "tampered": []
  }
}
```

`http://localhost:8002`의 블록: **187개.**

하지만 추박하의 화면에서 본 전체 PoW 체인: **47,913개.**

`47,913 - 187 = 47,726`

47,726개의 블록이 `http://localhost:8002`가 **아닌** agent_id에 속해 있었다.

*그 47,726개가 전부 `internal-ops`의 블록이라면...*

가람은 계산했다. 넥서스AI 설립 2005년부터 2026년까지 21년. 47,726 ÷ (21 × 365) ≈ **6.2개/일.**

*하루 평균 6.2개의 기록. 20년 넘게. 매일.*

*추박하는 20년 넘게 매일 6~7건의 무언가를 기록해왔다.*

---

가람은 수첩의 새 페이지를 펴고 지금까지 알게 된 것을 모두 정리했다.

```
[genesis 프로젝트 — 알려진 사실]

프로젝트 ID: prj_00000000 (시스템 최초)
접근 권한: 일반 API Key로 접근 불가
agent_id: internal-ops (leaderboard에 미표시)
블록 수: ~47,726개 (전체 47,913 - 일반 187)
기간: 2005년~현재 (약 21년)
빈도: 하루 평균 6~7건

추팀장의 행동:
- 새벽에 혼자 와서 genesis에 dispatch
- 회장과의 회의, 상무와의 협의 내용을 기록
- 별도 API Key 사용
- "절대 삭제하면 안 된다"
- "이 시스템은 나를 증명해야 한다"

추정:
genesis = 넥서스AI의 20년간 모든 중요 결정/사건/비밀의 기록
추박하 = 기록자 (recorder)
목적 = ?
```

가람은 마지막 줄에서 멈추었다.

*목적은 뭘까?*

*보호? 보험? 증거? 무기?*

*추박하가 20년간 매일 기록한 이유는... 무엇일까?*

---

그날 저녁, 가람은 퇴근길에 추박하를 마주쳤다. 엘리베이터 앞.

"팀장님."

"응."

"한 가지만 물어봐도 될까요?"

추박하가 가람을 바라보았다.

"genesis는... 뭔가요?"

긴 침묵. 엘리베이터가 도착하고, 문이 열리고, 닫혔다. 두 사람은 타지 않았다.

추박하가 말했다.

"가람아. 세상에는 두 종류의 사람이 있어."

"네?"

"기록하는 사람과, 기록되는 사람."

"..."

"나는 기록하는 사람이야. 그리고 너도 그래."

추박하가 엘리베이터 버튼을 눌렀다.

"하지만 기록은 **그 자체로는 아무것도 아니야**. 기록이 의미를 갖는 건, **누군가가 그것을 읽을 때**야."

문이 열렸다. 추박하가 들어갔다.

"아직은 읽을 때가 아니야. 하지만 **곧** 올 거야."

문이 닫혔다.

가람은 닫힌 엘리베이터 문 앞에 서 있었다.

*"곧 올 거야."*

*무엇이?*
