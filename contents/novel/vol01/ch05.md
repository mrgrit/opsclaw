# Chapter 5: 낮은 울타리

---

월요일 오전. 김영훈은 정기 취약점 점검을 시작했다.

매분기 한 번, 넥서스AI의 모든 서버에 대해 보안 설정을 검토하는 작업이었다. ISO 27001 Annex A.8 기술적 통제 항목에 따른 점검이다.

"가람, 오늘 점검 보조 해. 내가 실행하는 명령 전부 기록해." 김영훈이 아침부터 가람을 불렀다.

"네."

"점검은 OpsClaw execute-plan으로 한다. 모든 결과가 evidence로 남아야 하니까."

가람은 노트북을 열고 옆자리에 앉았다.

---

김영훈이 OpsClaw에 점검 프로젝트를 생성했다.

```bash
PID=$(curl -s -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"name":"quarterly-audit-2026Q1","request_text":"2026년 1분기 정기 보안 점검","master_mode":"external"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['project']['id'])")
```

"이 프로젝트 ID를 적어놔. 점검 보고서에 들어갈 거야."

그가 plan→execute stage 전환을 마치고, 첫 번째 점검 항목을 실행했다.

---

### 점검 항목 1: A.8.5 안전한 인증 — 패스워드 정책

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/execute-plan" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "tasks": [
      {"order":1,"title":"opsclaw 패스워드정책","instruction_prompt":"grep -E \"PASS_MAX|PASS_MIN|PASS_WARN\" /etc/login.defs | grep -v \"^#\"","risk_level":"low","subagent_url":"http://localhost:8002"},
      {"order":2,"title":"secu 패스워드정책","instruction_prompt":"grep -E \"PASS_MAX|PASS_MIN|PASS_WARN\" /etc/login.defs | grep -v \"^#\"","risk_level":"low","subagent_url":"http://10.20.30.1:8002"},
      {"order":3,"title":"web 패스워드정책","instruction_prompt":"grep -E \"PASS_MAX|PASS_MIN|PASS_WARN\" /etc/login.defs | grep -v \"^#\"","risk_level":"low","subagent_url":"http://10.20.30.80:8002"},
      {"order":4,"title":"siem 패스워드정책","instruction_prompt":"grep -E \"PASS_MAX|PASS_MIN|PASS_WARN\" /etc/login.defs | grep -v \"^#\"","risk_level":"low","subagent_url":"http://10.20.30.100:8002"}
    ],
    "subagent_url":"http://localhost:8002",
    "parallel":true
  }'
```

결과가 돌아왔다. 4대 서버 모두 동일:

```
PASS_MAX_DAYS   99999
PASS_MIN_DAYS   0
PASS_WARN_AGE   7
```

김영훈이 화면을 보며 혀를 찼다.

"PASS_MAX_DAYS 99999. 사실상 **패스워드 만료 없음**."

가람이 물었다. "그게 왜 문제인 건가요?"

"ISO 27001 A.8.5는 '안전한 인증'을 요구해. 패스워드가 영원히 바뀌지 않으면, 유출됐을 때 **영원히** 악용 가능하다는 뜻이야. 최소 90일마다 변경을 강제해야 해."

김영훈이 점검 결과를 기록했다.

```
| 통제 ID | 항목 | 현재 상태 | 판정 |
|---------|------|---------|------|
| A.8.5   | 패스워드 최대 사용 기간 | 99999일 (무제한) | ❌ 부적합 |
| A.8.5   | 패스워드 최소 사용 기간 | 0일 (즉시 변경 가능) | ⚠️ 권고 |
| A.8.5   | 패스워드 만료 경고 | 7일 전 | ✓ 적합 |
```

"부적합이 하나 나왔으니, 시정 조치 계획도 써야 해."

### 점검 항목 2: A.8.5 안전한 인증 — sudo 권한

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/execute-plan" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "tasks": [
      {"order":1,"title":"web sudo 권한","instruction_prompt":"sudo -l 2>/dev/null","risk_level":"low","subagent_url":"http://10.20.30.80:8002"}
    ],
    "subagent_url":"http://localhost:8002"
  }'
```

결과:

```
User web may run the following commands on web:
    (ALL : ALL) ALL
    (ALL) NOPASSWD: ALL
```

김영훈의 눈이 가늘어졌다.

"NOPASSWD ALL." 그가 낮은 목소리로 말했다. "web 사용자가 **패스워드 없이 root 명령을 실행**할 수 있어. 이건 부적합이 아니라 **긴급 시정**이야."

"그렇게 심각한 건가요?" 가람이 물었다.

김영훈이 가람을 바라보았다. "SSH 비밀번호 하나만 뚫으면, 그 서버의 **모든 것**을 할 수 있다는 뜻이야. 사용자 생성, 파일 삭제, 서비스 정지, 로그 삭제까지. 지난주 SSH 브루트포스 때 guest03 계정이 만들어진 것도 이것 때문이야."

그가 점검표에 붉은 글씨로 적었다.

```
| A.8.5 | sudo NOPASSWD:ALL | web 서버 | 🔴 긴급 시정 |
```

### 점검 항목 3: A.8.15 로깅

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/execute-plan" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "tasks": [
      {"order":1,"title":"siem auditd","instruction_prompt":"systemctl is-active auditd","risk_level":"low","subagent_url":"http://10.20.30.100:8002"},
      {"order":2,"title":"siem wazuh","instruction_prompt":"systemctl is-active wazuh-manager","risk_level":"low","subagent_url":"http://10.20.30.100:8002"},
      {"order":3,"title":"siem rsyslog","instruction_prompt":"systemctl is-active rsyslog","risk_level":"low","subagent_url":"http://10.20.30.100:8002"},
      {"order":4,"title":"web auth.log","instruction_prompt":"ls -la /var/log/auth.log 2>/dev/null && echo OK || echo MISSING","risk_level":"low","subagent_url":"http://10.20.30.80:8002"}
    ],
    "subagent_url":"http://localhost:8002",
    "parallel":true
  }'
```

결과:

```
[1] auditd: active
[2] wazuh-manager: active
[3] rsyslog: active
[4] auth.log: OK
```

"로깅은 정상." 김영훈이 고개를 끄덕였다. "auditd, Wazuh, rsyslog 모두 동작 중. auth.log도 있어."

```
| A.8.15 | 시스템 감사 로깅 | auditd active, Wazuh active | ✓ 적합 |
| A.8.15 | 원격 로그 수집 | rsyslog active | ✓ 적합 |
| A.8.15 | 인증 로그 보존 | auth.log 존재 | ✓ 적합 |
```

### 점검 항목 4: A.8.9 설정 관리 — SSH

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"grep -E \"PermitRootLogin|PasswordAuthentication|MaxAuthTries\" /etc/ssh/sshd_config | grep -v \"^#\"","subagent_url":"http://10.20.30.80:8002"}'
```

결과:

```
(빈 출력)
```

"빈 출력?" 가람이 물었다.

"설정 파일에 해당 항목이 **주석 처리**되어 있거나 **기본값**을 쓰고 있다는 뜻이야." 김영훈이 설명했다. "PasswordAuthentication의 기본값은 `yes`. 즉, **비밀번호 로그인이 허용**되고 있어."

"키 기반 인증만 쓰는 게 더 안전하지 않나요?"

"맞아. 하지만 현실에서 키 기반으로 전환하려면 모든 관리자에게 키를 배포해야 하고, 키 분실 대비 복구 절차도 필요해. 그래서 많은 회사가 '나중에 하겠다'며 비밀번호 인증을 그대로 두지."

김영훈이 가람을 바라보았다.

"보안에서 가장 무서운 말이 뭔지 알아?"

"... 뭔데요?"

"**'나중에 하겠다.'**" 김영훈이 씁쓸하게 웃었다. "'나중'은 사고가 난 다음이 되기 십상이야."

```
| A.8.9 | SSH 패스워드 인증 | 활성화(기본값) | ⚠️ 권고: 키 기반 전환 |
| A.8.9 | SSH root 로그인 | 기본값(허용) | ⚠️ 권고: 비활성화 |
| A.8.9 | SSH 최대 시도 | 기본값(6회) | ⚠️ 권고: 3회로 제한 |
```

---

오후 2시. 점검이 끝났다. 김영훈이 점검 결과를 정리했다.

```
[2026년 1분기 정기 보안 점검 요약]

적합: 5건
권고: 4건
부적합: 1건 (패스워드 미만료)
긴급 시정: 1건 (sudo NOPASSWD:ALL)

총 11건 점검, 6건 개선 필요
```

"이걸 누구한테 보고하나요?" 가람이 물었다.

"보안 담당 임원에게 먼저 가고, 그다음 컴플라이언스 실장(강혜윤)에게 간다." 김영훈이 대답했다. "그리고 점검 evidence는 OpsClaw에 남아 있으니까, 나중에 인증 심사 때 '점검을 수행했다'는 증거로 쓸 수 있어."

```bash
# 점검 프로젝트 완료 보고
curl -s -X POST "http://localhost:8000/projects/$PID/completion-report" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "summary": "2026년 1분기 정기 보안 점검 완료",
    "outcome": "partial",
    "work_details": ["패스워드정책 점검","sudo 권한 점검","로깅 점검","SSH 설정 점검"],
    "issues": ["PASS_MAX_DAYS=99999 부적합","sudo NOPASSWD:ALL 긴급 시정"],
    "next_steps": ["web 서버 sudo 제한 설정","전 서버 PASS_MAX_DAYS=90 적용"]
  }'
```

가람은 그 과정을 지켜보며 배웠다. 점검은 명령을 실행하는 것이 아니라, **결과를 판단하고 기록하는 것**이라는 것을.

그리고 하나 더 — 넥서스AI의 보안이 생각만큼 견고하지 않다는 것을.

*99999일 패스워드. NOPASSWD ALL. SSH 비밀번호 인증.*

이 회사는 국내 최고 수준의 보안 솔루션을 **만드는** 회사인데, 정작 **자기 서버의 보안**은 이 정도인가?

---

점검 보고서를 보낸 후, 김영훈이 가람에게 말했다.

"가람."

"네."

"이 보고서 내용은 팀 외부에 말하지 마. 특히 AI 연구소(서문기 상무)나 인프라 본부(정태식 본부장)에는 절대."

"왜요?"

김영훈의 눈이 차가워졌다.

"sudo 문제를 알게 되면, 정태식 본부장이 가장 먼저 할 일이 뭔지 알아? **'지금 바꾸면 서비스가 죽는다'**며 시정을 막는 거야. 그 사람은 항상 그래. 보안보다 가용성. 우리가 고칠 때까지 **아무도 모르게** 해야 해."

가람은 고개를 끄덕였다. 하지만 속으로는 생각했다.

*팀 내부에서도 비밀이 있고, 팀 외부와도 비밀이 있다. 이 회사에는 도대체 얼마나 많은 비밀이 있는 걸까.*

관제실 안쪽에서 추박하가 만화책을 덮고 기지개를 켰다. 하품을 하며 일어서더니, 아무에게도 말없이 관제실을 나갔다.

"팀장님 어디 가시는 거예요?" 가람이 물었다.

민재가 대답했다. "글쎄. 가끔 저렇게 사라져. 몇 시간 후에 돌아올 때도 있고, 내일 올 때도 있고."

*그는 어디로 가는 걸까.*
