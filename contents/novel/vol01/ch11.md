# Chapter 11: 닫히지 않은 문

---

수요일. 박지후는 IT 자산관리팀에서 넘겨받은 노트북 한 대를 분석하고 있었다.

지난달 퇴직한 직원의 업무용 노트북. 부서는 AI 연구소. 이름은 — 비밀이었다. 인사실(진수현 실장)이 "내부 조사 중"이라며 이름을 공개하지 않았다.

"지후 선배, 이런 건 보통 얼마나 걸려요?" 가람이 물었다.

지후가 화면에서 눈을 떼지 않고 대답했다.

"하루에서 일주일. 뭘 찾느냐에 따라 다르지."

지후의 모니터에는 디지털 포렌식 도구가 열려 있었다. 하드디스크의 파일 시스템을 분석하는 화면.

---

### 포렌식 분석 과정

지후는 가람에게 설명하며 작업했다.

"포렌식의 첫 번째 원칙이 뭔지 알아?"

"증거 보존이요?"

"맞아. **원본을 건드리지 않는 것.**" 지후가 노트북을 가리켰다. "이 디스크의 이미지를 먼저 떠서, 이미지로만 작업해. 원본은 밀봉해서 증거실에 보관."

```bash
# 디스크 이미지 생성 (dd)
dd if=/dev/sda of=/evidence/case-20260328/disk.img bs=4M status=progress
# 해시 무결성 확인
sha256sum /evidence/case-20260328/disk.img > /evidence/case-20260328/disk.sha256
```

"해시를 떠두면, 나중에 '이미지를 조작하지 않았다'는 증거가 돼."

---

지후는 이미지 파일에서 파일 시스템을 분석했다.

"먼저 **최근 파일 접근 기록**을 볼 거야."

```bash
# MFT(Master File Table) 분석 — NTFS 파일 시스템의 메타데이터
# 최근 수정된 파일 목록
find /mnt/evidence -mtime -30 -type f | head -20

# Prefetch 파일 분석 — 최근 실행된 프로그램
ls -la /mnt/evidence/Windows/Prefetch/ | head -15
```

화면에 파일 목록이 나타났다. 가람은 하나씩 확인했다.

```
/Users/user/Documents/research_notes.docx
/Users/user/Documents/data_pipeline_config.xlsx
/Users/user/Downloads/competitor_report_2025.pdf
/Users/user/AppData/Local/Temp/upload_20260215.zip
```

"**competitor_report_2025.pdf**?" 가람이 화면을 가리켰다.

지후의 표정이 변했다. "경쟁사 보고서?"

그가 파일을 열었다. PDF의 첫 페이지:

```
[Confidential]
AlphaAI Corporation
Internal Research Report 2025
"Foundation Model Architecture: Lessons from GPT-5 Benchmark"
```

"이건..." 지후가 말을 멈추었다. "**AlphaAI의 내부 연구 보고서**야. 이걸 왜 넥서스AI 직원이 갖고 있어?"

---

지후는 파일의 메타데이터를 확인했다.

```bash
# PDF 메타데이터 추출
exiftool competitor_report_2025.pdf
```

```
File Name          : competitor_report_2025.pdf
Creator            : Kim, Jihye (AlphaAI Research)
Create Date        : 2025:08:15 14:30:22
Modify Date        : 2025:08:15 14:30:22
Producer           : Microsoft® Word 2019
```

"작성자가 AlphaAI의 김지혜. 생성일 2025년 8월." 지후가 말했다.

"어떻게 입수한 거죠?" 가람이 물었다.

"그걸 알아내야지." 지후가 `/Downloads` 폴더의 브라우저 히스토리를 확인했다.

```bash
# Chrome 브라우저 히스토리 (SQLite DB)
sqlite3 "/mnt/evidence/Users/user/AppData/Local/Google/Chrome/User Data/Default/History" \
  "SELECT url, title, last_visit_time FROM urls ORDER BY last_visit_time DESC LIMIT 20;"
```

결과 중 흥미로운 줄:

```
https://drive.google.com/file/d/1Ax3B...  | AlphaAI_research_2025 | 2026-02-14
https://mega.nz/file/XXXXXX              | data_package_feb      | 2026-02-15
```

"Google Drive와 Mega에서 다운로드했어." 지후가 말했다. "누군가가 **클라우드에 올려둔 걸** 이 직원이 받은 거야."

---

"이건 김영훈 과장에게 바로 보고해야 해." 지후가 말했다.

"경쟁사 내부 문서를 입수한 건... 산업 스파이 아닌가요?" 가람이 조심스럽게 물었다.

지후가 가람을 바라보았다.

"가람아. 이 얘기는 **너한테서 나가면 안 돼**. 알겠지?"

"네."

"포렌식에서 발견한 건 아직 **사실 확인**이 안 된 거야. 이 파일이 어떻게 여기 왔는지, 이 직원이 스스로 구한 건지, 누가 시킨 건지 아직 몰라. 섣부른 결론은 사람을 죽여."

가람은 고개를 끄덕였다. 하지만 머릿속에는 이미 질문이 떠올랐다.

*AlphaAI의 내부 보고서가 넥서스AI 연구원의 노트북에.*

*Nexus-1 학습 데이터의 출처가 의심스럽다던 이다겸의 말.*

*추팀장이 '학습 데이터 문제'라고 했던 것.*

*"정리된 게 아니라 묻은 것".*

조각들이 하나씩 맞춰지기 시작했다. 하지만 아직 완전한 그림은 보이지 않았다.

---

그날 저녁, 김영훈이 지후의 분석 결과를 보고 받았다.

"competitor_report_2025.pdf." 김영훈이 파일명을 되뇌었다. "이거 서문기 상무에게 보고해야 하나?"

"서문기 상무요?" 지후가 의아한 표정을 지었다. "이건 법무/컴플라이언스(강혜윤 실장)에게 먼저 가야 하는 거 아닙니까?"

"그래, 맞아." 김영훈이 고개를 끄덕였다. "강혜윤 실장에게 먼저."

"영훈 과장." 지후가 조심스럽게 말했다. "이 파일이 **AI 연구소** 직원의 노트북에서 나왔잖아요. 서문기 상무가 알면..."

"상무가 뭘 하겠어?"

"**덮을 수도** 있죠."

김영훈이 잠시 침묵했다.

"우선 증거를 OpsClaw에 기록해. evidence로."

```bash
curl -s -X POST "http://localhost:8000/projects/$FORENSIC_PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"sha256sum /evidence/case-20260328/competitor_report_2025.pdf","subagent_url":"http://localhost:8002"}'
```

"해시가 기록되면, **나중에 파일이 사라져도** 이 파일이 존재했다는 증거가 OpsClaw에 남아."

지후가 고개를 끄덕였다. "영훈 과장답네요."

"뭐가?"

"증거부터 확보하는 거."

---

가람은 자리에서 그 대화를 들었다.

> `AI 연구소 퇴직자 노트북에서 경쟁사(AlphaAI) 내부 보고서 발견.`
> `Google Drive + Mega 경유 다운로드.`
> `서문기 상무에게 알리면 덮을 수도? → 강혜윤 실장에게 먼저.`
> `evidence로 해시 기록 완료 — 파일이 사라져도 증거는 OpsClaw에.`

가람은 수첩을 닫으며 생각했다.

*이 회사에는 문이 많다. 그리고 그 문들은... 닫히지 않고 있다.*
