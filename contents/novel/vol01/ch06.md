# Chapter 6: 모든 것을 기록한다

---

수요일 오후 2시. SOC 팀 전체 회의.

관제실의 대형 모니터에 프레젠테이션이 띄워졌다. 제목:

> **OpsClaw v1.0 — 공식 가동 보고**

발표자는 최민재. 하지만 자료를 만든 것은 — 모두가 알고 있었다 — 김영훈이었다.

"자, 다들 알다시피 OpsClaw가 지난달 내부 테스트를 마쳤고, 오늘부터 **정식 운영**에 들어갑니다." 민재가 화면을 넘겼다.

가람은 수첩을 펼쳤다. 입사 이후 가장 관심 있는 주제였다.

---

"OpsClaw의 핵심은 세 가지입니다."

민재가 화이트보드에 그림을 그렸다.

```
┌──────────┐
│  Master  │ ← 계획을 세우는 사람 (LLM 또는 사람)
│  (두뇌)  │
└────┬─────┘
     │ REST API
┌────┴─────┐
│ Manager  │ ← 실행을 관리하고 기록하는 중앙 관제
│ (관리자) │   프로젝트, 증적, 보상, Playbook
└────┬─────┘
     │ 태스크 위임
┌────┴─────┐
│SubAgent  │ ← 실제 명령을 실행하는 현장 요원
│  (손)    │   각 서버에 1개씩 배치
└──────────┘
```

"Master가 '이걸 해'라고 지시하면, Manager가 '어떤 서버에서, 어떤 순서로, 어떤 위험도로 실행할지' 관리하고, SubAgent가 실제로 실행합니다."

"Master는 누구예요?" 도윤이 물었다.

"두 가지 모드가 있어." 민재가 화면을 넘겼다.

```
Mode A (Native): OpsClaw 내장 LLM이 계획 수립
Mode B (External): 외부 AI(예: Claude Code)가 API로 직접 호출
```

"지금은 Mode B를 주로 씁니다. 우리가 직접 API를 호출하는 거죠. 나중에 Mode A가 완성되면 LLM이 자율적으로 계획을 세울 수도 있습니다."

송진호가 끼어들었다. "Mode A면 AI가 알아서 서버를 점검하는 거예요? 사람 없이?"

"이론적으로는요." 민재가 조심스럽게 말했다. "하지만 `risk_level=critical`인 태스크는 **자동으로 dry_run**이 강제됩니다. 사람이 확인하기 전까지 실제 실행은 안 돼요."

"그건 누가 정한 거야?" 박지후가 물었다.

민재가 관제실 안쪽을 가리켰다. 추박하의 자리. 추박하는 이번에도 만화책을 보고 있었다.

"팀장님이요."

---

"두 번째 핵심은 **증적(Evidence)**입니다."

```
실행 → evidence 기록 → PoW 블록 생성 → 보상(reward) 산출
```

"모든 태스크는 실행 즉시 evidence로 기록됩니다. command, stdout, stderr, exit_code, 실행 시간. 그리고 각 evidence에 대해 **PoW 블록**이 자동 생성됩니다."

"PoW가 뭐예요?" 가람이 물었다.

"Proof of Work. 작업 증명." 민재가 화이트보드에 그림을 그렸다.

```
Block #1          Block #2          Block #3
┌──────────┐     ┌──────────┐     ┌──────────┐
│ 작업 내용 │     │ 작업 내용 │     │ 작업 내용 │
│ 실행 결과 │     │ 실행 결과 │     │ 실행 결과 │
│          │     │          │     │          │
│ prev_hash│ ←── │ prev_hash│ ←── │ prev_hash│
│ block_hash│    │ block_hash│    │ block_hash│
└──────────┘     └──────────┘     └──────────┘
```

"각 블록의 `block_hash`는 이전 블록의 해시를 포함해서 만듭니다. 즉, **누군가 중간 블록 하나를 수정하면**, 그 뒤의 모든 해시가 달라지기 때문에 **변조가 탐지**됩니다."

김영훈이 보충했다. "완벽한 위변조 방지는 아니야. DB 관리자가 체인 **전체**를 재작성하면 이론적으로 탐지가 안 돼. 하지만 실수나 부분 수정은 100% 잡아낸다."

"그래도 어차피 기록만 남기는 건데, 왜 굳이 해시 체인까지?" 지후가 물었다.

추박하의 목소리가 들렸다. 모두가 깜짝 놀라 뒤를 돌아보았다 — 추박하가 만화책을 덮고 말하고 있었다.

"검증 가능한 영수증이야."

침묵.

추박하가 일어서서 천천히 걸어왔다.

"너희가 식당에서 밥을 먹고, 카드 결제하면 영수증이 나오잖아. 그 영수증이 **너희가 밥을 먹었다는 증거**야. 영수증이 없으면 '밥을 먹었다'고 주장해도 증명할 수 없어."

그가 모니터를 가리켰다.

"OpsClaw의 PoW 블록은 **일의 영수증**이야. '이 에이전트가, 이 시간에, 이 명령을, 이 결과로 실행했다'는 영수증. 영수증이 있으면 '얼마나 일했는지'를 **양쪽이 검증**할 수 있어."

"양쪽이요?" 가람이 물었다.

"일을 시킨 쪽과 일을 한 쪽. 둘 다 같은 영수증을 보고 '이만큼 일했다'를 확인할 수 있어. 영수증을 조작하면 해시가 깨지니까 바로 들켜."

추박하가 다시 자리로 걸어가며 덧붙였다.

"그래서 이 영수증은 **절대 삭제하면 안 돼**. 삭제하면 증거가 사라지는 거야."

그는 만화책을 다시 집어들었다. 회의 끝인 것처럼.

---

민재가 헛기침을 하고 다시 진행했다.

"세 번째 핵심은 **보상(Reward)과 강화학습(RL)**입니다."

```
각 태스크 실행 시 보상이 자동 산출:
  성공(exit_code=0): base = +1.0
  실패: base = -1.0
  빠른 실행(<5초): speed_bonus = +0.3
  위험 태스크 실패: risk_penalty = -0.2

보상 범위: [-1.2, +1.3]
```

"이 보상이 쌓이면, Q-learning 알고리즘이 **최적의 실행 전략**을 자동으로 학습합니다."

```bash
# RL 추천 예시
curl -s "http://localhost:8000/rl/recommend?agent_id=http://localhost:8002&risk_level=low&exploration=ucb1" \
  -H "X-API-Key: opsclaw-api-key-2026"
```

화면에 나타난 결과:

```json
{
  "recommended_risk_level": "medium",
  "confidence": "trained",
  "exploration": "ucb1"
}
```

"UCB1이라는 탐색 전략이 '아직 시도하지 않은 행동'을 추천합니다. low만 계속 실행했으면, 이제 medium도 해보라고 제안하는 거죠."

---

회의가 끝난 후, 가람은 자리에 앉아 오늘 배운 것을 정리했다.

```
OpsClaw 핵심 3가지:
1. 3계층 위임: Master → Manager → SubAgent
2. 증적 + PoW: 모든 실행의 "검증 가능한 영수증"
3. 보상 + RL: 실행 데이터로 최적 전략 자동 학습
```

가람은 수첩을 넘기다가, 이전에 적어둔 메모를 보았다.

> `PoW 체인. 47,832 블록. 팀장님이 "절대 삭제하지 마라"고 한 것.`
> `genesis 프로젝트. 접근 거부.`
> `추팀장 새벽 출근. genesis에 dispatch. PoW +1.`

오늘 추팀장이 한 말이 머릿속에서 맴돌았다.

*"검증 가능한 영수증."*
*"일을 시킨 쪽과 일을 한 쪽. 둘 다 같은 영수증을 보고 확인할 수 있어."*
*"절대 삭제하면 안 돼."*

가람은 갑자기 소름이 돋았다.

*만약 OpsClaw의 진짜 목적이, 보안 작업의 기록이 아니라면?*

*만약 이 47,000개의 블록이, 누군가가 **다른 일을** 했다는 영수증이라면?*

*그리고 그 영수증을 "절대 삭제하면 안 된다"고 말한 사람이, 그 일을 시킨 사람이자 실행한 사람이라면?*

가람은 고개를 저었다. 아직 상상에 불과했다. 증거도 없고, 추팀장이 무엇을 했는지도 모른다.

하지만 한 가지는 확실했다.

**모든 것을 기록한다**는 말은, **모든 것이 기록되어 있다**는 뜻이기도 했다.

---

관제실의 불이 꺼지기 시작했다. 야간 교대 시간이 다가왔다.

```

Valid: true
Tampered: 0
```

47,847.
