"된다. **내 키보드 방수**야." 도윤이 말했다. "민재 옆자리에 앉은 지 6개월 만에 방수 키보드로 바꿨어."

"전체요?" 가람이 물었다. "넥서스AI 웹 서비스가 몇 개예요?"

"Nikto, Nmap, Nuclei, testssl." 가람이 세었다. "4가지 도구를 동시에 돌리는 거예요?"

"**종합 점검**이니까." 민재가 말했다. "각 도구의 강점이 달라."

가람이 수첩에 정리했다.

> `[웹 취약점 종합 점검 도구]`
>
> `1. Nikto — 웹 서버 스캐너`
> `   서버 설정 오류, 기본 파일, 알려진 취약점 탐지`
> `   빠르지만 노이즈가 많음`
>
> `2. Nmap + NSE — 네트워크/서비스 스캐너`
> `   포트 스캔, 서비스 버전 탐지, 취약점 스크립트`
> `   가장 범용적`
>
> `3. Nuclei — 템플릿 기반 취약점 스캐너`
> `   YAML 템플릿으로 최신 취약점 탐지`
> `   커뮤니티 템플릿 6,000+ 보유`
>
> `4. testssl.sh — TLS/SSL 점검`
> `   인증서, 프로토콜 버전, 암호 스위트 점검`
> `   BEAST, POODLE, Heartbleed 등 취약점 확인`
>
> `5. HTTP 헤더 점검`
> `   보안 헤더(CSP, HSTS, X-Frame-Options 등) 확인`

---

점검이 진행되는 동안 김영훈이 설명했다.

```
[종합 웹 취약점 점검 방법론]

1단계: 자산 식별 (Asset Discovery)
  - 웹 서비스 목록 확인
  - 도메인/서브도메인 열거
  - 포트/서비스 매핑

2단계: 자동 스캔 (Automated Scanning)
  - Nikto: 서버 설정 + 알려진 취약점
  - Nuclei: 최신 CVE + 설정 오류
  - Nmap: 서비스 버전 + 취약점 스크립트
  - testssl: TLS 보안

3단계: 수동 검증 (Manual Verification)
  - 자동 스캔 결과의 오탐(False Positive) 제거
  - 비즈니스 로직 취약점 점검 (자동화 불가)
  - 인증/인가 우회 테스트

4단계: 보고서 작성
  - 취약점별 위험도 분류 (CVSS 기반)
  - 재현 절차 (PoC) 포함
  - 조치 권고 + 우선순위

5단계: 재점검 (Re-test)
  - 조치 완료 후 재점검
  - 조치 확인서 발행
```

"5단계까지 가야 **완성된 점검**이에요." 김영훈이 말했다. "오늘은 2단계까지. 3단계는 다음 주에."

"Critical 3건." 김영훈이 말했다. "admin-console에 2건. 이건 **즉시 조치** 대상이야."

"어제 점검에서는 개선율 86%라고 했잖아요." 가람이 말했다. "근데 아직 Critical이 있네요?"

"보안은 **완료**가 없어." 김영훈이 말했다. "새로운 서비스가 추가되고, 새로운 취약점이 발견돼. 86%는 **과거 알려진 취약점** 기준이야. 오늘 발견한 건 **새로운 취약점**이야."

"끝이 없네요." 민재가 말했다.

"끝이 없는 게 **보안**이야." 추박하가 말했다.

정규가 Critical 취약점의 상세를 확인했다.

```
[Critical 취약점 상세]

CRIT-001: admin-console SQL Injection
  URL: /admin/api/users?filter=
  유형: Error-based SQL Injection
  영향: 관리자 DB 전체 접근 가능
  CVSS: 9.8
  조치: 파라미터 바인딩 적용 필요

CRIT-002: admin-console 인증 우회
  URL: /admin/api/config (GET)
  유형: Missing Authentication
  영향: 인증 없이 시스템 설정 열람 가능
  CVSS: 9.1
  조치: 인증 미들웨어 추가

CRIT-003: nexus-portal Remote Code Execution
  URL: /upload/process
  유형: File Upload → Code Execution
  영향: 서버 임의 명령 실행
  CVSS: 9.8
  조치: 파일 업로드 검증 강화, 실행 권한 제거
```

"이 세 건은 오늘 중으로 개발팀에 전달해야 해." 김영훈이 말했다.

```bash

    "command": "echo \"[CRITICAL] 3건의 Critical 취약점 발견. admin-console SQLi, 인증우회, nexus-portal RCE. 즉시 조치 필요.\" | tee /tmp/critical_alert.txt",
    "subagent_url": "http://localhost:8002"
  }'
```

"첫째, 현재 보안 상태의 **객관적 기록**. 둘째, 자진 신고 후 **개선 의지**의 증거."

"점검만 하면 되는 게 아니라, **조치까지** 해야 증거가 되는 거죠?" 가람이 물었다.
