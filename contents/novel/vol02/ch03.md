# Chapter 3: 흔들리는 벽

---

토요일 오전 10시. 비상 소집.

관제실에 전원이 출근했다. 주말인데도. 추박하마저 9시에 도착해 있었다 — 이것만으로 상황의 심각성을 알 수 있었다.

김영훈이 브리핑을 시작했다.

"새벽 공격 요약. 표면은 SQLi/XSS, 실제 목표는 Nexus-1 API. Apache 프록시가 인증 없이 노출. 현재 프록시 비활성화 완료."

"추가 발견 사항이 있어." 정규가 말했다. "새벽 이후 로그를 더 분석했는데, SQLi 공격 중에 **성공한 것**이 있어."

관제실이 조용해졌다.

---

정규가 화면을 열었다.

```bash
# web 서버 Suricata eve.json에서 HTTP 응답 코드 분석
sshpass -p1 ssh -o StrictHostKeyChecking=no secu@10.20.30.1 \
  "echo 1 | sudo -S cat /var/log/suricata/eve.json 2>/dev/null | \
   python3 -c \"
import sys, json
for line in sys.stdin:
    try:
        e = json.loads(line)
        if e.get('event_type') == 'http' and '/rest/user/login' in str(e.get('http',{}).get('url','')):
            status = e.get('http',{}).get('status', 0)
            if status == 200:
                src = e.get('src_ip','?')
                print(f'200 OK: {src} → {e[\"http\"][\"url\"]} at {e[\"timestamp\"]}')
    except: pass
\" 2>/dev/null | head -5"
```

```
200 OK: 91.132.x.x → /rest/user/login at 2026-04-04T02:51:23
200 OK: 185.220.101.x → /rest/user/login at 2026-04-04T02:52:47
```

"두 건의 **로그인 성공**." 정규의 목소리가 낮아졌다. "91.132와 185.220이 각각 한 번씩 JuiceShop 로그인을 성공했어."

"SQLi로?" 김영훈이 물었다.

"아마. `' OR 1=1--` 패턴이 바로 직전에 있어."

"JWT 토큰을 획득한 거네." 김영훈이 말했다. "그 토큰으로 뭘 했는지 확인해."

---

```bash
# JuiceShop API 호출 로그 확인 (JWT 인증된 요청)
sshpass -p1 ssh -o StrictHostKeyChecking=no secu@10.20.30.1 \
  "echo 1 | sudo -S cat /var/log/suricata/eve.json 2>/dev/null | \
   python3 -c \"
import sys, json
for line in sys.stdin:
    try:
        e = json.loads(line)
        if e.get('event_type') == 'http':
            h = e.get('http',{})
            if 'Bearer' in str(h.get('http_request_headers',[])):
                print(f'{e[\"src_ip\"]} → {h[\"http_method\"]} {h[\"url\"]} [{h.get(\"status\",\"?\")}]')
    except: pass
\" 2>/dev/null | head -10"
```

```

...

```

"Users 전체 열거. admin 설정 접근. FTP 디렉토리 열람. package.json 접근." 김영훈이 하나씩 읽었다.

"**데이터 유출이야.**" 지후가 말했다. "사용자 목록, 설정 파일, 백업 파일까지."

"하지만 이건 JuiceShop이야." 진호가 말했다. "JuiceShop은 **의도적으로 취약한 테스트 앱**이잖아. 거기서 뭘 빼가 봤자—"

"문제는 그게 아니야." 김영훈이 진호의 말을 끊었다. "문제는 이 공격 패턴이 **Nexus-1 API에도 동일하게 적용**되었을 가능성이야."

"프롬프트 인젝션이 성공했는지는?" 정규가 물었다.

"응답 본문을 봐야 해. 근데 로그에 응답 본문이 기록되어 있으면..."

"응답 본문 로그가 없어." 김영훈이 한숨을 쉬었다. "Nexus-1 서비스가 요청/응답 본문을 **기록하지 않고 있었어**."

"그럼 프롬프트 인젝션이 성공했는지 못 알아내는 건가요?" 도윤이 물었다.

"현재로서는." 김영훈이 고개를 끄덕였다. "공격자가 **시스템 프롬프트를 이미 빼갔을 수도** 있어. 모델 응답 패턴을 수집했을 수도 있어. **증거가 없어서 확인할 수 없다**는 게 문제야."

---

"이게 바로 **로깅의 중요성**이야." 추박하가 처음으로 입을 열었다.

모두가 돌아보았다.

"Nexus-1 서비스는 **API 요청은 기록하지만 응답은 기록하지 않았어**. 만약 OpsClaw를 통해 실행했다면, 모든 입출력이 evidence로 남았을 텐데."

김영훈이 이를 악물었다. "AI 연구소가 보안팀에 알리지 않고 배포했으니까요."

추박하가 고개를 끄덕였다. "맞아. 그리고 이건 **A.8.15 로깅 부적합**이야. 응답 본문 로그 없이 AI 모델 API를 외부에 노출한 건 심각한 컴플라이언스 위반이야."

가람은 수첩에 적었다.

> `Nexus-1 API: 17건의 completions 호출 성공 (인증 없이)`
> `응답 본문 로그 미존재 → 프롬프트 인젝션 성공 여부 확인 불가`
> `"증거가 없어서 확인할 수 없다" — 로깅의 부재가 가장 큰 취약점`
> `추팀장: "OpsClaw를 통해 실행했다면 evidence로 남았을 텐데"`

---

오후 2시. 서문기 상무에게 보고 전화.

김영훈이 회의실에서 30분간 통화했다. 나올 때 표정이 굳어 있었다.

"뭐래요?" 진호가 물었다.

"'조용히 처리하라.'" 김영훈이 말했다. "'외부에 알리면 Nexus-1 릴리즈에 차질이 생긴다. 내부에서 해결하라.'"

관제실이 술렁였다.

"외부에 알리지 말라고?" 지후가 말했다. "KISA(한국인터넷진흥원)에 침해사고 신고를 안 한다는 건가요?"

"정보통신망법상 **개인정보 유출이 확인되면** 72시간 내 신고 의무가 있어." 김영훈이 말했다. "하지만 아직 개인정보 유출이 **확인된 건 아니야**. JuiceShop의 Users 데이터는 테스트 데이터니까."

"Nexus-1 쪽은?" 정규가 물었다.

"응답 로그가 없어서 **유출 여부를 확인할 수 없어**." 김영훈의 입꼬리가 씰룩거렸다. "서문기 상무 입장에서는, '확인할 수 없으니 신고 의무도 없다'는 논리지."

"그건..." 도윤이 말을 흐렸다. "**증거를 안 만들면 사고도 안 난 셈**이라는 건가요?"

정적.

추박하가 만화책 뒤에서 말했다.

"그래서 **모든 것을 기록해야 하는** 거야."

---

*나는 왜 적고 있는 걸까. 그리고 추팀장은 왜 나에게 적으라고 한 걸까.*

*혹시... 추팀장도 나와 같은 이유로 기록하고 있는 건 아닐까?*

*누군가 나중에 **진실을 알아야 할 때**를 위해.*
