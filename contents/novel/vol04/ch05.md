# Chapter 5: 숫자의 언어

---

금요일 오전 8시 30분. 넥서스AI 재무전략팀 사무실.

백승현은 세 번째 에스프레소를 마시며 엑셀 시트를 노려보고 있었다. 스프레드시트는 42인치 모니터에서도 글씨가 개미 크기였다. 백승현은 안경 너머로 눈을 찡그리며 커서를 움직였다.

"승현아, 아침부터 뭘 그렇게 보냐?"

옆자리의 한주영 과장이 커피를 홀짝이며 물었다.

"Nexus-1 프로젝트 분기별 예산 집행 내역. 서문기 상무 쪽에서 요청한 결산이야."

"그래서?"

"숫자가 안 맞아."

한주영이 의자를 끌고 다가왔다. "어디가?"

백승현이 화면의 한 셀을 가리켰다.

```
[Nexus-1 프로젝트 예산 집행 — 2026 Q1]

항목                        예산(억)    집행(억)    차이
─────────────────────────────────────────────────
인건비 (연구원 32명)         18.0       17.8       -0.2
GPU 클러스터 (DGX)           45.0       44.2       -0.8
클라우드 (AWS/GCP)            8.0       12.4       +4.4  ← ⚠️
데이터 라이선스               3.0        3.0        0.0
외부 용역                     2.0        7.8       +5.8  ← ⚠️
기타 운영비                   4.0        3.9       -0.1
─────────────────────────────────────────────────
합계                         80.0       89.1       +9.1
```

"**클라우드가 4.4억 초과**, **외부 용역이 5.8억 초과**야." 백승현이 말했다. "합쳐서 **10.2억이 초과 집행**됐어."

"예산 초과야 흔한 거 아냐? AI 프로젝트는 원래 GPU 비용이—"

"GPU는 오히려 예산 이내야. **클라우드와 외부 용역**만 튀어." 백승현이 안경을 벗어 닦았다. "그리고 '외부 용역' 항목의 상세 내역이 **비어 있어**."

"뭐?"

"보통 외부 용역이면 계약서, 발주서, 검수서가 붙어야 하잖아. 근데 7.8억짜리 용역에 **증빙이 하나도 없어**."

한주영의 표정이 변했다. "그건... 좀 문제인데."

"문제 정도가 아니야." 백승현이 모니터를 바라보았다. "이건 **감사에 걸려**."

---

백승현은 넥서스AI에 5년차 재무 분석가였다. **최민재와 동기 입사**. 민재가 보안팀에서 Playbook을 만들 때, 승현은 재무팀에서 예산 시트를 만들었다. 둘 다 "시스템에서 진실을 찾는" 사람이었지만, 보는 시스템이 달랐을 뿐이다.

민재와는 분기에 한 번 정도 점심을 먹었다. 동기 입사는 회사에서 유일하게 편하게 말할 수 있는 관계니까.

"오늘 민재한테 연락해야겠다." 승현이 혼잣말을 했다.

"보안팀 민재?" 한주영이 물었다.

"응. 이 숫자... 보안팀이 알아야 할 것 같아서."

---

점심 시간. 넥서스AI 구내식당 2층 — 비교적 한산한 곳.

"야, 민재야."

최민재가 쟁반을 들고 다가왔다. 승현과 민재는 대학 시절처럼 반말을 했다. 회사에서 반말을 하는 관계는 둘뿐이었다.

"오랜만이다. 무슨 일이야? 밥 사줘?" 민재가 앉으며 물었다.

"내가 사줄게. 대신 좀 들어봐."

민재가 김치찌개를 한 숟갈 뜨다 멈추었다. 승현의 표정이 진지했다.

"왜, 무슨 일?"

승현이 주변을 둘러보고 목소리를 낮추었다.

"Nexus-1 프로젝트 예산에 이상한 게 있어."

민재의 젓가락이 멈추었다. 요즘 관제실에서 그 이름이 나오지 않는 날이 없었다. Nexus-1. 이주원. 서문기. 학습 데이터.

"이상한 게 뭔데?"

"외부 용역 7.8억. **증빙 없음**. 그리고 AWS 클라우드 비용이 예산 대비 55% 초과. 클라우드 비용 상세를 뜯어보니까—"

승현이 휴대폰에서 스크린샷을 보여주었다.

```
[AWS 비용 상세 — 2026 Q1]

리전            서비스          비용(만원)
─────────────────────────────────────
ap-northeast-2  EC2 + EBS      38,200    ← 정상 (학습용)
ap-northeast-2  S3             12,100    ← 정상 (데이터 저장)
us-east-1       S3              8,400    ← ⚠️ 미등록 리전
us-east-1       EC2             3,700    ← ⚠️ 미등록 리전
```

"**us-east-1이랑 eu-west-1**?" 민재가 눈을 가늘게 떴다. "우리 회사 AWS는 **ap-northeast-2(서울)만** 쓰잖아."

"맞아. 근데 미국 동부랑 유럽에서 **S3 비용**이 나오고 있어."

민재는 김치찌개를 내려놓았다.

"잠깐. 그 us-east-1 S3... 혹시 **외부에서 데이터를 가져오는** 용도 아니야?"

승현이 고개를 끄덕였다. "나도 그렇게 의심하고 있어. S3 교차 리전 전송 비용이 섞여 있어. **다른 AWS 계정에서 우리 계정으로** 데이터를 가져오는 패턴이야."

민재의 머릿속에서 퍼즐 조각이 맞춰졌다. *이다겸이 말한 supplemental_source 파이프라인. 외부 S3 버킷. 외부 AWS 계정.*

"승현아."

"응."

"그 외부 용역 7.8억의 **계약 상대방**은 확인할 수 있어?"

"계약서가 없으니까 상대방도 모르지. 근데..." 승현이 스마트폰을 넘겼다. "결제 이력은 있어."

```
[결제 이력 — 외부 용역]

일자          금액(만원)    수취인               비고
```

"**DataBridge LLC.** 같은 회사에 5회 결제. 전부 '데이터 수집 대행'이야."

"DataBridge LLC..." 민재가 되뇌었다. "들어본 적 없는 회사인데."

"나도. 미국 델라웨어 등록 LLC야. 실체가 있는 회사인지 **확인이 안 돼**."

민재가 밥을 먹는 것을 포기하고 생각에 잠겼다.

"승현아. 이거 혼자 알고 있으면 안 돼."

"알아. 그래서 너한테 말하는 거잖아."

"나한테 말하는 것만으로는 부족해. 이건 **보안팀과 감사** 범위야."

승현의 얼굴이 어두워졌다.

"민재야. Nexus-1 결제 승인자가 누군지 알아?"

"서문기 상무?"

"맞아. **전결 권한**이 서문기 상무에게 있어. 7.8억짜리 용역을 서문기가 직접 승인했어. 이걸 문제 삼으면 상무한테 찍혀."

"그건 알아. 하지만—"

"그리고 우리 CFO(재무이사)한테 보고하면? CFO는 서문기 상무 편이야. **Nexus-1이 성공하면 회사 주가가 올라가거든**. 예산 초과쯤은 눈감아줄 가능성이 높아."

민재가 한숨을 쉬었다. "**어디서 많이 들어본 패턴인데.** 정태식 본부장도 비슷한 말을 했어. '서비스 먼저'라고."

"서비스 먼저, 매출 먼저, 주가 먼저." 승현이 쓴웃음을 지었다. "회사에서 '먼저'가 아닌 게 뭐야."

"보안." 민재가 대답했다.

둘 다 잠시 침묵했다.

"계정 ID가 **471923847561**이야." 정규가 말했다.

민재가 수첩에 적었다. 그리고 승현에게 문자를 보냈다.

> "AWS 계정 471923847561 — DataBridge LLC의 AWS 계정인지 확인 가능해?"

3분 후 답장이 왔다.

> "AWS 비용 청구서에 교차 계정 전송 내역이 있어. 471923847561에서 우리 계정으로 전송한 기록 있음. 단, 이 계정이 DataBridge 소유인지는 AWS에 문의해야 알 수 있어."

민재는 고개를 끄덕이며 김영훈에게 갔다.

"과장님."

김영훈이 모니터에서 눈을 떼고 민재를 바라보았다. 민재의 표정이 평소와 달랐다.

"재무전략팀에서 정보를 입수했습니다."

민재가 승현에게 들은 내용을 보고했다. 외부 용역 7.8억, 증빙 없음, DataBridge LLC, 미등록 리전의 AWS 비용, 그리고 이다겸의 supplemental_source 파이프라인과의 연결 가능성.

김영훈이 팔짱을 끼고 들었다. 표정이 점점 단단해졌다.

"정리하면—"

김영훈이 화이트보드에 적었다.

```
[Nexus-1 자금 흐름 이상 — 증거 연결]
```

"맞아." 김영훈이 고개를 끄덕였다. "하지만 증빙 없는 7.8억짜리 용역은 그 자체로 **내부 감사 대상**이야."

추박하가 만화책 뒤에서 말했다.

"그 백승현이라는 친구. 재무팀에서 이 정보를 갖고 나온 거면, 위험하지 않아?"

민재의 얼굴이 어두워졌다. "... 위험할 수 있습니다."

"내부 감사에 정식으로 올라가기 전에, 이 정보가 서문기 귀에 들어가면 **승현이가 먼저 찍혀**."

"알고 있습니다."

추박하가 만화책을 덮었다. 오늘은 일찍 덮었다.

"그 친구한테 하나만 전해줘."

"**숫자는 거짓말을 안 해. 하지만 숫자를 발견한 사람은 보호받아야 해.** 정식 경로로 올릴 때까지, 원본 데이터의 사본을 안전한 곳에 보관하라고."

민재가 고개를 끄덕였다.

"그리고—" 추박하가 덧붙였다. "OpsClaw에 기록해둬."

"네."

민재는 승현에게 전화를 걸었다.

"승현아. 네가 발견한 것, 우리 팀장님이 심각하게 보고 계셔."

"그래?"

"네가 가진 예산 집행 내역, 결제 이력, AWS 비용 상세 — 원본 사본을 개인 드라이브에 보관해. 그리고 하나 더."

"뭔데?"

"네가 이 데이터를 발견한 과정을 **시간순으로 정리**해둬. 언제, 어떤 파일을 열었고, 뭘 확인했고, 누구에게 보고했는지."

"그건 왜?"

"**감사 증적**이야." 민재가 말했다. "나중에 '누가 언제 이 이상을 발견했는지' 증명해야 할 때가 올 거거든."

승현이 잠시 침묵했다.

"민재야."

"응."

"이거 단순한 예산 초과가 아니지?"

민재는 대답하지 않았다. 대답할 수 없었다.

"... 조심해라."

"너도."

---

가람은 관제실에서 민재와 정규의 대화, 김영훈의 화이트보드, 추박하의 지시를 모두 보았다. 수첩에 적었다.

> `[감사 증적(Audit Trail)이란]`
> `누가(WHO) — 언제(WHEN) — 무엇을(WHAT) — 왜(WHY) 했는지의 기록`
> `감사 증적이 없으면:`
> `  - "내가 이걸 발견했다"고 주장해도 증명 불가`
> `  - "이때 보고했다"고 해도 확인 불가`
> `  - 법적 분쟁에서 "사후 조작"으로 의심받을 수 있음`
> `감사 증적이 있으면:`
> `  - 발견 시점, 보고 시점, 대응 시점이 모두 증명됨`
> `  - 위변조 불가능하면 법정 증거로 사용 가능`
> `  → OpsClaw의 PoW 체인 = 위변조 불가 감사 증적`

그리고 이어서 적었다.

> `백승현 (재무전략팀, 민재 선배 동기)`
> `  - Nexus-1 예산 초과 10.2억 발견`
> `  - DataBridge LLC에 7.8억 결제 (증빙 없음)`
> `  - AWS 미등록 리전(us-east-1, eu-west-1) 비용 발생`
> `  - 교차 계정 전송: 471923847561 → 넥서스AI`
> `  → 이다겸의 supplemental_source 파이프라인과 연결됨`
> `  → "데이터의 흐름을 따라가면 돈의 흐름이 나오고,`
> `     돈의 흐름을 따라가면 진실이 나온다."`

### 컴플라이언스 교육

```
[SOC 2 Trust Service Criteria]

1. Security (보안)
   → 시스템을 무단 접근으로부터 보호
   → 방화벽, IDS, 접근제어, 암호화
   예: nftables 룰, SSH 키 인증, Suricata

2. Availability (가용성)
   → 약정된 서비스 수준(SLA) 유지
   → 백업, DR, 모니터링, 패치 관리
   예: 정태식 본부장의 "99.97% uptime"

3. Processing Integrity (처리 무결성)
   → 데이터 처리가 정확하고 완전
   → 입력 검증, 에러 처리, 감사 추적
   예: OpsClaw의 evidence 기록

4. Confidentiality (기밀성)
   → 기밀 정보의 보호
   → 암호화, 접근 통제, DLP
   예: Nexus-1 모델 가중치 접근 제한

5. Privacy (프라이버시)
   → 개인정보의 수집·사용·보관·폐기
   → 개인정보보호법, GDPR 준수
   예: 학습 데이터에 개인정보가 포함되어 있다면...
```
