# Chapter 3: 파이프라인의 균열

---

수요일 점심. 넥서스AI 구내식당.

윤정규는 혼자 밥을 먹고 있었다. 늘 그랬다. 정규는 점심 시간에 사람과 말하는 것을 좋아하지 않았다. 밥을 먹으면서도 머릿속에서는 로그가 흘러가고 있었다.

"정규야."

목소리에 고개를 들었다. 이다겸이 쟁반을 들고 서 있었다.

이다겸. 데이터 플랫폼팀. 대학 동기. 정규가 유일하게 식당에서 같이 밥을 먹는 사람이었다 — 다겸이 앉으면 둘 다 말없이 먹을 수 있었으니까. 서로 침묵이 편한 사이.

"앉아." 정규가 말했다.

다겸이 앞에 앉았다. 평소와 달리 표정이 어두웠다.

5분간 말없이 먹었다. 그리고 다겸이 먼저 입을 열었다.

"정규야, 내가 좀 이상한 걸 봤어."

---

다겸은 넥서스AI의 **데이터 파이프라인**을 관리하는 사람이었다. Kafka, Spark, 데이터 적재. AI 연구소가 학습 데이터를 수집하면, 다겸의 팀이 그것을 **정제하고, 변환하고, 저장**하는 역할을 했다.

"Nexus-1 학습 데이터 ETL(Extract-Transform-Load)을 내가 담당하잖아."

"응."

"원래 학습 데이터는 **정해진 소스**에서만 들어와야 해. 공개 데이터셋, 라이선스 계약 데이터, 자체 생성 데이터. 이 세 가지."

"그런데?"

"2월에 **네 번째 경로**가 생겼어."

정규가 젓가락을 멈추었다.

"네 번째?"

"'supplemental_source'라는 이름으로 새 데이터 파이프라인이 추가됐어. S3 버킷 → Kafka → Spark → 학습 데이터 저장소. 근데 이 **S3 버킷의 소유자가 AI 연구소 계정이 아니야**."

"누구 거야?"

"**외부 AWS 계정**이야. 계정 ID만 있고 누구 건지는 몰라. 하지만 데이터 양이 엄청나. 한 달에 2TB."

정규가 다겸을 바라보았다. "그걸 서문기 상무에게 보고했어?"

다겸이 고개를 저었다. "못 했어. supplemental_source 파이프라인을 추가한 건 **서문기 상무 직접 요청**이었거든. 인프라 변경 티켓에 상무 이름이 있어."

"그러면 상무가 **알고 있다**는 거잖아."

"응. 그래서 보고할 곳이 없어서 **너한테** 말하는 거야."

---

정규는 잠시 생각했다.

"다겸아. 그 S3 버킷의 데이터... 한 번이라도 내용을 봤어?"

"직접 열어본 적은 없어. 근데 ETL 로그에 파일 이름이 찍히잖아. 파일 이름이 이상했어."

```
파일명 예시:
  alphaai_research_2024_q3_translated.jsonl
  competitor_crawl_20240815_cleaned.jsonl
  scraped_arxiv_unpublished_202401.jsonl
```

"**alphaai_research**." 정규가 눈을 가늘게 떴다. "AlphaAI의 연구 데이터?"

"'translated'가 붙어 있어. 번역한 거야." 다겸이 말했다. "영어 데이터를 **한국어로 번역해서** 학습 데이터에 섞은 것 같아."

"competitor_crawl은?"

"경쟁사 웹사이트를 **크롤링**한 거겠지."

"scraped_arxiv_unpublished?"

"아직 **공개되지 않은** 학술 논문을 수집한 거. 이건... 논문 사전 리뷰어 권한을 악용한 거 아니면 **해킹**이야."

---

정규가 밥을 내려놓았다. 식욕이 사라졌다.

"다겸아. 이 얘기를 보안팀에 해야 해."

"나는 보안팀 사람이 아니야. 데이터 엔지니어야. 이걸 말하면 '왜 지금까지 몰랐냐'고 할 거잖아."

"지금까지 몰랐던 게 아니라 **의심만 있었던** 거잖아."

"의심만으로 상무를 고발할 수 없어."

"다겸아." 정규가 눈을 들었다. 평소에 볼 수 없는 진지한 눈.

"나한테 말한 건 이미 **걸음을 뗀** 거야. 나머지는 내가 할게."

다겸이 잠시 망설이다 고개를 끄덕였다.

---

오후 2시. 정규가 김영훈에게 보고했다. 1:1로.

"supplemental_source 파이프라인. 외부 S3. 파일명에 alphaai_research, competitor_crawl, scraped_arxiv_unpublished."

김영훈의 표정이 단단해졌다. "이다겸이 증언할 수 있어?"

"공식적으로는 어려울 거야. 하지만 **ETL 로그는 데이터 플랫폼 시스템에 남아 있어.**"

"로그를 확보할 수 있어?"

"다겸이 로그 접근 권한이 있어. 하지만 이 로그를 **외부로 가져오면** 다겸이 위험해져."

"OpsClaw로 기록하면?" 김영훈이 물었다. "다겸이 직접 로그를 확인하고, 그 **결과를** OpsClaw evidence에 기록하는 거야. 다겸이 아니라 **시스템이** 기록한 거가 되니까."

정규가 고개를 끄덕였다. "다겸한테 물어볼게."

---

저녁. 가람은 관제실에서 정규가 조용히 전화하는 것을 보았다.

정규가 전화를 끊고 가람에게 말했다.

"가람아."

"네, 선배."

"내가 기술적으로 처리할 게 있어. **조용히** 해줘."

"... 네."

정규는 OpsClaw에 새 프로젝트를 만들었다.

```bash
PID=$(curl -s -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"name":"data-provenance-audit","request_text":"학습 데이터 출처 감사","master_mode":"external"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['project']['id'])")
```

그리고 다겸에게 전달한 것은 — 이 프로젝트 ID와, Kafka ETL 로그에서 supplemental_source 관련 기록을 **OpsClaw dispatch로 실행**하는 방법이었다.

다겸이 자기 자리에서 실행한 명령:

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"cat /data/etl-logs/supplemental_source/*.log 2>/dev/null | grep -E \"alphaai|competitor|unpublished\" | wc -l","subagent_url":"http://localhost:8002"}'
```

결과:

```
2,847
```

2,847개의 파일이 supplemental_source 경로를 통해 학습 데이터에 유입되었다.

이 숫자는 OpsClaw의 evidence에 기록되었다. 영구적으로. 삭제 불가능하게.

---

가람은 수첩에 적었다.

> `이다겸 증언: supplemental_source 파이프라인 — 외부 S3 → Kafka → 학습 데이터`
> `파일명: alphaai_research, competitor_crawl, scraped_arxiv_unpublished`
> `2,847개 파일. 서문기 상무 직접 요청.`
> `OpsClaw evidence로 기록됨 — 삭제 불가.`
>
> `"정리된 게 아니라 묻은 것" — 추박하가 맞았다.`
> `학습 데이터에 경쟁사 데이터, 불법 크롤링, 미공개 논문이 섞여 있다.`
> `이제 training_data_manifest.xlsx를 열지 않아도, ETL 로그가 증거다.`
