### ATT&CK 매핑

김영훈이 공격을 MITRE ATT&CK에 매핑했다.

```
[실제 공격 ATT&CK 매핑]

Initial Access:
  → 아직 불명 (어떻게 web 서버에 비콘이 설치되었는가?)

Execution:
  T1059.001 — PowerShell 또는 bash를 통한 비콘 실행

Command and Control:
  T1071.001 — HTTPS를 통한 C2 통신
  T1573.002 — Asymmetric Cryptography (암호화된 C2)
  T1571     — Non-Standard Port (443이지만 비표준 프로토콜)

Discovery:
  → 아직 관찰 안 됨 (비콘이 막 설치된 상태)
```

"**어떻게 설치됐는지**가 핵심이야." 김영훈이 말했다.

지후가 web 서버의 프로세스를 확인했다.

curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \

### 대응

"C2 비콘 프로세스를 **즉시 종료**하고, 웹셸을 삭제하고, C2 IP를 **차단**한다."

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/execute-plan" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "tasks": [
      {"order":1,"title":"C2 비콘 프로세스 종료","instruction_prompt":"echo 1 | sudo -S kill -9 12847 2>/dev/null && echo 1 | sudo -S rm -f /tmp/.cache/update && echo \"Beacon killed + payload removed\"","risk_level":"critical","subagent_url":"http://10.20.30.80:8002","confirmed":true},
      {"order":2,"title":"웹셸 삭제","instruction_prompt":"echo 1 | sudo -S rm -f /opt/juice-shop/uploads/.hidden_update.sh && echo \"Webshell removed\"","risk_level":"high","subagent_url":"http://10.20.30.80:8002"},
      {"order":3,"title":"C2 IP 차단","instruction_prompt":"echo 1 | sudo -S nft add rule inet filter input ip saddr 45.33.32.156 drop && echo 1 | sudo -S nft add rule inet filter output ip daddr 45.33.32.156 drop && echo \"45.33.32.156 BLOCKED (in+out)\"","risk_level":"high","subagent_url":"http://10.20.30.1:8002"},
      {"order":4,"title":"Suricata 룰 추가","instruction_prompt":"echo 1 | sudo -S bash -c 'echo \"drop ip 45.33.32.156 any -> any any (msg:\\\"BLOCK C2 45.33.32.156\\\"; sid:9000010; rev:1;)\" >> /etc/suricata/rules/local.rules' && echo 1 | sudo -S systemctl reload suricata && echo \"Suricata C2 block added\"","risk_level":"high","subagent_url":"http://10.20.30.1:8002"}
    ],
    "subagent_url":"http://localhost:8002",
    "parallel": false
  }'
```

"**순차 실행.** 비콘 종료 → 웹셸 삭제 → IP 차단 → Suricata 룰."

결과:

```
Beacon killed + payload removed ✅
Webshell removed ✅
45.33.32.156 BLOCKED (in+out) ✅
Suricata C2 block added ✅
```

"**대응 완료.** 4단계, 3분." 김영훈이 말했다.

"3권 때는 C2 비콘을 **10일간** 발견 못 했어요." 가람이 — 전화로 참여하고 있었다, 새벽 4시에 — 말했다.

"그때와 지금은 다르지." 김영훈이 말했다. "Suricata 룰이 강화됐고, OpsClaw LLM 분석이 돌아가고 있으니까."

> `[로그 분석 + ATT&CK 매핑 — 교육 정리]`
>
> `Red Team vs 실제 APT 구분법:`
> `  1. 출발지 IP: Red Team IP 목록과 대조`
> `  2. 활동 시간: Red Team은 업무 시간, APT는 타겟 야간`
> `  3. 도구: Red Team은 사전 합의된 도구, APT는 Cobalt Strike 등`
> `  4. 한승우에게 직접 확인`
>
> `Cobalt Strike C2 비콘 특징:`
> `  - 주기적 콜백 (sleep interval, 보통 5~60분)`
> `  - HTTPS/DNS over HTTPS로 암호화`
> `  - 소량 데이터 전송 → 탐지 어려움`
> `  - 시간이 지나면 데이터 전송량 증가 (탈취 단계)`
>
> `ATT&CK 매핑의 가치:`
> `  공격의 각 단계를 표준 프레임워크에 매핑`
> `  → "이 공격은 어느 단계에 있는가?"를 객관적으로 판단`
> `  → 다음 단계를 예측하여 선제 방어 가능`
>
> `[핵심 교훈]`
> `"모의해킹 중 진짜 공격이 올 수 있다."`
> `"축제 때 도둑이 들기 쉬운 것처럼, 노이즈 속에 진짜가 숨는다."`
> `"Red Team IP 목록이 있으면, 진짜와 가짜를 구분할 수 있다."`
