# 1. 서론 (Introduction)

> **Abstract**: 사이버보안 교육은 이론, 실습, 평가가 분절되어 학습 경험의 연속성이 보장되지 않는 구조적 문제를 안고 있다. 본 논문은 IT 운영·보안 자동화 플랫폼 OpsClaw를 오케스트레이터로 활용하는 4계층 통합 교육 파이프라인(System→Education→Novel→CTF)을 제안한다. 120개 구조화 교안, 120장 기술 소설, CTFd 기반 평가가 실환경 인프라 위에서 OpsClaw에 의해 유기적으로 연결되며, 모든 학습 활동이 불변 증적(evidence + PoW 해시 체인)으로 자동 기록된다. 3개 주제(초급·중급·고급) × 4계층 = 12태스크 실험에서 100% 태스크 성공률, 100% 증적 완전성, PoW 무결성 검증 통과, 약 3초 내 병렬 완료를 달성하였다. 본 연구는 IT 운영 플랫폼이 교육 오케스트레이터로 재활용될 수 있음을 실증하며, 운영 증적이 학습 증적으로, 실제 보안 도구가 교육 도구로 기능하는 새로운 보안 교육 패러다임을 제시한다.

> **Keywords**: 보안 교육, CTF, 에이전트 오케스트레이션, 교육 파이프라인, Proof of Work, 불변 증적, 기술 소설, 적응형 학습

## 1.1 연구 배경

사이버보안 위협이 고도화됨에 따라 숙련된 보안 인력에 대한 수요가 급증하고 있다. (ISC)² 2024 보고서에 따르면 글로벌 사이버보안 인력 부족은 약 400만 명에 달하며 [1], 한국의 경우에도 2025년 기준 약 3만 명의 보안 전문 인력 부족이 추산된다. 이러한 인력난의 근본 원인 중 하나는 **보안 교육의 구조적 분절(fragmentation)**이다.

현행 보안 교육은 크게 세 가지 접근으로 나뉜다. **강의 중심 교육(lecture-based)**은 이론적 체계를 제공하지만, 실습과의 연계가 부족하여 학습자가 실제 공격·방어 시나리오를 경험하지 못한다. **CTF(Capture The Flag) 중심 교육**은 실습적 몰입감을 제공하지만, 체계적 이론 학습 없이 단편적 기술 습득에 그치는 경우가 많다 [2]. **실습실(lab) 중심 교육**은 실환경을 제공하지만, 학습 동기 부여를 위한 맥락(context)이 부재하고 평가가 수동적이다.

이 세 접근의 공통적 한계는 **분절된 학습 경험**이다. 이론을 배우는 시점과 실습하는 시점이 분리되어 있고, 학습 활동의 결과가 자동으로 기록·검증되지 않으며, 학습자의 동기를 지속시킬 서사적 맥락(narrative context)이 부재하다. 결과적으로 학습자는 "왜 이것을 배우는가"에 대한 답을 스스로 구성해야 하며, 교수자는 학습 성취를 수동으로 평가해야 한다.

이 분절은 단순한 불편함을 넘어 구조적 비효율을 초래한다. Svabensky et al. [16]의 체계적 문헌 조사에 따르면, 보안 교육 관련 논문의 대다수는 단일 교육 방법(강의, CTF, 또는 Lab)에 초점을 맞추며, 이들을 유기적으로 통합하는 프레임워크에 대한 연구는 극히 드물다. Vykopal et al. [31]은 대학 과목에 CTF를 통합할 때의 이점과 함정을 분석하며, CTF가 체계적 커리큘럼 없이 독립적으로 운영될 때 학습 효과가 제한됨을 보고하였다.

더 나아가, 보안 교육에서 **학습 증적(learning evidence)**의 관리는 거의 논의되지 않는 영역이다. 전통적 교육에서 학습자의 활동은 출석 기록, 과제 제출, 시험 성적으로만 추적되며, 학습자가 실습 환경에서 어떤 명령을 실행하고, 어떤 결과를 얻었으며, 어떤 순서로 문제를 풀었는지에 대한 세부 기록은 체계적으로 관리되지 않는다. 이는 학업 무결성(academic integrity) 검증, 개인화된 피드백, 학습 분석(learning analytics)의 근본적 제약이 된다.

본 논문은 이러한 배경에서, 이론·실습·서사·평가를 단일 파이프라인으로 통합하고, 모든 학습 활동을 불변 증적으로 자동 기록하는 플랫폼의 필요성을 제기한다.

## 1.2 기존 연구의 한계

### 1.2.1 CTF 플랫폼의 한계

CTFd [3], PicoCTF [4], HackTheBox [5] 등 CTF 플랫폼은 보안 교육에서 널리 활용되고 있다. 이들 플랫폼은 게임화(gamification)를 통해 학습 몰입도를 높이지만, 다음과 같은 한계를 가진다:

- **이론적 맥락 부재**: 챌린지가 개별 기술에 초점을 맞추며, 해당 기술이 전체 보안 체계에서 어떤 위치를 차지하는지에 대한 맥락이 없다.
- **증적 추적 불가**: 학습자가 flag를 제출하면 "통과/실패"만 기록되며, 풀이 과정(evidence)이 자동으로 수집·보존되지 않는다.
- **서사적 동기 부재**: 챌린지 간의 연결 서사가 없어, 학습자의 장기적 몰입이 어렵다.
- **실환경 격리**: 대부분 가상화된 샌드박스에서 실행되어, 실제 운영 환경의 복잡성을 반영하지 못한다.

### 1.2.2 강의 중심 교육의 한계

NICE Cybersecurity Workforce Framework [6]과 CyBOK(Cyber Security Body of Knowledge) [7]은 체계적인 교육 프레임워크를 제공하지만, 이를 기반으로 한 강의식 교육은 다음과 같은 한계를 노출한다:

- **실습 연계 부족**: 이론 강의 후 별도의 실습 세션이 필요하며, 두 활동이 시간적·공간적으로 분리된다. 학습자가 강의에서 "nftables의 input 체인"을 배운 후 실습실에서 이를 직접 설정하기까지 수일이 소요되는 경우가 일반적이다.
- **자동 평가 불가**: 학습 성취 평가가 시험이나 보고서에 의존하며, 실습 활동의 자동 채점이 이루어지지 않는다. 교수자는 학습자의 실습 결과를 일일이 확인해야 하며, 이는 대규모 수업에서 병목이 된다.
- **학습 동기**: 추상적 이론의 나열은 학습자의 내재적 동기를 자극하기 어렵다. "왜 방화벽 룰을 이렇게 작성해야 하는가"에 대한 실제적 맥락이 부재하면, 학습자는 명령어를 기계적으로 암기하는 데 그치게 된다.
- **재현성 부족**: 학습자의 실습 과정이 체계적으로 기록되지 않아, 교수자가 학습자의 실수를 사후에 재현하고 피드백하기 어렵다.

### 1.2.3 AI 에이전트와 교육

ChatGPT, Claude Code 등 LLM 기반 에이전트의 교육 활용이 증가하고 있으나 [8][9], 이들은 "대화형 튜터(conversational tutor)"에 그치는 경우가 많다. 에이전트가 실제 인프라에서 명령을 실행하고 그 결과를 구조적으로 기록하는 **실행 기반 교육 오케스트레이션(execution-based educational orchestration)**은 아직 보고된 바 없다.

대화형 튜터의 한계는 명확하다. 학습자가 "nftables에서 SSH 포트를 허용하는 방법"을 질문하면 에이전트는 정확한 명령어를 제시할 수 있지만, 그 명령이 실제로 실행되었는지, 실행 결과가 예상과 일치하는지, 다른 룰과의 충돌은 없는지를 확인하지 않는다. **실행 없는 교육은 시뮬레이션 없는 비행 훈련과 같다.** 본 논문은 LLM 에이전트의 역할을 대화형 튜터에서 **실행 기반 오케스트레이터**로 확장할 것을 제안한다.

### 1.2.4 실습 환경의 격리 문제

기존 보안 교육 플랫폼(TryHackMe [12], SEED Labs [14])은 학습자에게 격리된 가상 머신(VM) 환경을 제공한다. 이러한 격리는 안전성을 보장하지만, 다음과 같은 교육적 한계를 초래한다:

- **단일 서비스 환경**: 대부분의 실습이 단일 VM에서 이루어지며, 실제 운영 환경의 다중 서버(방화벽, IPS, WAF, SIEM) 연동을 경험할 수 없다.
- **운영 현실과의 괴리**: 격리 환경에서의 성공이 실제 환경에서의 역량으로 직결되지 않는다. 실제 환경에서는 방화벽 룰 변경이 IPS 탐지에 영향을 주고, WAF 설정이 SIEM 경보 패턴을 변화시키는 연쇄 효과가 존재한다.
- **일회성**: 실습 환경이 세션 종료 후 초기화되어, 장기적인 학습 이력 관리가 불가능하다.

## 1.3 제안: 4계층 통합 교육 파이프라인

본 논문은 이러한 분절 문제를 해결하기 위해 **OpsClaw 기반 4계층 통합 보안 교육 파이프라인**을 제안한다. 4계층은 다음과 같다:

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 4: CTF          — CTFd 기반 실전 평가                 │
│  Layer 3: Novel        — 120장 기술 소설 (서사적 동기)        │
│  Layer 2: Education    — 120개 구조화 교안 (체계적 이론)       │
│  Layer 1: System       — 실환경 인프라 + SubAgent 실행        │
│  ──────── OpsClaw Orchestrator ────────                     │
│  (evidence → PoW → reward → RL)                             │
└─────────────────────────────────────────────────────────────┘
```

**핵심 주장(Key Claim)**: 단일 오케스트레이션 플랫폼(OpsClaw)이 이론(Education), 실습(System), 서사적 맥락(Novel), 실전 평가(CTF)를 통합하고, 모든 학습 활동을 불변 증적(immutable evidence)으로 자동 기록함으로써, 보안 교육의 분절 문제를 구조적으로 해결할 수 있다.

이 파이프라인에서 OpsClaw는 교육 콘텐츠를 전달하는 LMS(Learning Management System)가 아니라, **학습 활동의 실행과 증적을 관리하는 오케스트레이터**로 기능한다. 학습자가 수행한 모든 실습 명령은 SubAgent를 통해 실행되고, 그 결과는 evidence로 기록되며, SHA-256 해시 체인(PoW)으로 무결성이 보장된다. 이는 학업 무결성(academic integrity) 검증과 자동 평가를 동시에 가능하게 한다.

4계층의 핵심 설계 원칙은 **동일 플랫폼(single platform)**이다. 실제 보안 운영에 사용되는 OpsClaw가 교육 오케스트레이터를 겸함으로써, "교육 환경"과 "운영 환경"의 경계가 사라진다. 학습자는 실제 보안 도구(nftables, Suricata, Wazuh, BunkerWeb)가 구동되는 인프라에서 실습하며, 그 결과가 운영 수준의 증적으로 기록된다. 이는 "교실에서 배운 것이 현장에서도 동일하게 작동한다"는 교육적 일관성을 보장한다.

또한, 소설(Novel) 계층의 도입은 보안 교육에서 시도되지 않은 혁신적 접근이다. 120장의 사이버보안 스릴러 소설이 120개 교안과 1:1로 매핑되어, 학습자는 등장인물의 시점에서 보안 사고를 경험하고, 해당 사고에서 사용된 기술을 교안에서 학습한 뒤, CTF에서 직접 실행한다. 이러한 서사적 맥락은 학습 동기를 지속시키는 핵심 장치이다.

## 1.4 주요 기여

본 논문의 주요 기여는 다음과 같다:

본 논문의 주요 기여는 다음 네 가지로 요약된다. 각 기여는 5장의 실험을 통해 검증된다.

- **C1. 4계층 통합 교육 파이프라인 아키텍처 제안.** System(실행)→Education(이론)→Novel(서사)→CTF(평가)를 단일 오케스트레이터로 연결하는 파이프라인 아키텍처를 설계하고 구현하였다. 120개 교안, 120장 소설, CTFd 챌린지가 1:1로 매핑되어 교육 콘텐츠의 완전한 정합성(coherence)을 보장한다. (→ RQ1 검증)

- **C2. 불변 증적 기반 자동 평가 메커니즘.** OpsClaw의 PoW(Proof of Work) 체인을 활용하여 학습 활동의 모든 실행 결과를 자동 기록·검증하는 메커니즘을 구현하였다. 12개 실험 태스크에서 100%의 증적 완전성(evidence completeness)을 달성하였다. (→ RQ2, RQ3 검증)

- **C3. 3주제 × 4계층 교차 실험을 통한 파이프라인 검증.** 초급(nftables 방화벽), 중급(Wazuh 경보 분석), 고급(LLM 프롬프트 인젝션 방어) 3개 주제에 대해 4계층 파이프라인의 완전한 작동을 실증하였으며, 12/12 태스크 성공, 12건 증적 자동 생성, 12개 PoW 블록 체인, 약 3초 내 병렬 실행을 달성하였다. (→ RQ1~RQ4 종합 검증)

- **C4. 소설(내러티브)의 교육적 효용성 논증.** 기술 소설이 보안 교육에서 학습 동기 부여와 맥락 제공에 기여할 수 있음을 아키텍처적으로 논증하고, 120장의 소설이 120개 교안과 1:1로 매핑되는 구조를 제시하였다.

이상의 기여는 기존의 보안 교육 연구가 단일 교육 방법(CTF, 강의, Lab)에 초점을 맞추는 것과 대비되며, 다수의 교육 방법을 단일 파이프라인으로 통합하고 그 동작을 실험적으로 검증한 최초의 사례이다.

## 1.5 연구 질문

본 논문은 다음의 연구 질문(Research Question)에 답한다:

- **RQ1. 파이프라인 완전성**: 4계층(System, Education, Novel, CTF)의 태스크가 단일 오케스트레이션 호출로 모두 성공적으로 실행될 수 있는가?
- **RQ2. 증적 자동 생성**: 모든 태스크에 대해 evidence와 PoW 블록이 자동으로 생성되는가?
- **RQ3. 무결성 검증**: 생성된 PoW 체인이 무결성 검증(verify)을 통과하는가?
- **RQ4. 병렬 실행**: 다수의 태스크가 병렬로 실행되어 교육 세션의 지연을 최소화할 수 있는가?

## 1.6 연구 범위

본 논문은 파이프라인의 **기술적 아키텍처와 동작 검증**에 초점을 맞춘다. 실제 학습자 코호트를 대상으로 한 교육 효과 실증(학습 성취도 변화, 동기 측정 등)은 향후 연구로 남겨둔다. 본 실험은 3개 주제(초급·중급·고급) × 4계층 = 12개 태스크를 단일 OpsClaw `execute-plan` 호출로 실행하고, 증적 완전성과 PoW 무결성을 검증하는 것을 목표로 한다.

구체적 범위는 다음과 같다:

- **포함**: 파이프라인 아키텍처 설계, 4계층 구현, 12태스크 실행 실험, 증적/PoW 검증, 전통 접근법과의 정성적 비교
- **제외**: 학습자 대상 교육 효과 실험, 소설의 학습 효과 실증, 다기관 배포, 적응형 CTF 생성

실험 인프라는 4대의 서버(opsclaw, v-secu, v-web, v-siem)로 구성되며, 각 서버에 실제 보안 도구(nftables, Suricata IPS, BunkerWeb WAF, Wazuh SIEM)가 구동된다. 이는 격리된 가상 환경이 아닌 실환경(production-grade) 인프라이며, 동일한 인프라가 교육과 운영에 공유된다.

본 실험은 파이프라인의 **필요조건(necessary condition)** — 기술적으로 동작하는가 — 을 검증한다. **충분조건(sufficient condition)** — 교육적으로 효과적인가 — 의 검증은 향후 학습자 대상 연구로 남겨둔다.

실험 결과의 핵심 수치를 미리 요약하면 다음과 같다:

| 메트릭 | 값 |
|--------|-----|
| 총 태스크 수 | 12 (3주제 × 4계층) |
| 태스크 성공률 | 12/12 = 100% |
| 자동 생성 evidence | 12건 |
| 자동 생성 PoW 블록 | 12개 |
| PoW 체인 무결성 | valid=true, orphans=0, tampered=[] |
| 병렬 실행 시간 | ~3초 |
| 태스크당 평균 시간 | ~250ms |

## 1.7 용어 정의

본 논문에서 사용하는 핵심 용어를 정의한다:

| 용어 | 정의 |
|------|------|
| 파이프라인(Pipeline) | System→Education→Novel→CTF의 4계층이 OpsClaw로 연결된 통합 교육 체계 |
| 증적(Evidence) | SubAgent가 명령을 실행한 결과 (command, stdout, stderr, exit_code, duration_ms) |
| PoW 블록 | Evidence의 SHA-256 해시와 nonce를 포함하는 불변 기록 단위 |
| 교안(Lecture) | 8과목 × 15주 = 120개의 구조화된 교육 콘텐츠 |
| 매핑(Mapping) | 교안, 소설 장, CTF 챌린지 간의 1:1 대응 관계 |
| 오케스트레이터(Orchestrator) | 4계층의 태스크 실행, 증적 기록, 검증을 관리하는 중앙 플랫폼 (OpsClaw) |
| SubAgent | 대상 서버에서 명령을 실행하는 경량 런타임 |

## 1.8 논문 구성

본 논문의 이후 구성은 다음과 같다:

- **2장 (배경 연구)**: 보안 교육의 도전 과제, CTF 플랫폼, LLM 에이전트의 교육 활용, 블록체인 기반 교육 증적, OpsClaw 아키텍처 개요를 분석한다.
- **3장 (파이프라인 아키텍처)**: 4계층 파이프라인의 설계 원칙, 각 계층의 구조, OpsClaw 오케스트레이터의 역할, 확장성 설계를 기술한다.
- **4장 (구현)**: 서버 인프라, 8과목 커리큘럼, 소설 통합, CTF 연동, OpsClaw API 활용의 구현 세부사항을 설명한다.
- **5장 (실험)**: 3주제 × 4계층 = 12태스크 교차 실험의 설계, 실행, 결과를 제시하고, 전통 접근법과 비교한다.
- **6장 (논의)**: 발견사항 해석, 소설의 교육적 가치, PoW 기반 학업 무결성, RL 적응형 난이도, 한계점과 타당성 위협을 논의한다.
- **7장 (결론)**: 기여 정리, 핵심 발견, 향후 연구 방향을 제시한다.
