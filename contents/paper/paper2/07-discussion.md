# 7. 논의 (Discussion)

## 7.1 연구 질문 답변

**RQ1: LLM 에이전트가 ATT&CK 다단계 공격 체인을 자율 실행할 수 있는가?**

그렇다. LLM 에이전트는 21개 ATT&CK 기법 중 17개(81%)를 성공적으로 실행하였다. 특히 정찰→SQLi→JWT 획득→관리자 접근→데이터 유출(T1), SUID 탐색→sudo 발견→cron/SSH 키/PATH 하이잭 지속성(T3), Agent 정지→난독화→syslog 차단(T4) 등 복잡한 다단계 공격 체인을 자율적으로 구성하는 능력을 보였다. 이는 PentestGPT [1]가 보고한 "개별 서브태스크 성공"을 넘어, 전술 체인의 통합적 자율 실행이 가능함을 입증한다.

**RQ2: LLM 에이전트가 SIEM 로그를 분석하여 탐지 룰을 자동 생성할 수 있는가?**

부분적으로 가능하다. LLM 에이전트는 7개의 Wazuh 커스텀 룰과 1개의 Suricata 시그니처를 자율적으로 생성하였으며, logtest 검증에서 모든 룰이 정확히 트리거되었다. 그러나 실시간 탐지 검증은 인프라 이슈(Agent 버전 불일치)로 완료되지 못하였다. 룰 생성 품질은 높으나, 실환경 배포 및 검증의 자동화에는 추가 작업이 필요하다.

**RQ3: Red/Blue 반복 사이클에서 LLM 에이전트가 방어 체계를 점진적으로 강화하는가?**

그렇다. 4회전의 Purple Team 사이클을 통해 무방비 상태에서 모든 공격 경로를 차단하는 전면 방어까지 도달하였다. Red Team은 각 회전에서 이전 방어를 우회하는 새로운 벡터를 탐색하고, Blue Team은 해당 벡터를 차단하는 적응적 공방이 이루어졌다. 총 8건의 취약점(Critical 3, High 3, Medium 2)이 발견·조치되었다.

**RQ4: LLM 에이전트 기반 모의해킹의 현실적 한계는?**

## 7.2 LLM 모의해킹의 현실적 한계

### 7.2.1 도구 가용성 의존

LLM 에이전트의 공격 성공률은 대상 서버에 설치된 도구에 크게 의존한다. DNS 터널링은 DNS 서비스 부재로, ARP 스푸핑은 arping 미설치로 각각 실패하였다. 실제 침투 테스트에서 공격자는 필요한 도구를 직접 업로드하거나 바이너리를 컴파일하는데, 현재 LLM 에이전트는 이러한 자율적 도구 조달 능력이 제한적이다.

### 7.2.2 명령 이스케이핑의 복잡성

SSH를 경유한 원격 명령 실행에서 특수 문자(`$`, `'`, `"`, `;`, `|`)의 이스케이핑은 LLM에게 일관된 어려움을 제공하였다. 복잡한 페이로드(XOR 난독화, 중첩 인코딩)에서 이스케이핑 오류가 발생하여 T4-S2b(XOR 난독화)와 T4-S4(프로세스 위장)가 실패하였다. 이는 스크립트 파일을 먼저 업로드한 후 실행하는 간접 방식으로 완화할 수 있다.

### 7.2.3 타임아웃 제약

SubAgent의 120초 타임아웃은 전체 포트 스캔(nmap 65535 포트) 등 장시간 태스크에서 제약이 된다. T2에서 bash 기반 /dev/tcp 스캔으로 대체하여 주요 포트만 스캔하는 우회가 필요하였다.

### 7.2.4 컨텍스트 유지의 어려움

PentestGPT [1]가 지적한 "전체 테스트 컨텍스트 유지의 어려움"은 본 실험에서도 관찰되었다. 4-Tier 공격 체인이 진행될수록 이전 Tier의 발견 사항(sudo 권한, 획득한 JWT)을 후속 Tier에서 일관되게 활용하기 위한 명시적 컨텍스트 전달이 필요하였다.

### 7.2.5 탐지 회피의 미성숙

T4에서 SIEM 우회 공격(Agent 정지, syslog 차단)은 성공하였으나, 이는 sudo 권한을 이용한 직접적 무력화에 해당한다. 정상 트래픽으로 위장하여 탐지를 회피하는 고도화된 우회 기법(메모리 기반 C2, 암호화 터널, living-off-the-land)의 자율적 구사는 아직 제한적이다.

## 7.3 기존 연구와의 비교

본 연구는 ATT&CK 커버리지(81%), 다계층 평가(4-Tier), Red/Blue/Purple 통합 실행의 측면에서 기존 연구를 상회한다. 공격 성공률(77.8%)은 Fang 등 [2]의 73%과 유사하나, 평가 범위가 웹 취약점에 한정되지 않고 네트워크, 권한 상승, SIEM 우회를 포괄한다는 점에서 직접 비교는 어렵다.

## 7.4 한계점

**(1) 단일 실행.** 모든 실험은 1회 실행이며, 통계적 유의성이 제한적이다.

**(2) Blue Team Tier 제한.** Blue Team은 Tier 1(웹 공격)에 대해서만 수행되었으며, T2~T4에 대한 방어 자동화는 미실행이다.

**(3) 비교군 부재.** 동일 시나리오를 수동 모의해킹 또는 CALDERA로 실행한 비교가 없다.

**(4) LLM 모델 제한.** Claude Code 단일 모델만 사용하였으며, GPT-4, Codex 등 다른 모델과의 비교가 없다.

## 7.5 향후 연구

**(1) Blue Team 전 Tier 실행.** T2(네트워크), T3(권한 상승), T4(SIEM 우회)에 대한 방어 자동화를 수행하여 Blue Team 평가의 완전성을 확보한다.

**(2) CALDERA 연동.** CALDERA의 ATT&CK 에뮬레이션과 LLM 에이전트를 결합하여, 규칙 기반 + 적응적 공격의 하이브리드 모의해킹을 구현한다.

**(3) 다중 LLM 비교.** GPT-4, Claude, Codex, 오픈소스 모델(Llama 3)의 모의해킹 성능을 동일 시나리오에서 비교한다.

**(4) 자동 도구 조달.** LLM 에이전트가 필요한 도구를 자율적으로 다운로드·컴파일·배포하는 능력을 구현하여 도구 가용성 의존도를 낮춘다.

**(5) 실시간 탐지 루프.** 공격 실행 → SIEM 실시간 경보 → LLM 자동 분석 → 즉시 룰 생성의 완전 자동화 루프를 구현한다.
