# 7. 결론 (Conclusion)

본 논문은 LLM 에이전트 하네스의 두 패러다임 — 클라이언트 하네스(Claude Code)와 서버 하네스(OpsClaw) — 을 10개 차원에서 체계적으로 비교하고, 실험을 통해 각 패러다임의 강점과 한계를 정량적으로 평가하였다.

**핵심 발견:**

1. **속도 vs 증적의 트레이드오프.** 클라이언트 하네스가 27% 더 빠르지만, 서버 하네스는 실행당 10건의 자동 증적(evidence + PoW)과 보상 추적을 제공한다. 순수 실행 속도가 중요한 환경에서는 클라이언트 하네스가, 감사·추적·학습이 필요한 환경에서는 서버 하네스가 유리하다.

2. **유연성 vs 결정론의 트레이드오프.** Claude Code Skill은 LLM의 적응적 판단으로 새로운 상황에 대응하지만 비결정론적이다. OpsClaw Playbook은 파라미터 바인딩으로 100% 재현을 보장하지만 사전 정의된 절차만 수행한다.

3. **개인 vs 조직의 트레이드오프.** Claude Code 메모리는 개인 개발자에 최적화(머신 로컬, auto-memory)되어 있고, OpsClaw 메모리는 팀/조직 단위 지식 공유(서버 DB, API 조회)에 최적화되어 있다.

4. **10개 차원에서 클라이언트 5승, 서버 5승** — 어느 쪽이 "더 좋다"가 아니라, 적용 영역이 다르다. 판단 기준은: "이 작업의 결과를 나중에 **증명하거나 재현**해야 하는가?"

5. **하이브리드가 최적.** 본 연구의 모든 실험 자체가 "Claude Code as Master + OpsClaw as Control Plane" 하이브리드 모드로 수행되었다. 클라이언트 하네스의 유연성과 서버 하네스의 증적·학습을 결합하는 하이브리드 아키텍처가 양쪽의 장점을 모두 취할 수 있다.

향후 연구에서는 Claude Code 훅 → OpsClaw 자동 연동, Playbook ↔ Skill 상호 변환, auto-memory ↔ Experience 동기화를 통해 하이브리드 아키텍처를 더욱 긴밀하게 통합하고, Cursor, Devin, CALDERA 등 다른 시스템으로 비교 범위를 확대할 계획이다.
