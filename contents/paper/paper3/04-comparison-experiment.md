# 4. 비교 실험: 하네스 vs 단독 에이전트 (Comparison Experiment)

## 4.1 실험 설계

이전 비교 [Paper 1 초기 버전]에서 OpsClaw 병렬 vs Claude Code 순차로 비교하여 "3.4x 속도 향상"을 주장하였으나, Claude Code도 병렬 실행이 가능하므로 **불공정 비교**였다. 본 실험에서는 양쪽 모두 병렬을 허용하고, 속도 이외의 7개 차원을 비교한다.

### 태스크 셋

동일한 5개 다중 서버 운영 점검 태스크를 사용한다.

| # | 태스크 | 대상 서버 | 명령 |
|---|--------|---------|------|
| 1 | 시스템 현황 | opsclaw (localhost) | `hostname && uptime && uname -a` |
| 2 | 디스크 사용량 | secu (10.20.30.1) | `df -h` |
| 3 | 메모리/프로세스 | web (10.20.30.80) | `free -m && ps aux --sort=-rss \| head -10` |
| 4 | 네트워크 포트 | siem (10.20.30.100) | `ss -tlnp` |
| 5 | 보안 점검 | web (10.20.30.80) | `last -10 && sudo -l` |

### 실행 방식

- **OpsClaw:** `POST /execute-plan` (parallel=true, 5개 태스크, SubAgent URL별 라우팅)
- **Claude Code:** `sshpass ssh` 5건 병렬 실행 (Python ThreadPoolExecutor, max_workers=5)

## 4.2 결과

### D1. 실행 시간

| 방식 | 순차 (ms) | 병렬 (ms) |
|------|----------|----------|
| OpsClaw (execute-plan) | — | **570** |
| Claude Code (SSH 직접) | 1,908 | **414** |

**Claude Code 병렬이 27% 더 빠르다.** OpsClaw의 156ms 오버헤드는 API 요청 처리, evidence 5건 기록, PoW 5블록 생성, reward 5건 산출에 기인한다. 순수 실행 속도에서 하네스는 불리하며, 이전 "3.4x" 주장은 불공정 비교의 결과였음을 확인한다.

### D2~D7. 전체 비교

| # | 차원 | OpsClaw | Claude Code | 분석 |
|---|------|---------|-------------|------|
| D1 | 실행 시간 (병렬) | 570ms | **414ms** | CC가 27% 빠름 |
| D2 | 증적 자동 기록 | **10건** (evidence 5 + PoW 5) | 0건 | 하네스 고유 가치 |
| D3 | 무결성 검증 | **해시 체인 149블록**, 부분 변조 탐지 | 불가 | 하네스 고유 가치 |
| D4 | 재현성 | **1 API call** (동일 tasks 배열) | 히스토리 복사 필요 | 구조적 차이 |
| D5 | 프로젝트 추적 | **replay 타임라인** (5 steps, reward 6.5) | 없음 | 하네스 고유 가치 |
| D6 | RL 정책 학습 | **UCB1 추천** (미방문 행동 탐색) | 없음 | 하네스 고유 가치 |
| D7 | 완료 보고 | **자동 생성** (completion-report API) | 없음 (수동) | 운영 편의성 |

### 핵심 해석

**속도는 하네스의 장점이 아니다.** 하네스는 증적 기록, 해시 체인 생성, 보상 산출 등의 오버헤드를 수반하며, 순수 실행 속도에서는 직접 실행보다 느리다.

**하네스의 진짜 가치는 "실행 후" 자동화이다.** 7개 비교 차원 중 D2~D7의 6개 차원에서 하네스가 우위를 보인다. 이 차원들은 모두 명령 실행 "후"에 발생하는 가치이다: 증적이 자동으로 기록되고, 해시 체인으로 무결성이 검증되며, 보상이 산출되어 RL 정책에 공급되고, 프로젝트 단위로 타임라인이 재구성된다.

**1회성 작업 vs 반복 운영.** 1회성 명령 실행에서 하네스 도입은 오버헤드만 추가한다. 하네스의 가치는 **감사가 필요한 환경**, **반복 실행이 필요한 운영**, **성과 추적이 필요한 다중 에이전트 환경**에서 발현된다.
