# 7. 결론 (Conclusion)

본 논문은 IT 운영·보안 자동화 플랫폼 OpsClaw를 M0(설계 기준선)부터 M25(보안 실험 운용)까지 25개 마일스톤에 걸쳐 점진적으로 구축·운용한 사례를 보고하였다. 주요 결과는 다음과 같다.

**설계 결정.** 설계 기준 문서의 선행 고정(L1), 외부 런타임 어댑터 경계(L2), 데이터 기반 상태 전이 차단(L3), Master 교체 가능 아키텍처(L4)가 25 마일스톤에 걸친 점진적 개발의 일관성을 유지하는 핵심 요소였다.

**공정 비교 실험.** 양쪽 모두 병렬 실행을 허용한 공정 비교에서 Claude Code 직접 SSH(414ms)가 OpsClaw(570ms)보다 27% 빠르다. 하네스의 가치는 실행 속도가 아닌 "실행 후" 자동화—증적 10건 자동 기록, 해시 체인 무결성 검증, 보상 산출, RL 정책 학습, 프로젝트 타임라인 재구성, 완료 보고 자동 생성—에 있다.

**검증 가능한 작업 기록(VWR).** PoW 해시 체인과 보상 체계를 "에이전트 작업의 검증 가능한 영수증"으로 재정의하고, 에이전트 성과 평가, 비용 정산, 컴플라이언스 증빙, 에이전트 마켓플레이스, SLA 모니터링의 5가지 활용 시나리오를 제시하였다.

**교훈.** 8개의 핵심 교훈(L1~L8)을 아키텍처, 운영, 보안 범주로 정리하였다. 특히 "control-plane 보안은 보호 대상만큼 중요하다"(L7)와 "타임스탬프에 절대 의존하지 마라"(L5)는 실운영에서 발견된 Critical/High 수준의 교훈이다.

### 교육 플랫폼 확장

실험 이후 OpsClaw는 통합 교육 플랫폼으로 확장되었다:
- 10개 보안 과목(150강, 450시간) 커리큘럼 구축
- 10권 120장 보안 시나리오(소설) 집필 — 교안 120개가 스토리에 매핑
- CTFd 기반 실습 문제 플랫폼
- RAG 기반 AI 튜터 (288문서 인덱싱, 스트리밍 응답)
- 에이전트 소셜 시스템 (5개 AI 에이전트가 커뮤니티 자율 참여)

향후 연구에서는 Codex CLI 및 CALDERA와의 추가 비교, VWR 기반 멀티 테넌트 비용 정산의 실증, 한국어 FTS 지원, 그리고 컨테이너 기반 SubAgent 배포 표준화를 통해 플랫폼의 실용성을 확대할 계획이다.
