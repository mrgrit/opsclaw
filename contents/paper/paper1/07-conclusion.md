# 7. 결론 (Conclusion)

본 논문은 IT 운영·보안 자동화를 위한 자기 개선형 에이전트 하네스 **OpsClaw**를 제안하였다. OpsClaw는 Master→Manager→SubAgent 3계층 위임 아키텍처, 검증 가능한 작업 기록(Verifiable Work Record) 기반 해시 체인, Q-learning + UCB1 강화학습 정책 엔진, Playbook 기반 결정론적 재현, 4-Layer 경험 메모리와 RAG를 통합하여, 에이전트 실행의 추적성, 재현성, 자율 개선을 동시에 보장한다.

실환경 4대 서버에서의 8개 기반 검증 실험과 단독 에이전트 비교 실험을 통해 다음의 결과를 입증하였다:

- **작업 기록 무결성:** 해시 체인 부분 변조 탐지율 100%, 위양성 0%, 5개 시간대에서 해시 일치 100% (실험 A, H)
- **RL 정책 수렴:** 5 에포크 내 Q-value delta < 0.001 수렴, UCB1 미방문 행동 정확 추천 (실험 B)
- **Playbook 재현성:** 10회 반복 의미적 완전 일치 100% (실험 G)
- **아키텍처 안전성:** 무효 전이 차단율 100%, 증거 완전율 100%, 병렬 4.0배 가속 (실험 E, C, F)
- **경험 재활용:** 자동 승급 20건, 컨텍스트 주입 시 응답 품질 51% 향상 (실험 D)
- **하네스 프리미엄:** Claude Code 단독 대비 실행당 10건 자동 증적 + 구조적 재현·감사·보상 추적 (순수 실행 속도는 하네스 오버헤드로 직접 실행 대비 다소 느리나, "실행 후" 가치가 이를 상쇄)

이러한 결과는 오케스트레이션 하네스가 단독 LLM 에이전트의 구조적 한계—증적 부재, 비결정론, 정책 학습 불가—를 효과적으로 해결할 수 있음을 보여준다. 특히 검증 가능한 작업 기록과 보상 체계는 에이전트 성과 평가, 비용 정산, 컴플라이언스 증빙, 에이전트 마켓플레이스 등 다양한 운영 시나리오로 확장 가능하며, RL 기반 자율 정책 개선과의 통합은 기존 에이전트 프레임워크에서 다루지 않은 새로운 연구 영역을 개척한다.

향후 연구에서는 Deep RL을 통한 상태 공간 확장, 다중 SubAgent 분산 아키텍처, Multi-LLM 오케스트레이션, 검증 가능한 작업 기록 기반 멀티 테넌트 비용 정산, 그리고 한국어 FTS 지원을 통해 OpsClaw의 적용 범위와 성능을 더욱 확대할 계획이다.
