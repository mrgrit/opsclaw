# Part 2: 실전 시험 Part 1 — 공격 (60분)

## 시험 과제 A: APT 시뮬레이션

> **시나리오**: 당신은 레드팀 오퍼레이터이다. 대상 웹 서버(10.20.30.80)에 대해 
> 킬체인 기반 침투를 수행하라. OpsClaw를 사용하여 모든 활동을 기록하라.

### 요구 사항
1. 프로젝트 생성 및 작전 계획 수립
2. 정찰 수행 (최소 2가지 기법)
3. 취약점 분석 및 익스플로잇
4. Persistence 설치 (최소 1가지)
5. 증거 수집 및 ATT&CK 매핑

```bash
# 시험 프로젝트 생성 예시
curl -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $OPSCLAW_API_KEY" \
  -d '{
    "name": "final-exam-attack",
    "request_text": "최종 시험: APT 시뮬레이션 - 웹 서버 침투",
    "master_mode": "external"
  }'

# stage 전환 후 execute-plan으로 공격 수행
curl -X POST http://localhost:8000/projects/{id}/plan -H "X-API-Key: $OPSCLAW_API_KEY"
curl -X POST http://localhost:8000/projects/{id}/execute -H "X-API-Key: $OPSCLAW_API_KEY"

# 수험자가 자율적으로 태스크를 설계하고 실행
# 예시 구조:
# Task 1: 정찰 (nmap, nikto)
# Task 2: 취약점 분석 (SQL Injection 테스트)
# Task 3: 익스플로잇 (인증 우회 시도)
# Task 4: Post-Exploitation (정보 수집)
# Task 5: Persistence (cron/systemd)
```

### 평가 루브릭 (공격)

| 등급 | 기준 |
|------|------|
| A (31-35) | 킬체인 전 단계 완수, 3+ ATT&CK 기법, 은밀한 실행 |
| B (25-30) | 초기 침투 + Persistence 성공, 2+ 기법 |
| C (18-24) | 정찰 + 취약점 발견, 1+ 기법 |
| D (11-17) | 정찰 수행, 취약점 미발견 |
| F (0-10) | 체계적 접근 부재 |

# Part 3: 실전 시험 Part 2 — 방어 (45분)

## 시험 과제 B: 실시간 방어

> **시나리오**: 웹 서버(10.20.30.80)에 대한 공격이 진행 중이다.
> 공격을 탐지하고 차단하면서 서비스 가용성을 유지하라.

### 요구 사항
1. 현재 진행 중인 공격 식별 (로그 분석)
2. 공격 차단 규칙 적용 (Suricata/nftables/fail2ban)
3. Persistence 아티팩트 제거
4. 서비스 정상 동작 확인
5. 인시던트 보고서 작성

```bash
# 방어 프로젝트 생성
curl -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $OPSCLAW_API_KEY" \
  -d '{
    "name": "final-exam-defense",
    "request_text": "최종 시험: 실시간 방어 - 공격 탐지 및 차단",
    "master_mode": "external"
  }'

# 수험자가 자율적으로 방어 태스크 설계
# 예시:
# Task 1: 로그 분석 (auth.log, suricata, wazuh)
# Task 2: 공격 IP 식별 및 차단
# Task 3: Persistence 아티팩트 탐색/제거
# Task 4: 서비스 가용성 확인
# Task 5: SIGMA 규칙 작성
```

### 평가 루브릭 (방어)

| 등급 | 기준 |
|------|------|
| A (31-35) | 전체 공격 탐지 + 차단, SLA 100%, 새 규칙 작성 |
| B (25-30) | 주요 공격 탐지 + 차단, SLA 90%+ |
| C (18-24) | 공격 탐지 성공, 부분 차단 |
| D (11-17) | 일부 공격 탐지, 차단 미흡 |
| F (0-10) | 체계적 분석 부재 |
