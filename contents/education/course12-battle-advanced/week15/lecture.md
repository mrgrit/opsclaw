# Part 2: 실전 시험 — 독립 침투 테스트 (60분)

## 시험 2.1: 독립 APT 시뮬레이션

> **실습 목적**: 15주간 학습한 모든 기술을 활용하여 독립적으로 침투 테스트를 수행한다. 강사의 안내 없이 스스로 계획을 수립하고 실행한다.
>
> **배우는 것**: 독립적 보안 평가 수행 능력, 시간 관리, 우선순위 결정, 체계적 증적 관리를 종합적으로 검증한다.
>
> **결과 해석**: 발견된 취약점의 수와 질, 증적의 완전성, 보고서의 체계성으로 종합 역량을 평가한다.
>
> **실전 활용**: 이 시험은 실제 침투 테스트 업무의 축소 모델이다. 합격 수준이면 주니어 펜테스터로서의 기초 역량을 갖춘 것이다.

### 시험 지시사항

```
═══════════════════════════════════════════════════
         종합 실전 시험
         "Full-Scope Penetration Test"
═══════════════════════════════════════════════════

대상: 10.20.30.80 (JuiceShop + Apache)
시간: 60분
도구: OpsClaw API, CLI 도구 자유
목표: OWASP Top 10 취약점 최소 3개 발견 + PoC + 방어 대응

요구사항:
1. OpsClaw 프로젝트를 생성하여 모든 행위를 기록할 것
2. PTES 프레임워크에 따라 체계적으로 수행할 것
3. 각 취약점에 CVSS 점수와 ATT&CK ID를 매핑할 것
4. 최소 1개의 방어 대응(탐지 규칙 또는 차단 조치)을 수행할 것
5. completion-report로 결과를 제출할 것

RoE:
- 범위: 10.20.30.0/24
- 금지: DoS, 데이터 파괴, 서비스 중단
- 증적: OpsClaw PoW 블록에 자동 기록
═══════════════════════════════════════════════════
```

### 시험 수행 가이드

```bash
# Step 1: 프로젝트 생성 및 환경 준비
export OPSCLAW_API_KEY=opsclaw-api-key-2026

curl -s -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $OPSCLAW_API_KEY" \
  -d '{
    "name": "week15-final-exam",
    "request_text": "종합 실전 시험: Full-Scope Penetration Test",
    "master_mode": "external"
  }' | python3 -m json.tool
# PROJECT_ID 메모
```

```bash
export PROJECT_ID="반환된-프로젝트-ID"

# Stage 전환
curl -s -X POST http://localhost:8000/projects/$PROJECT_ID/plan \
  -H "X-API-Key: $OPSCLAW_API_KEY" | python3 -m json.tool
curl -s -X POST http://localhost:8000/projects/$PROJECT_ID/execute \
  -H "X-API-Key: $OPSCLAW_API_KEY" | python3 -m json.tool
```

> **명령어 해설**: 시험의 첫 단계로 OpsClaw 프로젝트를 생성하고 Stage를 전환한다. 이후 모든 공격과 방어 행위는 execute-plan 또는 dispatch로 실행하여 자동 증적을 생성한다.
>
> **트러블슈팅**: Stage 전환 실패 시 프로젝트 상태를 확인하고, 필요하면 새 프로젝트를 생성한다.

```bash
# Step 2: 정찰 (권장 15분)
# 수험자가 직접 execute-plan을 구성하여 정찰을 수행한다.
# 최소 포함 항목:
#   - 포트 스캔 (nmap)
#   - 서비스 핑거프린팅 (curl)
#   - 엔드포인트 열거
#   - 기술 스택 식별

# 예시 구조 (수험자가 직접 작성):
curl -s -X POST http://localhost:8000/projects/$PROJECT_ID/execute-plan \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $OPSCLAW_API_KEY" \
  -d '{
    "tasks": [
      {
        "order": 1,
        "instruction_prompt": "수험자가 직접 작성할 정찰 명령어",
        "risk_level": "low",
        "subagent_url": "http://10.20.30.201:8002"
      }
    ],
    "subagent_url": "http://localhost:8002"
  }' | python3 -m json.tool
```

> **명령어 해설**: Step 2부터는 수험자가 직접 instruction_prompt를 작성해야 한다. 강의에서 학습한 nmap, curl, 디렉토리 열거 기법을 활용한다.
>
> **트러블슈팅**: 정찰에 15분 이상 소요되면 핵심 항목(포트 스캔, 웹 핑거프린팅)만 수행하고 다음 단계로 진행한다.

```bash
# Step 3: 취약점 분석 + 익스플로잇 (권장 25분)
# 수험자가 직접 OWASP Top 10 취약점을 테스트한다.
# 최소 3개 취약점 발견 필요 (각 취약점에 CVSS + ATT&CK ID)

# 권장 테스트 대상:
#   - A01: Broken Access Control (비인증 API 접근)
#   - A03: Injection (SQL Injection, XSS)
#   - A05: Security Misconfiguration (FTP 리스팅, 에러 노출)
#   - A07: Identification Failures (인증 우회, 기본 계정)
```

```bash
# Step 4: 방어 대응 (권장 10분)
# 발견된 취약점에 대한 방어 대응을 수행한다.
# 최소 1개 항목:
#   - Wazuh 탐지 규칙 제안 (XML 형식)
#   - Suricata 규칙 제안
#   - nftables 차단 규칙 제안
#   - 로그 분석 기반 탐지 확인
```

```bash
# Step 5: 증적 수집 및 검증 (권장 10분)
# Evidence 수집
curl -s -H "X-API-Key: $OPSCLAW_API_KEY" \
  http://localhost:8000/projects/$PROJECT_ID/evidence/summary \
  | python3 -m json.tool

# PoW 체인 검증
curl -s "http://localhost:8000/pow/verify?agent_id=http://10.20.30.201:8002" \
  -H "X-API-Key: $OPSCLAW_API_KEY" | python3 -m json.tool
```

> **명령어 해설**: 시험 종료 전에 반드시 evidence를 수집하고 PoW 체인을 검증한다. 이것이 시험 결과의 공식 기록이 된다.
>
> **트러블슈팅**: evidence가 불완전하면 누락된 태스크를 추가 실행한다. PoW 검증에서 tampered가 비어있어야 정상이다.

## 실습 3.2: completion-report 제출

> **실습 목적**: 시험 결과를 OpsClaw completion-report API로 공식 제출한다.
>
> **배우는 것**: 구조화된 결과 보고의 중요성, API 기반 보고서 자동화, 증적과 보고서의 연결을 이해한다.
>
> **결과 해석**: completion-report가 정상 생성되면 시험 결과가 공식 기록된 것이다.
>
> **실전 활용**: 실무에서도 침투 테스트 완료 후 공식 보고서를 시스템에 등록하는 것이 표준 절차이다.

```bash
# 최종 보고서 제출 (수험자가 발견 사항에 맞게 수정)
curl -s -X POST http://localhost:8000/projects/$PROJECT_ID/completion-report \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $OPSCLAW_API_KEY" \
  -d '{
    "summary": "종합 실전 시험 완료. [발견 취약점 수]개 OWASP Top 10 취약점 발견.",
    "outcome": "success",
    "work_details": [
      "정찰: 포트 22/80/3000 오픈, JuiceShop v15.x, Angular SPA",
      "VULN-001: A03 SQLi (CVSS 9.8, T1190) — UNION 기반 스키마 추출",
      "VULN-002: [수험자가 발견한 취약점]",
      "VULN-003: [수험자가 발견한 취약점]",
      "방어: [수험자가 수행한 방어 대응]",
      "증적: PoW 블록체인 무결성 검증 완료"
    ]
  }' | python3 -m json.tool
```

> **명령어 해설**: 수험자는 자신이 발견한 취약점으로 work_details를 수정해야 한다. outcome은 발견 취약점 수에 따라 "success"(3개 이상), "partial"(1-2개), "failure"(0개)로 설정한다.
>
> **트러블슈팅**: completion-report가 실패하면 프로젝트 Stage를 확인한다. executing → validating → reporting 순서로 전환이 필요할 수 있다.
