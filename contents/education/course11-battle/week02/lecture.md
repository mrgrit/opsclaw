# Part 1: 취약점 스캐닝 이론 (30분)

## 1.1 취약점 스캐닝의 목적

취약점 스캐닝은 정찰 단계에서 수집한 정보를 기반으로 대상 시스템의 **알려진 취약점**을 자동으로 탐지하는 과정이다. MITRE ATT&CK에서 **T1595.002 (Vulnerability Scanning)** 기법에 해당한다.

### 정찰 vs 취약점 스캐닝

| 구분 | 정찰 (Week 01) | 취약점 스캐닝 (Week 02) |
|------|---------------|----------------------|
| 목적 | 호스트/서비스 발견 | 취약점 식별 |
| 노이즈 | 낮음~중간 | 높음 |
| 결과물 | IP, 포트, 서비스 목록 | CVE 목록, 위험도 |
| 도구 | nmap -sV | NSE, nikto, OpenVAS |

## 1.2 NSE(Nmap Scripting Engine) 개요

NSE는 nmap에 내장된 스크립트 엔진으로, Lua 언어로 작성된 스크립트를 통해 다양한 보안 점검을 자동화한다.

### NSE 스크립트 카테고리

| 카테고리 | 설명 | 예시 |
|---------|------|------|
| `auth` | 인증 관련 점검 | 기본 계정 시도 |
| `vuln` | 취약점 탐지 | CVE 확인 |
| `exploit` | 취약점 실제 활용 | 주의: 대상에 영향 가능 |
| `discovery` | 추가 정보 수집 | DNS, SNMP |
| `safe` | 안전한 스크립트만 | 대상에 영향 없음 |
| `default` | 기본 실행 세트 | `-sC`와 동일 |

# Part 2: 실습 가이드

## 실습 2.1: NSE 스크립트 활용

> **목적**: NSE를 사용하여 대상 서비스의 취약점을 자동 탐지한다
> **배우는 것**: NSE 스크립트 선택, 실행, 결과 해석

```bash
# 기본 스크립트 스캔 (-sC = --script=default)
nmap -sC -sV 10.20.30.80

# vuln 카테고리 전체 실행
nmap --script vuln 10.20.30.80

# 특정 스크립트 실행
nmap --script http-title,http-headers 10.20.30.80

# SSL/TLS 점검
nmap --script ssl-enum-ciphers -p 443 10.20.30.80

# SMB 취약점 점검
nmap --script smb-vuln* -p 445 10.20.30.80

# 설치된 스크립트 목록 확인
ls /usr/share/nmap/scripts/ | grep http
```

> **결과 해석**: 스크립트 출력에서 `VULNERABLE` 표시가 있으면 해당 취약점이 존재한다. CVE 번호가 함께 표시되므로 심각도를 확인할 수 있다.
> **실전 활용**: 공방전에서 `-sC -sV` 조합은 빠른 정보 수집의 기본이다.

## 실습 2.2: nikto 웹 취약점 스캐닝

> **목적**: 웹 서버의 설정 오류와 알려진 취약점을 탐지한다
> **배우는 것**: nikto 사용법과 결과 분석

```bash
# 기본 스캔
nikto -h http://10.20.30.80

# 특정 포트 지정
nikto -h 10.20.30.80 -p 3000

# SSL 사이트 스캔
nikto -h https://10.20.30.80 -ssl

# 출력을 HTML 보고서로 저장
nikto -h http://10.20.30.80 -o /tmp/nikto_report.html -Format html

# 튜닝: 특정 테스트만 수행
# 1=파일, 2=설정오류, 3=정보노출, 4=XSS
nikto -h http://10.20.30.80 -Tuning 234
```

> **결과 해석**: 각 항목에 OSVDB 또는 CVE 번호가 표시된다. `+` 기호로 시작하는 라인이 발견된 이슈이다.
> **실전 활용**: 웹 서버 대상 공방전에서 초기 취약점 파악에 효과적이다.

## 실습 2.3: 배너 그래빙

> **목적**: 서비스가 반환하는 배너 정보에서 버전을 추출한다
> **배우는 것**: netcat, curl을 활용한 수동 배너 그래빙

```bash
# netcat으로 배너 그래빙
echo "" | nc -w 3 10.20.30.80 22

# HTTP 헤더에서 서버 정보 추출
curl -sI http://10.20.30.80 | grep -i server

# nmap 배너 그래빙 스크립트
nmap --script banner -p 21,22,25,80 10.20.30.80

# telnet으로 SMTP 배너 확인
echo "QUIT" | nc -w 3 10.20.30.80 25
```

> **결과 해석**: 배너에서 소프트웨어 이름과 버전을 확인하고 CVE 데이터베이스에서 해당 버전의 취약점을 검색한다.

# Part 3: 심화 학습

## 3.1 CVE 매핑과 위험도 평가

스캔 결과에서 발견된 서비스 버전을 CVE와 매핑하는 절차:

1. 서비스 이름 + 버전 확인 (예: Apache 2.4.49)
2. CVE 검색 사이트에서 조회 (cve.mitre.org, nvd.nist.gov)
3. CVSS 점수로 위험도 분류 (Critical: 9.0+, High: 7.0+)
4. 공격 가능성과 영향도를 종합 평가

## 3.2 방어자 대응

- 불필요한 배너 정보 제거 (서버 버전 숨기기)
- IDS에서 스캐닝 패턴 탐지 룰 설정
- 취약한 버전의 서비스를 즉시 패치
