# Part 1: 네트워크 정찰 이론 (30분)

## 1.1 정찰이란?

정찰(Reconnaissance)은 공격의 첫 번째 단계로, 대상 시스템과 네트워크에 대한 정보를 수집하는 과정이다. MITRE ATT&CK 프레임워크에서 **TA0043 (Reconnaissance)** 전술에 해당한다.

### 정찰의 분류

| 유형 | 설명 | 예시 | 탐지 난이도 |
|------|------|------|------------|
| **수동 정찰** | 대상과 직접 통신하지 않음 | OSINT, DNS 조회, WHOIS | 매우 어려움 |
| **능동 정찰** | 대상에 직접 패킷 전송 | 포트 스캔, 배너 그래빙 | 탐지 가능 |

### 공방전에서의 정찰

공방전(CTF/공방 훈련)에서 정찰은 **승패를 결정짓는 핵심 단계**이다.

- **Red Team**: 대상의 공격 표면을 파악하여 침투 경로를 설계
- **Blue Team**: 자산 목록과 노출 서비스를 파악하여 방어 우선순위 설정

## 1.2 네트워크 스캐닝 기법

### ping sweep

ICMP Echo Request를 이용하여 네트워크 내 활성 호스트를 탐색하는 기법이다.

```
호스트 A (스캐너)                    네트워크
   │                               ┌─ 호스트 1 (응답)
   ├── ICMP Echo Request ──────────┤─ 호스트 2 (무응답)
   │                               ├─ 호스트 3 (응답)
   │                               └─ 호스트 4 (방화벽 차단)
   └── 결과: 호스트 1, 3 활성 확인
```

### 포트 스캔 유형

| 스캔 유형 | nmap 옵션 | 특징 | 탐지 용이성 |
|-----------|----------|------|------------|
| TCP Connect | `-sT` | 완전한 3-way handshake | 높음 (로그 남음) |
| SYN 스캔 | `-sS` | half-open, 빠름 | 중간 |
| UDP 스캔 | `-sU` | UDP 서비스 탐지 | 낮음 (느림) |
| FIN 스캔 | `-sF` | 스텔스 스캔 | 낮음 |
| NULL 스캔 | `-sN` | 플래그 없는 패킷 | 낮음 |

# Part 2: 실습 가이드

## 실습 2.1: ping sweep로 활성 호스트 탐색

> **목적**: 네트워크 내 활성 호스트를 빠르게 식별한다
> **배우는 것**: ICMP 기반 호스트 탐색과 nmap ping sweep 사용법

```bash
# 기본 ping으로 단일 호스트 확인
ping -c 3 10.20.30.1

# nmap ping sweep (ICMP + ARP)
nmap -sn 10.20.30.0/24

# ARP 스캔 (같은 서브넷 내에서 더 정확)
nmap -sn -PR 10.20.30.0/24

# 결과를 파일로 저장
nmap -sn 10.20.30.0/24 -oN /tmp/ping_sweep.txt
```

> **결과 해석**: `Host is up` 표시가 있는 IP가 활성 호스트이다. 응답 시간(latency)이 함께 표시되며, 이를 통해 네트워크 거리를 추정할 수 있다.
> **실전 활용**: 공방전 시작 직후 가장 먼저 수행하여 대상 범위를 파악한다.

## 실습 2.2: 포트 스캔과 서비스 탐지

> **목적**: 대상 호스트의 열린 포트와 실행 중인 서비스를 파악한다
> **배우는 것**: nmap 포트 스캔 옵션과 서비스/버전 탐지

```bash
# 기본 포트 스캔 (상위 1000개 포트)
nmap 10.20.30.80

# 전체 포트 스캔 (1-65535)
nmap -p- 10.20.30.80

# 서비스 버전 탐지
nmap -sV 10.20.30.80

# OS 탐지 + 서비스 버전 + 스크립트 스캔
nmap -A 10.20.30.80

# 특정 포트 범위만 스캔
nmap -p 80,443,8080,8443 10.20.30.80

# 스캔 속도 조절 (T0=느림/스텔스 ~ T5=빠름/노이즈)
nmap -sS -T2 10.20.30.80
```

> **결과 해석**: `STATE` 컬럼에서 `open`은 서비스가 수신 대기 중, `filtered`는 방화벽이 차단, `closed`는 포트가 닫힌 상태를 의미한다.
> **실전 활용**: `-sV` 결과에서 서비스 버전을 확인하고 알려진 취약점(CVE)을 검색한다.

## 실습 2.3: 스캔 결과 정리 및 분석

> **목적**: 스캔 결과를 체계적으로 정리하여 공격 계획을 수립한다
> **배우는 것**: nmap 출력 포맷과 결과 분석 방법

```bash
# XML 출력 (후속 도구 연동용)
nmap -sV -oX /tmp/scan_result.xml 10.20.30.80

# 모든 포맷으로 동시 저장
nmap -sV -oA /tmp/full_scan 10.20.30.0/24

# grep 가능한 출력
nmap -sV -oG /tmp/scan_grep.txt 10.20.30.0/24

# 결과에서 열린 포트만 추출
grep "open" /tmp/scan_grep.txt
```

> **결과 해석**: XML 출력은 Metasploit 등 후속 도구와 연동할 수 있다. grepable 포맷은 스크립트로 빠르게 파싱할 때 유용하다.

# Part 3: 심화 학습

## 3.1 Blue Team 관점의 스캔 탐지

방어자 입장에서 네트워크 스캔을 탐지하는 방법을 이해해야 한다.

- **방화벽 로그**: 짧은 시간 내 다수 포트 접근 시도 기록
- **IDS 룰**: Suricata/Snort에서 포트 스캔 탐지 룰 활용
- **네트워크 모니터링**: 비정상적인 ICMP 트래픽 패턴 감시

```bash
# Suricata 포트 스캔 탐지 로그 확인 (siem 서버)
cat /var/log/suricata/fast.log | grep "portscan"
```

## 3.2 스캔 회피 기법

공격자가 사용하는 스캔 회피 기법을 알아야 방어도 가능하다.

- **Decoy 스캔**: 가짜 소스 IP를 섞어 출처를 은폐
- **Fragmentation**: 패킷 분할로 IDS 우회 시도
- **타이밍 조절**: 느린 스캔으로 탐지 임계값 회피
