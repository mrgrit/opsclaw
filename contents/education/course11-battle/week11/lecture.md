# Part 2: 실습 가이드

## 실습 2.1: Red Team — 정찰 수행

> **목적**: 탐지를 최소화하면서 Blue Team 인프라를 정찰한다
> **배우는 것**: 스텔스 스캐닝, 정찰 계획 수립

```bash
# 정찰 계획 (60분 타임라인)
# 0-10분: 느린 ping sweep
# 10-25분: 주요 포트 스텔스 스캔
# 25-40분: 서비스 버전 탐지
# 40-55분: 취약점 스캔
# 55-60분: 결과 정리

# 1단계: 느린 ping sweep (탐지 회피)
nmap -sn -T1 --max-rate 5 10.20.30.0/24

# 2단계: 스텔스 SYN 스캔 (주요 포트만)
nmap -sS -T2 --max-rate 10 -p 22,80,443,3000,8080 10.20.30.Y

# 3단계: 서비스 버전 탐지 (특정 포트만)
nmap -sV -T2 -p 22,80 10.20.30.Y

# 4단계: 특정 NSE 스크립트만 실행
nmap --script http-title,http-server-header -p 80 10.20.30.Y

# 결과 저장
nmap -sS -sV -oA /tmp/recon_result 10.20.30.Y
```

> **결과 해석**: `-T1`이나 `-T2`로 속도를 낮추면 탐지 확률이 줄어든다. `--max-rate`로 초당 패킷 수를 제한한다.
> **실전 활용**: 공방전에서 무작정 `-T5`로 빠르게 스캔하면 즉시 탐지되므로, 속도와 정보량의 균형을 맞춘다.

## 실습 2.2: Blue Team — 탐지 체계 구축

> **목적**: Red Team의 정찰 활동을 실시간 탐지하고 대응한다
> **배우는 것**: IDS 모니터링, 로그 실시간 감시, 차단 대응

```bash
# 준비 시간 (15분) 체크리스트
# 1. 방화벽 기본 정책 적용
sudo nft flush ruleset
sudo nft -f /etc/nftables.conf

# 2. Suricata 시작 + 커스텀 룰
sudo systemctl start suricata

# 3. 실시간 모니터링 시작 (터미널 분할)
# 터미널 1: IDS 알림
tail -f /var/log/suricata/fast.log

# 터미널 2: 방화벽 로그
sudo journalctl -kf | grep "NFT-"

# 터미널 3: 인증 로그
tail -f /var/log/auth.log

# 4. 탐지 시 즉시 차단
# 공격 IP가 확인되면:
sudo nft add element inet filter blocked_ips { 10.20.30.X }

# 5. 탐지 보고서 작성
echo "$(date): 스캔 탐지 - 소스 IP: 10.20.30.X, 유형: SYN scan" >> /tmp/detection_log.txt
```

> **결과 해석**: Suricata fast.log에서 "PORT SCAN" 알림이 뜨면 정찰 활동이 탐지된 것이다. 소스 IP를 즉시 차단하고 기록한다.

# Part 3: 심화 학습

## 3.1 허니팟 기법

Blue Team이 가짜 서비스를 열어 공격자를 유인하고 탐지할 수 있다.

```bash
# 간단한 허니팟: 사용하지 않는 포트를 열어 접근 시도 탐지
sudo nft add rule inet filter input tcp dport 3389 log prefix "HONEYPOT-RDP: " drop
sudo nft add rule inet filter input tcp dport 445 log prefix "HONEYPOT-SMB: " drop
```

## 3.2 Decoy 스캔 (Red Team)

```bash
# 가짜 소스 IP를 섞어 실제 공격 IP를 은폐
nmap -D 10.20.30.50,10.20.30.51,ME 10.20.30.Y
```
