# Part 1: 보안 보고서 작성 방법론 (30분)

## 1.1 보고서 유형

| 유형 | 대상 | 핵심 내용 | 분량 |
|------|------|----------|------|
| **경영진 요약** | 임원, 경영자 | 위험도, 비즈니스 영향, 권고사항 | 1-2페이지 |
| **기술 보고서** | 보안팀, IT팀 | 취약점 상세, 공격 분석, 기술적 조치 | 10-30페이지 |
| **인시던트 보고서** | 관련 이해관계자 | 타임라인, 영향 범위, 대응 내역 | 5-15페이지 |
| **침투 테스트 보고서** | 고객, 보안팀 | 발견 취약점, 증거, 개선 권고 | 20-50페이지 |

## 1.2 보고서 구조

### 표준 보안 보고서 구조

```
1. 경영진 요약 (Executive Summary)
   └── 핵심 발견사항, 위험 등급, 권고사항 요약

2. 범위 및 방법론 (Scope & Methodology)
   └── 테스트 대상, 사용 도구, 규칙

3. 발견사항 (Findings)
   └── 취약점 목록, 위험도 평가, 증거

4. 공격 타임라인 (Attack Timeline)
   └── 시간순 이벤트 기록

5. 영향 분석 (Impact Analysis)
   └── 비즈니스 영향, 데이터 위험, 서비스 영향

6. 권고사항 (Recommendations)
   └── 단기/중기/장기 조치사항

7. 부록 (Appendix)
   └── 상세 로그, 스크린샷, 도구 출력
```

# Part 2: 실습 가이드

## 실습 2.1: 정량 분석

> **목적**: 공방전 결과를 수치로 분석하여 객관적으로 평가한다
> **배우는 것**: 통계 기반 분석, 지표 산출

```bash
# 공격 통계 분석
echo "=== 공격 통계 ==="

# 총 공격 시도 횟수
TOTAL_ATTACKS=$(wc -l < /var/log/suricata/fast.log)
echo "총 IDS 알림: $TOTAL_ATTACKS"

# 공격 유형별 분류
echo "--- 유형별 ---"
grep -oP '\[Classification: [^\]]+' /var/log/suricata/fast.log | \
  sort | uniq -c | sort -rn

# SSH 공격 통계
FAILED_SSH=$(grep -c "Failed password" /var/log/auth.log)
SUCCESS_SSH=$(grep -c "Accepted" /var/log/auth.log)
echo "SSH 실패: $FAILED_SSH, 성공: $SUCCESS_SSH"

# 방어 통계 분석
echo "=== 방어 통계 ==="

# 차단된 IP 수
BLOCKED=$(cat /tmp/blocked_ips.txt 2>/dev/null | wc -l)
echo "차단 IP: $BLOCKED"

# 평균 탐지 시간 (첫 공격 ~ 첫 차단)
echo "탐지 시간 분석은 타임라인에서 수동 계산"

# 서비스 가용성
echo "서비스 다운타임: 계산 필요"
```

> **결과 해석**: 정량적 지표를 통해 공격의 규모와 방어의 효과를 객관적으로 평가한다. 탐지율(탐지 건수/총 공격 건수)과 차단율은 핵심 지표이다.
