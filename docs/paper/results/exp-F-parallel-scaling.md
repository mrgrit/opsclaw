## 분석

- **N=1:** 병렬 오버헤드로 약간 느림 (ThreadPoolExecutor 생성 비용)
- **N=2:** 1.81x — 이론 대비 90.5% 효율
- **N=3:** 2.66x — 이론 대비 88.7% 효율
- **N=5:** 4.00x — 이론 대비 80.0% 효율

### 효율 저하 원인
- ThreadPoolExecutor 생성/수거 오버헤드 (~200ms)
- 로컬 SubAgent가 단일 프로세스 → 동시 요청 시 경합 발생
- PoW 블록 생성이 순차적 (블록 간 prev_hash 의존)

### 실운영 기대치
- 분산 SubAgent (서버별 독립 실행) 시 경합 해소 → 이론치에 근접 예상
- max_workers=5 제한은 SubAgent 과부하 방지에 적절

## 발견사항

### 우수성
- 단일 API 호출(`parallel:true`)로 병렬 실행 — 사용자 개입 불필요
- PoW 블록이 병렬 태스크에서도 모두 정상 생성됨

### 한계/개선 필요
- **단일 SubAgent 병목:** 로컬 SubAgent가 동시 요청을 순차 처리하는 경향
  → 개선안: SubAgent를 asyncio 기반으로 전환, 또는 서버별 SubAgent 분리
- **PoW 블록 순서:** 병렬 실행 시 PoW 블록이 완료 순서대로 생성됨 (task order와 다를 수 있음)
  → 기능적 문제는 아니나, 타임라인 재구성 시 주의 필요
