## 1) M3-4.1에서 최종적으로 ‘성공한’ 것(확정)
### 1.1 Target Registry
- Manager API에 Targets CRUD 구현
  - `GET /targets`
  - `GET /targets/{target_id}`
  - `POST /targets` (upsert)
  - `DELETE /targets/{target_id}`
- targets 저장 파일: `/data/state/_targets.json`
  - 빈 파일이면 `JSONDecodeError`로 500났던 이슈가 있었고, 초기화로 복구함.

### 1.2 원격 SubAgent 준비(원격 서버)
- 원격 서버 IP: **192.168.208.138**
- 원격 SubAgent 컨테이너가 55123 리슨(0.0.0.0) 및 정상 동작
  - `GET /health` ✅
  - `POST /a2a/run_script` ✅ (실제 명령 실행 및 결과 반환)
- 원격 evidence는 SubAgent 컨테이너 내부 `/data/evidence`에 생성됨.

### 1.3 Manager → Target 기반 원격 실행 라우팅 성공
- target 등록 예:
  - id: `remote-1`
  - base_url: `http://192.168.208.138:55123`
- Manager에서 프로젝트 run 호출 시 `target_id=remote-1`로 원격 SubAgent에 라우팅되어 실행 성공:
  - `POST /projects/{project_id}/run`
  - body에 `"target_id":"remote-1"` + `"command":"uname -a"` 등
  - 응답에서 `exit_code=0`, stdout 정상, evidence_refs 생성 확인
- 핵심 포인트:
  - SubAgent 실행 API 경로는 `/run`이 아니라 **`/a2a/run_script`** 임.
  - Manager는 `target_id -> base_url` resolve 후 해당 SubAgent로 실행 요청을 보냄.

## 2) 지금 남은 큰 문제/제약(기억해야 할 것)
- Manager 응답의 `evidence_refs`는 “원격 SubAgent의 파일경로”를 그대로 담음.
  - 즉, 현재 Evidence Pack ZIP에 **원격 evidence를 자동으로 pull해서 포함하는 기능은 아직 없음.**
- “멀티 타겟 작업 분배(팬아웃) / 의존성(DAG) / 플레이북 기반 실행”은 아직 미구현.

## 3) M3-4.2에서 하려는 것(시작 범위)
### 목표: Playbook을 “target 기반”으로 실행 가능하게 만들기
- playbook 실행 요청에 `target_id` 또는 `target_ids`를 넣고,
- playbook step 단위 또는 job 단위로 target을 지정해서,
- Manager가 각 step 실행을 해당 target의 SubAgent로 라우팅하게 만든다.

### 최소 요구사항(권장)
- playbook 스키마(v0):
  - `steps: [{ id, title, target_id?, script, timeout_s?, evidence_requests? }]`
  - step에 target_id가 없으면 default target 사용
- 실행 결과를 프로젝트 state에 step/run 단위로 누적 저장
- 실패 처리(초기): step 하나라도 실패하면 stop (옵션으로 continue 추가 가능)

## 4) 기존에 겪은 대표 이슈(재발 방지)
- main.py 패치 과정에서 `IndentationError`, `return outside function`, `NameError(Request/TargetReg/authz/list_targets/resolve_subagent_url)` 자주 발생
  - 해결 방식: `python3 -m py_compile api/main.py`로 문법 선검증 후 docker rebuild
- subagent endpoint 혼동(`/run` vs `/a2a/run_script`)로 404
- `/data/state/_targets.json`이 빈 파일이면 JSONDecodeError로 500
