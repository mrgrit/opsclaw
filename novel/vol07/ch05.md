### 점검 결과

**1. Privileged 모드: false**

"**--privileged 모드가 아니야.** 이건 가장 중요한 항목이야." 정규가 말했다. "privileged 컨테이너는 호스트의 모든 장치에 접근할 수 있고, 커널 모듈을 로드할 수 있어. **컨테이너 탈출(container escape)**의 가장 흔한 원인이야."

"genesis는 privileged가 아니니까 **탈출 위험이 크게 줄어들어.**" 가람이 정리했다.

**2. seccomp 프로필: 적용됨**

"**seccomp 프로필**이 적용되어 있어. genesis_seccomp.json." 정규가 말했다.

```bash
# genesis seccomp 프로필 확인
cat /etc/docker/seccomp/genesis_seccomp.json | python3 -c "
import sys, json
data = json.load(sys.stdin)
print(f'Default action: {data[\"defaultAction\"]}')
print(f'Allowed syscalls: {len(data[\"syscalls\"])}')
for sc in data['syscalls'][:5]:
    print(f'  {sc[\"names\"][:3]} → {sc[\"action\"]}')
print('  ...')
"
```

```
Default action: SCMP_ACT_ERRNO
Allowed syscalls: 48
  ['read', 'write', 'open'] → SCMP_ACT_ALLOW
  ['close', 'stat', 'fstat'] → SCMP_ACT_ALLOW
  ['mmap', 'mprotect', 'munmap'] → SCMP_ACT_ALLOW
  ['brk', 'rt_sigaction', 'rt_sigprocmask'] → SCMP_ACT_ALLOW
  ['access', 'getpid', 'socket'] → SCMP_ACT_ALLOW
  ...
```

"기본 정책이 **ERRNO(거부)**이고, **48개 시스콜만 허용.** Docker 기본 seccomp 프로필은 약 300개를 허용하는데, genesis는 **48개만.** 최소 권한 원칙을 철저히 적용한 거야." 정규가 감탄했다.

"이건 추팀장님이 직접 만드신 건가요?" 가람이 물었다.

"genesis가 필요로 하는 시스콜만 **하나씩 추가**했어. 10년 넘게 다듬었지." 추박하가 말했다. "불필요한 시스콜을 차단하면 **커널 취약점 공격을 원천 봉쇄**할 수 있어."

**3. 비root 사용자: genesis**

"User: genesis. **root가 아닌 전용 사용자**로 실행 중." 정규가 말했다. "root로 컨테이너를 실행하면, 컨테이너 탈출 시 **호스트에서도 root 권한**을 가질 수 있어."

**4. Linux Capabilities: 최소화**

"**CapDrop: ALL.** 모든 Linux capability를 제거하고, NET_BIND_SERVICE만 추가. 이건 1024 이하 포트 바인딩에만 필요한 최소 권한이야." 정규가 말했다.

"genesis가 내부적으로 포트 443을 쓰나 봐요." 가람이 추측했다.

"맞아. 내부 API가 443에서 TLS로 통신해." 추박하가 확인했다.

**5. 리소스 제한**

"Memory: 512MB, CPU Quota: 50000(50%). 리소스 제한이 걸려 있어서 **DoS 공격이나 자원 고갈로부터 호스트를 보호**할 수 있어." 정규가 말했다.


"nftables에서 관제실 네트워크만 허용하고 있으니까 SSH는 막혀 있어." 가람이 말했다.

### Volume 암호화

추박하가 끄덕였다. "원래 암호화를 적용하려고 했는데, 2006년에 genesis를 처음 만들 때는 **dm-crypt가 지금처럼 안정적이지 않았어.** 지금은 적용할 수 있어."

```bash
# genesis 데이터 Volume 암호화 — LUKS 설정
# 1단계: genesis 컨테이너 일시 정지
docker pause genesis_core

# 2단계: 데이터 백업
cp -a /opt/genesis/data /opt/genesis/data.backup

# 3단계: LUKS 암호화 볼륨 생성
dd if=/dev/zero of=/opt/genesis/genesis_vault.img bs=1M count=4096
sudo cryptsetup luksFormat /opt/genesis/genesis_vault.img \
  --key-file /root/.genesis_key --batch-mode
sudo cryptsetup luksOpen /opt/genesis/genesis_vault.img genesis_vault \
  --key-file /root/.genesis_key
sudo mkfs.ext4 /dev/mapper/genesis_vault

# 4단계: 암호화 볼륨에 데이터 복사
sudo mount /dev/mapper/genesis_vault /opt/genesis/data_encrypted
sudo cp -a /opt/genesis/data.backup/* /opt/genesis/data_encrypted/
sudo umount /opt/genesis/data_encrypted
```

"이제 genesis의 데이터가 **LUKS 암호화 볼륨** 안에 있어." 추박하가 말했다. "키 파일 없이는 **호스트에서도 읽을 수 없어.**"

```bash
# docker-compose.yml 수정 — 암호화 볼륨으로 마운트 변경
# (실제로는 systemd 서비스에서 luksOpen 후 마운트)

# genesis 컨테이너 재시작
docker unpause genesis_core

# 무결성 확인 — OpsClaw로 evidence 카운트 검증
curl -s -H "X-API-Key: $OPSCLAW_API_KEY" \
  http://localhost:8000/projects/genesis-audit/dispatch \
  -X POST -H "Content-Type: application/json" \
  -d '{
    "command": "docker exec genesis_core python3 -c \"import sqlite3; conn=sqlite3.connect(\\\"/data/genesis.db\\\"); print(conn.execute(\\\"SELECT COUNT(*) FROM evidence\\\").fetchone()[0])\"",
    "subagent_url": "http://localhost:8002"
  }'
```

```json
{
  "status": "success",
  "output": "40132",
  "exit_code": 0
}
```

"**40,132건. 데이터 무결성 확인.**" 김영훈이 말했다. "암호화 전후로 레코드 수가 동일해."

가람이 안도의 한숨을 내쉬었다.
