# Chapter 7: 첫 번째 경보

---

목요일 오후 3시 22분. 윤정규의 모니터에 빨간 알림이 떴다.

```
[Level 12] ALERT — Multiple SSH authentication failures from 45.33.32.156
  Rule: 5710 (SSH brute force)
  Count: 23 attempts in 5 minutes
  Target: web (10.20.30.80)
  Source: 45.33.32.156
```

정규의 눈이 화면에 고정되었다. level 12. 최고 경보 수준.

하지만 정규는 즉시 반응하지 않았다. 그는 먼저 **다른 것**을 확인했다.

---

"5분에 23회면... 브루트포스 맞아." 정규가 혼잣말을 했다.

하지만 지난주의 교훈이 있었다. 203.0.113.50의 SSH 브루트포스 — 847회 시도 후 1회 성공, 사용자 생성. 그때는 **브루트포스 자체보다 그 뒤의 행동**이 문제였다.

정규는 Suricata fast.log를 먼저 확인했다.

```bash
sshpass -p1 ssh -o StrictHostKeyChecking=no secu@10.20.30.1 \
  "echo 1 | sudo -S grep '45.33.32.156' /var/log/suricata/fast.log 2>/dev/null | tail -10"
```

```
03/28/2026-15:18:33  [**] [1:2200015:2] ET SCAN SSH Brute Force [**]
    {TCP} 45.33.32.156:49821 -> 10.20.30.80:22
03/28/2026-15:19:47  [**] [1:2200015:2] ET SCAN SSH Brute Force [**]
    {TCP} 45.33.32.156:49822 -> 10.20.30.80:22
03/28/2026-15:21:03  [**] [1:2200054:3] ET SCAN Potential SSH Scan OUTBOUND [**]
    {TCP} 10.20.30.80:22 -> 45.33.32.156:49823
```

마지막 줄. 정규의 눈이 좁아졌다.

`OUTBOUND`. web 서버(10.20.30.80)에서 공격자(45.33.32.156)로 **나가는** 트래픽.

이건 정상적인 SSH 응답이 아니다. SSH 서버가 클라이언트에게 패킷을 보내는 것은 정상이지만, Suricata가 이것을 "Potential SSH Scan **OUTBOUND**"로 분류했다는 것은...

"리버스 셸?" 정규가 중얼거렸다.

---

정규가 자리에서 일어섰다.

관제실이 조용해졌다. 도윤이 등을 곧게 세웠다. 가람이 수첩을 꺼냈다.

"과장님." 정규가 김영훈을 불렀다. "web 서버에 아웃바운드 의심 트래픽이 있습니다."

김영훈이 다가왔다. 화면을 보더니 표정이 단단해졌다.

"시간을 봐. 15:21. 지금으로부터 1분 전이야." 김영훈이 말했다. "**지금 진행 중**이야."

---

김영훈이 지시를 내렸다. 빠르고 정확하게.

"민재, OpsClaw 인시던트 프로젝트 생성. 즉시."

```bash
PID=$(curl -s -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"name":"IR-20260328-ssh-outbound","request_text":"web 서버 아웃바운드 의심 트래픽","master_mode":"external"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['project']['id'])")
```

"정규, web 서버의 현재 네트워크 연결 상태 확인."

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"ss -tnp | grep ESTAB | grep -v \"127.0.0.1\"","subagent_url":"http://10.20.30.80:8002"}'
```

결과가 돌아왔다.

```
ESTAB  0  0  10.20.30.80:22    45.33.32.156:49823   users:(("sshd",pid=78234,fd=4))
ESTAB  0  0  10.20.30.80:3000  10.20.30.201:52441   users:(("node",pid=12345,fd=11))
ESTAB  0  0  10.20.30.80:22    10.20.30.201:38721   users:(("sshd",pid=8002,fd=3))
```

**첫 번째 줄.** `45.33.32.156:49823`에서 web 서버의 SSH(22)로 연결이 **수립**(ESTABLISHED)되어 있었다.

"SSH 연결이 살아 있어." 정규가 말했다.

"pid 78234." 김영훈이 말했다. "그 프로세스가 뭘 하고 있는지 확인해."

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"ps aux | grep 78234 | grep -v grep; ls -la /proc/78234/fd/ 2>/dev/null | head -10","subagent_url":"http://10.20.30.80:8002"}'
```

```
web      78234  0.0  0.1  12340  2456 ?  S    15:21   0:00 sshd: web [priv]
```

"sshd 세션이야. web 사용자로 로그인되어 있어." 정규가 보고했다.

"web 사용자의 비밀번호가 '1'이잖아." 김영훈의 목소리가 차가워졌다. "23번 시도했는데, 그중에 '1'을 **입력**한 게 있었던 거야."

---

"진호, 45.33.32.156 OSINT."

송진호가 즉시 동작했다.

```bash
# VirusTotal IP 조회
curl -s "https://www.virustotal.com/api/v3/ip_addresses/45.33.32.156" \
  -H "x-apikey: $VT_API_KEY" 2>/dev/null | python3 -c "
import sys,json
d = json.load(sys.stdin)
stats = d.get('data',{}).get('attributes',{}).get('last_analysis_stats',{})
print(f'VirusTotal: malicious={stats.get(\"malicious\",0)} suspicious={stats.get(\"suspicious\",0)}')
print(f'AS: {d.get(\"data\",{}).get(\"attributes\",{}).get(\"as_owner\",\"unknown\")}')
print(f'Country: {d.get(\"data\",{}).get(\"attributes\",{}).get(\"country\",\"unknown\")}')

```

진호가 결과를 보고했다. "45.33.32.156 — Linode 서버. 미국. VirusTotal에서 malicious 3건 플래그. 알려진 스캐닝 IP야."

"Shodan은?"

"포트 22, 80, 443 오픈. 리눅스 서버로 보여. **여러 건의 abuse report**가 있어."

---

"결정." 김영훈이 말했다.

"첫째, 45.33.32.156 즉시 차단."

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"echo 1 | sudo -S nft add rule inet filter input ip saddr 45.33.32.156 drop && echo 1 | sudo -S nft add rule inet filter forward ip saddr 45.33.32.156 drop","subagent_url":"http://10.20.30.1:8002"}'
```

"둘째, web 서버에서 해당 SSH 세션 강제 종료."

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"echo 1 | sudo -S kill -9 78234","subagent_url":"http://10.20.30.80:8002"}'
```

"셋째, web 서버 비밀번호 즉시 변경."

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"echo 1 | sudo -S sh -c \"echo web:$(openssl rand -base64 16) | chpasswd\"","subagent_url":"http://10.20.30.80:8002"}'
```

"넷째, 지후. web 서버 메모리 덤프 확보."

박지후가 고개를 끄덕이며 자기 터미널을 열었다.

---

5분 후. 상황이 정리되었다.

```
[인시던트 요약]
시각: 15:18~15:22
공격자: 45.33.32.156 (Linode, US, abuse 다수)
공격: SSH 브루트포스 23회 → 비밀번호 '1' 성공 → SSH 세션 수립
대응: IP 차단 + 세션 종료 + 비밀번호 변경 + 메모리 덤프
TTD (탐지 시간): ~4분 (15:18 공격 시작 → 15:22 탐지)
TTR (대응 시간): ~3분 (15:22 탐지 → 15:25 차단 완료)
```

김영훈이 팀을 바라보았다.

"TTD 4분, TTR 3분. 나쁘지 않아. 하지만 **지난번보다 나아진 건 뭐야**?"

정규가 대답했다. "지난번은 공격 후 **8시간**이 지나서야 탐지했습니다. 이번엔 Wazuh 경보와 Suricata 로그를 **동시에** 확인해서 아웃바운드 트래픽을 잡았습니다."

"좋아. 그리고 이번에 잘된 게 하나 더 있어." 김영훈이 OpsClaw 화면을 가리켰다. "모든 대응 명령이 **evidence로 기록**됐어. 지난번에는 직접 SSH로 차단하고 기록이 없었지만, 이번에는 replay로 전체 타임라인을 재구성할 수 있어."

```bash
curl -s "http://localhost:8000/projects/$PID/replay" \
  -H "X-API-Key: opsclaw-api-key-2026"
```

```
steps_total: 5
total_reward: 6.5
timeline:
  [1] 네트워크 연결 확인     exit=0  reward=+1.3
  [2] 프로세스 확인          exit=0  reward=+1.3
  [3] IP 차단 (secu)        exit=0  reward=+1.3

  [5] 비밀번호 변경          exit=0  reward=+1.3
```

"이게 **인시던트 대응의 증적**이야." 김영훈이 말했다. "감사관이 '언제 뭘 했느냐'고 물으면, 이 replay 하나면 끝이야."

---

가람은 전체 과정을 수첩에 기록했다.

```
인시던트 대응 흐름:
1. 탐지 (Wazuh level 12 + Suricata OUTBOUND)
2. 분석 (ss -tnp 연결 확인, ps 프로세스 확인, OSINT)
3. 격리 (nftables IP 차단, SSH 세션 종료)
4. 근절 (비밀번호 변경, 메모리 덤프)
5. 기록 (OpsClaw evidence + replay)
```

도윤이 옆에서 한숨을 쉬었다.

"이번에는 내가 야간이 아니라서 다행이다..."

가람은 도윤의 어깨를 가볍게 두드렸다. 도윤이 놀라 바라보았다. 가람은 아무 말도 하지 않고 미소만 지었다.

---

그날 저녁, 가람은 퇴근하며 관제실을 마지막으로 돌아보았다.

추박하의 자리는 비어 있었다. 오후에 사라진 후 돌아오지 않았다.

하지만 그의 모니터에는 한 줄이 남아 있었다.

