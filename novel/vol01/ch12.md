# Chapter 12: 고요한 밤

---

금요일 밤 11시. 오가람의 첫 야간 교대 근무.

도윤이 인수인계를 마치고 "잘 부탁해"라며 나갔다. 관제실에 가람 혼자 남았다.

대형 모니터가 어둠 속에서 빛나고 있었다. Wazuh Dashboard의 시간당 경보 카운트: 38건. 녹색 숫자가 점멸했다.

가람은 도윤이 알려준 야간 근무 루틴을 따랐다.

1. 경보 레벨 12 이상 확인 → 없음
2. 경보 레벨 7~11 분류 → 15건 (대부분 SSH 관련, 이미 차단된 IP)
3. Suricata fast.log 확인 → LLDP 알림 반복 (정상)
4. 방화벽 상태 확인 → 정상

**정상적인 밤이었다.**

가람은 커피를 마시며 인수인계 문서를 작성하기 시작했다.

---

자정이 지났다. 새벽 1시.

가람은 졸기 시작했다. 하지만 의식적으로 깨어 있으려 했다. 도윤이 놓친 SSH 브루트포스의 교훈을 잊지 않았다.

새벽 1시 23분. Wazuh에 새로운 경보가 떴다.

```
[Level 5] File integrity monitoring: /etc/crontab modified
  Agent: (000) siem
  Time: 01:23:07
```

Level 5. 보통이라면 무시할 수준이다. 하지만 가람은 무언가가 걸렸다.

*/etc/crontab이 수정됐다?*

crontab은 리눅스에서 자동 반복 작업을 설정하는 파일이다. 서버 관리자가 백업이나 로그 정리를 위해 수정하는 것은 정상이다. 하지만 **새벽 1시 23분**에?

가람은 상세 내용을 확인했다.

```bash
sshpass -p1 ssh -o StrictHostKeyChecking=no siem@10.20.30.100 \
  "echo 1 | sudo -S cat /var/ossec/logs/alerts/alerts.json 2>/dev/null | \
   python3 -c \"
import sys, json
for line in sys.stdin:
    try:
        a = json.loads(line)
        if 'syscheck' in str(a.get('rule',{}).get('groups',[])):
            if '/etc/crontab' in str(a):
                print(json.dumps(a, indent=2, ensure_ascii=False))
    except: pass
\" 2>/dev/null | tail -30"
```

결과:

```json
{
  "rule": {
    "id": "550",
    "level": 5,
    "description": "File integrity checksum changed.",
    "groups": ["syscheck", "pci_dss_11.5"]
  },
  "syscheck": {
    "path": "/etc/crontab",
    "md5_before": "a1b2c3d4e5f6...",
    "md5_after": "f6e5d4c3b2a1...",
    "changed_attributes": ["md5", "mtime"],
    "event": "modified"
  },
  "agent": {
    "id": "000",
    "name": "siem"
  }
}
```

해시가 바뀌었다. `/etc/crontab`의 내용이 **실제로 수정**된 것이다.

---

가람은 고민했다.

*Level 5야. 긴급은 아니야. 하지만 새벽에 crontab이 바뀌는 건... 정상일까?*

*정규 선배라면 어떻게 할까?*

정규의 말이 떠올랐다. *"느낌상 이상하다"를 설명 못 해서 종종 무시당한다.*

*하지만 정규 선배는 **느낌이 맞을 때가 많다.***

가람은 실제 crontab 내용을 확인하기로 했다.

```bash
sshpass -p1 ssh -o StrictHostKeyChecking=no siem@10.20.30.100 \
  "echo 1 | sudo -S cat /etc/crontab 2>/dev/null"
```

```
# /etc/crontab: system-wide crontab
SHELL=/bin/sh
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin

# m h dom mon dow user  command
17 *  * * *  root    cd / && run-parts --report /etc/cron.hourly
25 6  * * *  root    test -x /usr/sbin/anacron || ( cd / && run-parts --report /etc/cron.daily )
47 6  * * 7  root    test -x /usr/sbin/anacron || ( cd / && run-parts --report /etc/cron.weekly )
52 6  1 * *  root    test -x /usr/sbin/anacron || ( cd / && run-parts --report /etc/cron.monthly )
*/30 * * * * root    curl -s http://10.20.30.201:8002/health > /dev/null 2>&1
```

마지막 줄이 추가된 것이었다.

```
*/30 * * * * root    curl -s http://10.20.30.201:8002/health > /dev/null 2>&1
```

30분마다 opsclaw 서버의 SubAgent 헬스 체크를 curl로 호출. 출력은 /dev/null로 버림.

*이건... 정상일 수도 있어. SubAgent 상태를 모니터링하는 크론잡.*

하지만 가람은 의문을 가졌다.

*왜 siem 서버에서 opsclaw의 헬스를 체크하지? 보통 그 반대 아닌가? 중앙 서버(opsclaw)가 에이전트(siem)를 모니터링하는 게 정상이잖아.*

*그리고 **새벽 1시 23분에** 추가됐다면... 누가?*

가람은 `last` 명령으로 최근 로그인 기록을 확인했다.

```bash
sshpass -p1 ssh -o StrictHostKeyChecking=no siem@10.20.30.100 \
  "last -10 2>/dev/null"
```

```
siem     pts/0    10.20.30.201     Fri Mar 28 01:22 - 01:24  (00:02)
siem     pts/0    10.20.30.201     Fri Mar 28 00:15 - 00:16  (00:01)
root     tty1                      Thu Mar 27 09:00   still logged in
```

01:22~01:24. **opsclaw(10.20.30.201)에서** siem 서버에 SSH로 접속하여 2분간 작업한 기록.

*새벽 1시 22분에 opsclaw에서 siem으로 접속한 사람.*

가람은 다음 질문을 떠올렸다.

*이 시간에 opsclaw 서버에 있었던 사람은 누구지?*

---

가람은 opsclaw 서버의 auth.log를 확인했다.

```bash
grep "01:2[0-5].*Accepted" /var/log/auth.log 2>/dev/null
```

```
Mar 28 01:20:15 opsclaw sshd[89012]: Accepted password for opsclaw from 10.20.30.201 port 54321 ssh2
```

*10.20.30.201은 opsclaw 서버 자체의 IP다. 로컬에서 자기 자신에게 SSH?*

아니. 이건 **콘솔에서 직접 로그인**한 것이 아니라, **다른 네트워크 인터페이스에서** 접속한 것일 수 있다.

가람은 더 깊이 파고 싶었지만, 멈추었다. 이것은 자신의 권한 범위를 넘는 조사였다.

대신, 인수인계 문서에 적었다.

```
[야간 특이사항]
1. 01:23 — siem 서버 /etc/crontab 수정 (FIM 탐지, level 5)
   - 추가된 내용: */30 cron, opsclaw:8002 헬스체크
   - siem에 01:22~01:24 SSH 접속 기록 (출발지: 10.20.30.201)
   - 조치: 주간 조에 확인 요청
```

이번에는 삭제하지 않았다.

---

새벽 5시. 가람은 모니터를 바라보며 생각했다.

이 회사에 입사한 지 2주.

그 사이에 알게 된 것:
- SSH 비밀번호가 '1'이라서 브루트포스에 뚫렸다
- sudo NOPASSWD:ALL로 root 동일 권한이 있었다
- 패스워드 만료가 없었다 (99999일)
- 경쟁사 내부 보고서가 퇴직자 노트북에서 발견됐다
- 공격자는 사용자명을 알고 있었다 (내부 정보 유출)
- 추팀장은 genesis 프로젝트에 매일 무언가를 기록한다
- 새벽에 누군가가 siem 서버의 crontab을 수정했다

그리고 알지 못하는 것:
- genesis 프로젝트에 무엇이 기록되어 있는지
- 추팀장이 왜 새벽에 오는지
- 내부 정보 유출자가 누구인지
- 경쟁사 보고서가 어떻게 넥서스AI에 왔는지
- Nexus-1 학습 데이터의 진짜 출처

가람은 수첩의 마지막 페이지에 적었다.

> `이 회사에는 비밀이 너무 많다.`
> `그리고 그 비밀을 가장 많이 아는 사람은 — 아마도 — 추박하 팀장이다.`
> `OpsClaw는 모든 것을 기록한다. 그렇다면, 이 비밀들도 어딘가에 기록되어 있을 것이다.`

가람은 창밖을 바라보았다. 하늘이 조금씩 밝아지고 있었다.

첫 야간 근무가 끝났다. 새벽이 오고 있었다.

하지만 이 회사의 밤은... 아직 끝나지 않은 것 같았다.
