# Chapter 2: 경비원과 CCTV

---

윤정규는 오전 9시에 출근해서 오전 9시 1분에 이미 로그 속에 빠져 있었다.

그의 하루는 항상 같은 루틴으로 시작했다. Wazuh Dashboard를 열고, 지난 8시간(야간 교대 시간)의 경보를 레벨별로 정렬한다. level 12 이상이 있으면 즉시 확인. level 7~11은 순서대로 분류. level 6 이하는 통계만 확인.

오늘은 level 12가 없었다. 정규는 그 사실에 안도하지 않았다. 그에게 "경보가 없다"는 것은 "아무 일도 없다"가 아니라, "아직 발견하지 못한 것이 있을 수 있다"를 의미했다.

```
이벤트 (수만~수백만 건/일)
  → 경보 (수십~수백 건/일)
    → 인시던트 (0~수 건/일)
```

정규는 이 피라미드를 매일 체감했다. 오늘 Wazuh에 쌓인 이벤트는 약 48만 건. 그중 경보로 올라온 것은 347건. 그리고 그중 **진짜**는 아마 0~2건일 것이다.

문제는 그 0~2건을 347건 속에서 찾아야 한다는 것이었다.

---

"정규야, 오늘 뭐 있어?"

최민재가 커피를 들고 다가왔다. 정규는 모니터에서 눈을 떼지 않았다.

"SSH 실패 로그가 좀 많아."

"얼마나?"

정규의 손이 터미널을 열었다.

```bash
# SSH 인증 실패 이벤트 카운트
sshpass -p1 ssh -o StrictHostKeyChecking=no siem@10.20.30.100 \
  "echo 1 | sudo -S cat /var/ossec/logs/alerts/alerts.json 2>/dev/null | \
   python3 -c \"
import sys, json
from collections import Counter
ips = Counter()
for line in sys.stdin:
    try:
        a = json.loads(line)
        if 'sshd' in a.get('rule',{}).get('groups',[]) or \
           'authentication_failed' in str(a.get('rule',{}).get('groups',[])):
            src = a.get('data',{}).get('srcip', a.get('srcip','unknown'))
            ips[src] += 1
    except: pass
for ip, cnt in ips.most_common(10):
    print(f'  {ip}: {cnt}회')
\""
```

화면에 결과가 나타났다.

```
  203.0.113.50: 847회
  198.51.100.22: 234회
  10.20.30.201: 12회
  10.20.30.80: 3회
```

민재가 화면을 보고 휘파람을 불었다. "847회? 브루트포스네."

"아마." 정규가 담담하게 말했다. "203.0.113.50. 어제 밤부터 계속 때리고 있어."

"차단했어?"

"아직." 정규가 잠시 멈추었다. "근데 이상한 게 있어."

"뭐?"

"847번 실패했는데... **한 번도 성공하지 못한** 게 아니야." 정규의 눈이 화면의 한 줄에 고정되었다. "03:42에 `Accepted password`가 한 번 있어. 그리고 03:43에 **새로운 사용자가 추가**됐어."

민재의 표정이 변했다. "뭐?"

정규는 다음 화면을 열었다.

```
[Level 12] User added: guest03 - by root - 03:43:12
[Level 10] File integrity changed: /etc/passwd - 03:43:12
```

"guest03이라는 사용자가 만들어졌어. 우리가 만든 게 아니야."

---

관제실의 분위기가 바뀌었다. 그 변화는 윤정규의 목소리 톤에서 시작되었다. 정규가 자리에서 일어났을 때, 한도윤이 본능적으로 등이 곧아졌다.

*정규 선배가 일어섰다.*

김영훈이 모니터에서 고개를 들었다.

"무슨 일이야?"

"web 서버에 비인가 사용자가 생성됐습니다. 새벽 3시 43분." 정규의 목소리는 평소보다 반 톤 높았다. "SSH 브루트포스 847회 시도 후, 1회 성공. 이후 root 권한으로 사용자 추가."

김영훈이 일어섰다. "web 서버? 10.20.30.80?"

"네."

"sudo 설정 확인해."

정규의 손이 움직였다.

```bash
sshpass -p1 ssh -o StrictHostKeyChecking=no web@10.20.30.80 "sudo -l 2>/dev/null"
```

결과:

```
User web may run the following commands on web:
    (ALL : ALL) ALL
    (ALL) NOPASSWD: ALL
```

김영훈의 입꼬리가 씰룩거렸다. 그건 웃음이 아니었다.

"NOPASSWD ALL. 패스워드 없이 root 명령을 실행할 수 있어." 그가 낮은 목소리로 말했다. "SSH 비밀번호 하나만 뚫으면, 사실상 **root를 얻는 거야**."

도윤이 조심스럽게 물었다. "그럼... 공격자가 root 권한을 가진 건가요?"

"가진 정도가 아니야." 김영훈이 고개를 저었다. "이미 **사용**한 거지."

---

이때 관제실 안쪽에서 움직임이 있었다. 추박하가 — 만화책을 내려놓고 — 의자를 돌렸다.

"영훈아."

김영훈이 돌아보았다. "팀장님."

"web 서버에 그 guest03... 언제 만들어졌다고?"

"새벽 3시 43분이요."

추박하는 잠시 생각하더니 말했다.

"03:43이면, 야간 교대는 도윤이 했겠네."

한도윤이 움찔했다. "네, 저였습니다. 하지만 그 시간에는 특별한 경보가 없었—"

"그럼 Suricata는?" 추박하가 물었다. "fast.log에 뭐 있어?"

정규가 빠르게 입력했다.

```bash
sshpass -p1 ssh -o StrictHostKeyChecking=no secu@10.20.30.1 \
  "echo 1 | sudo -S grep '03:4[0-5]' /var/log/suricata/fast.log 2>/dev/null | head -5"
```

결과가 화면에 나타났다.

```
03/27/2026-03:42:15  [**] [1:2200015:2] ET SCAN SSH Brute Force [**]
    {TCP} 203.0.113.50:54321 -> 10.20.30.80:22
03/27/2026-03:42:47  [**] [1:2200015:2] ET SCAN SSH Brute Force [**]
    {TCP} 203.0.113.50:54322 -> 10.20.30.80:22
```

"Suricata는 탐지했네." 추박하가 말했다. "ET SCAN SSH Brute Force. 근데 **alert만** 했고, **drop은 안 했어**."

"NFQUEUE 모드인데 IDS로 동작하고 있었던 건가요?" 정규가 물었다.

추박하가 고개를 끄덕였다. "그래. 지금 secu의 Suricata는 alert 모드야. 탐지는 하지만 차단은 안 해. **경비원이 도둑을 보고 기록만 하고 잡지 않은** 격이지."

가람은 수첩에 적었다.

> `IDS = 탐지만 (경비원이 본다) / IPS = 탐지+차단 (경비원이 잡는다)`
> `Suricata가 alert 모드 → 공격을 봤지만 막지 못했다`

---

김영훈이 지시를 내리기 시작했다.

"정규, web 서버의 auth.log 전체 추출해. 03:00~04:00 범위."

"지후, web 서버에서 guest03 계정의 **생성 이후 활동** 전부 추적해. 메모리 덤프 가능하면 확보."

"진호, 203.0.113.50에 대한 OSINT 돌려. WHOIS, VirusTotal, AbuseIPDB, Shodan."

"민재, OpsClaw로 이 대응을 프로젝트로 만들어. 모든 명령을 **evidence로 기록**."

"도윤, 네가 야간에 놓친 경보 목록 전부 정리해."

도윤의 얼굴이 하얗게 변했다. 가람은 도윤이 손을 꽉 쥐는 것을 보았다.

---

최민재가 터미널을 열었다.

```bash
# OpsClaw 프로젝트 생성 — 인시던트 대응
PID=$(curl -s -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "name": "IR-20260327-ssh-bruteforce",
    "request_text": "web 서버 SSH 브루트포스 침해, guest03 비인가 사용자 생성",
    "master_mode": "external"
  }' | python3 -c "import sys,json; print(json.load(sys.stdin)['project']['id'])")

echo "인시던트 프로젝트: $PID"

# Stage 전환
curl -s -X POST "http://localhost:8000/projects/$PID/plan" \
  -H "X-API-Key: opsclaw-api-key-2026" > /dev/null
curl -s -X POST "http://localhost:8000/projects/$PID/execute" \
  -H "X-API-Key: opsclaw-api-key-2026" > /dev/null
```

"프로젝트 생성 완료." 민재가 말했다. "이제부터 모든 대응 명령은 이 프로젝트 아래 기록돼."

김영훈이 고개를 끄덕였다. "긴급 차단부터."

```bash
# 긴급 대응: 공격자 IP 차단 (nftables)
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "command": "echo 1 | sudo -S nft add rule inet filter input ip saddr 203.0.113.50 drop",
    "subagent_url": "http://10.20.30.1:8002"
  }'
```

"secu에서 203.0.113.50 차단 완료." 민재가 보고했다.

"web 서버에서 guest03 계정 비활성화."

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "command": "echo 1 | sudo -S usermod -L guest03 && echo 1 | sudo -S passwd -l guest03",
    "subagent_url": "http://10.20.30.80:8002"
  }'
```

가람은 화면을 바라보았다. 명령 하나하나가 OpsClaw에 기록되고 있었다. evidence로. PoW 블록으로.

민재가 옆에서 말했다. "이게 **증적의 힘**이야. 지금 우리가 하는 모든 대응이 자동으로 기록돼. 나중에 '언제 차단했느냐, 왜 그때 차단했느냐' 물으면, replay 한 번이면 끝이야."

---

관제실의 맨 안쪽에서, 추박하가 조용히 모니터를 보고 있었다.

화면에는 OpsClaw의 replay 타임라인이 실시간으로 업데이트되고 있었다. 각 step에 task_order, exit_code, reward가 표시되어 있었다.

그는 마우스를 움직여 PoW 체인을 확인했다.

```
Blocks: 47,839  (+7 since morning)
Valid: true
Tampered: 0
```

47,839. 오늘 아침의 47,832에서 7개가 추가되었다.

추박하는 화면의 특정 부분을 클릭했다. `agent_id` 필드. 대부분의 블록은 `http://localhost:8002`나 `http://10.20.30.80:8002`였다.

하지만 맨 아래, 가장 오래된 블록들에는 다른 agent_id가 있었다.

`internal-ops`

추박하는 그 ID를 잠시 바라보다가, 조용히 터미널 창을 닫았다.

만화책을 다시 집어들었다.
