## 강의 연계: C7W09(최종) — OpsClaw

2027년 4월. 서울 코엑스 컨퍼런스홀.

**SecOps Korea 2027 — 보안 운영 자동화 컨퍼런스**

발표 프로그램 네 번째 세션.

```
[15:00-15:40]
"OpsClaw v2: 오픈소스 보안 운영 플랫폼의 탄생"
발표자: 김영훈 (넥서스시큐리티 SOC팀장)
```

김영훈이 무대에 올랐다. 500명의 보안 전문가가 자리를 채우고 있었다. 김영훈은 발표가 처음이 아니었지만, 이 규모는 처음이었다.

"안녕하세요. 넥서스시큐리티 SOC팀장 김영훈입니다."

슬라이드가 바뀌었다.

**OpsClaw v2 — 기록은 진실이다.**

---

"OpsClaw를 소개하기 전에, 왜 이걸 만들었는지부터 말씀드리겠습니다."

김영훈이 잠시 멈추었다.

"작년, 저희 회사에서 큰 사건이 있었습니다. 여러분 다 아시는 그 사건입니다. 고객 데이터 유출 은폐. CTO 구속. 과징금 1,200억."

객석이 조용해졌다.

"그 사건에서 회사를 구한 것은 보안 전문가의 직감도, 화려한 기술도 아니었습니다. 기록이었습니다. SIEM이 남긴 로그, OpsClaw가 생성한 변조 불가능한 증거 체인—이것들이 법정에서 진실을 증명했습니다."

"그래서 저희는 결정했습니다. 이 시스템을 오픈소스로 공개하자. 다른 조직도, 같은 도구로, 같은 투명성을 가질 수 있도록."

---

"OpsClaw는 세 계층으로 구성됩니다."

"첫째, Master. AI가 계획을 수립합니다. 어떤 서버에서, 어떤 명령을, 어떤 순서로 실행할지."

"둘째, Manager. 중앙 관제 시스템입니다. 모든 실행을 기록하고, 증거 체인을 생성하고, 상태를 관리합니다."

"셋째, SubAgent. 실제 명령을 실행하는 에이전트입니다. 각 서버에 배포됩니다."

---

가람이 객석에서 듣고 있었다. 옆에는 정규와 조은별이 있었다.

김영훈이 핵심 기능을 설명했다.

"OpsClaw의 핵심 기능 네 가지를 설명하겠습니다."

```
[OpsClaw v2 핵심 기능]

1. Proof of Work (PoW) 증거 체인
   - 모든 작업에 SHA-256 해시 생성
   - 이전 해시와 연결 → 블록체인 구조
   - 변조 시 체인 전체가 깨짐 → 즉시 탐지

2. 강화학습 (RL) 기반 정책 최적화
   - task_reward 데이터로 Q-learning
   - 최적 risk_level 자동 추천
   - 경험에서 학습하는 운영 시스템

3. 분산 지식 아키텍처
   - 각 SubAgent가 독립적 지식 베이스 보유
   - 서버별 특성에 맞는 자율 판단
   - Manager가 전체를 조율

4. 오픈소스 투명성
   - 소스코드 전체 공개 (GitHub)
   - 누구나 감사(audit) 가능
   - 커뮤니티 기여 환영
```

"특히 PoW 증거 체인이 법정에서 결정적이었습니다."

김영훈이 화면을 전환했다.

```bash
# 증거 체인 검증 — 한 줄이면 끝
curl http://localhost:8000/pow/verify?agent_id=http://localhost:8002

# 결과
{
  "valid": true,
  "blocks": 1247,
  "orphans": 0,
  "tampered": []
}
```

"1,247개의 작업 기록이 모두 유효하고, 변조된 것은 0건입니다. 이 한 줄의 응답이, 법정에서 검사와 판사를 설득했습니다."

객석에서 박수가 터졌다.

---

Q&A 시간.

질문자가 물었다.

"PoW 체인이 내부 관리자에 의해 조작될 수 있지 않습니까?"

김영훈이 답했다.

"좋은 질문입니다. 단일 관리자가 체인을 조작하려면, 변조한 블록 이후의 모든 블록을 재계산해야 합니다. 블록이 1,000개라면 1,000개의 해시를 모두 다시 만들어야 합니다. 그리고 그 작업 자체가 Wazuh SIEM에 로그로 남습니다."

"완벽한 시스템은 없습니다. 하지만 '조작 비용'을 극단적으로 높이는 것—그것이 우리의 전략입니다."

또 다른 질문.

"오픈소스로 공개하면 취약점이 노출되지 않습니까?"

"보안에서 '모호함에 의한 보안(security through obscurity)'은 이미 실패한 전략입니다. 코드를 공개하면 전 세계의 보안 전문가가 취약점을 찾아줍니다. 숨기는 것보다, 공개하고 함께 고치는 것이 더 안전합니다."

---

발표가 끝난 후, 김영훈이 무대에서 내려왔다.

가람이 다가갔다.

"팀장님, 멋있었어요."

김영훈이 넥타이를 풀며 말했다.

"땀 났어. 500명 앞에서 말하는 거, SOC 운영보다 무서워."

정규가 웃었다.

"팀장님, GitHub 스타가 벌써 200개 넘었어요."

```
[GitHub — opsclaw/opsclaw]
Stars: 217
Forks: 43
Contributors: 8
License: Apache 2.0

README.md 첫 줄:
"기록은 진실이다. — Record is Truth."
```

조은별이 말했다.

"'기록은 진실이다'가 공식 슬로건이 된 거네."

김영훈이 고개를 끄덕였다.

"그래. 이건 가람이가 처음 한 말이야."

가람이 놀랐다.

"제가요?"

"노트북에 적은 거. 작년 검찰에 보고서 제출하던 날. '보고서는 무기다. 정확한 보고서는 진실의 무기다.' 그날 네가 적은 걸 내가 봤거든."

가람이 얼굴이 빨개졌다.

---

컨퍼런스가 끝나고, SOC팀은 코엑스 근처 식당에서 저녁을 먹었다.

김영훈이 맥주를 들며 말했다.

"1년 전에는, 이게 법정 증거가 될 줄 몰랐어. OpsClaw를."

가람이 말했다.

"1년 전에는, 제가 이 자리에 있을 줄도 몰랐어요."

조은별이 웃었다.

"1년 전에는, 주주총회에서 SOC 2를 설명하게 될 줄 몰랐지."

정규가 말했다.

"1년 전에는, 추박하 선배의 룰을 제가 개선하게 될 줄 몰랐어요."

잠시 침묵이 흘렀다. 추박하의 이름이 나오면 항상 그랬다.

김영훈이 잔을 들었다.

"OpsClaw v2. 오픈소스. 기록은 진실이다. 건배."

"건배."
