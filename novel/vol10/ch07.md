# 10권 제7장: Wazuh가 지킨 것

## 강의 연계: C2W09 Wazuh(심화)

---

서울중앙지방법원 제417호 법정. 두 번째 공판.

이번에는 강혜윤이 증인석에 섰다. 검찰이 기술 증거의 신뢰성을 입증하기 위해 소환한 것이었다.

검사가 화면에 로그를 띄웠다.

"증인, 이 로그가 무엇인지 설명해 주십시오."

```
[Wazuh Alert — 2024-06-14 02:17:33]
{
  "rule": {
    "id": "100210",
    "level": 12,
    "description": "Database admin performed bulk DELETE on customer_data table"
  },
  "agent": {
    "id": "003",
    "name": "nexus-db-primary"
  },
  "data": {
    "srcuser": "svc_admin_smg",
    "query": "DELETE FROM customer_data WHERE leak_flag = true",
    "rows_affected": 47293,
    "srcip": "10.20.5.47"
  }
}
```

강혜윤이 차분하게 설명했다.

"이것은 Wazuh SIEM 시스템이 기록한 보안 경고입니다. 2024년 6월 14일 새벽 2시 17분, 'svc_admin_smg'라는 데이터베이스 계정이 customer_data 테이블에서 47,293건의 레코드를 일괄 삭제했습니다."

"이 계정은 누구의 것입니까?"

"'smg'는 서문기 상무의 이니셜입니다. 내부 계정 관리 대장에서 확인했습니다."

변호인이 이의를 제기했다.

"이 로그가 조작되지 않았다는 보장이 있습니까?"

---

강혜윤이 이 질문을 기다리고 있었다.

"Wazuh SIEM의 로그 무결성 보장 메커니즘을 설명드리겠습니다."

```
[Wazuh 로그 무결성 메커니즘]

1. 에이전트 → 서버 통신
   - TLS 1.3 암호화 채널 사용
   - 에이전트 인증: 등록 시 발급된 키로 상호 인증
   - 전송 중 변조 불가

2. 로그 저장
   - /var/ossec/logs/alerts/ 디렉토리에 JSON 형식으로 저장
   - Wazuh Indexer(OpenSearch)에 인덱싱
   - 두 곳에 독립적으로 저장 → 한 쪽만 조작하면 불일치 탐지

3. 파일 무결성 모니터링 (FIM)
   - Wazuh 자체가 로그 파일의 변경을 모니터링
   - syscheck 모듈이 해시값을 주기적으로 검증
   - 로그 파일 자체가 변조되면 FIM이 알림 생성

4. 타임스탬프 검증
   - NTP 동기화된 시스템 시간 사용
   - 에이전트 시간과 서버 시간의 교차 검증
```

"즉, Wazuh 로그를 조작하려면 세 가지를 동시에 해야 합니다. 첫째, 로컬 로그 파일을 수정한다. 둘째, OpenSearch 인덱스를 수정한다. 셋째, FIM이 그 변경을 탐지하지 못하도록 FIM 자체를 무력화한다. 이 세 가지를 동시에, 흔적 없이 수행하는 것은 사실상 불가능합니다."

판사가 물었다.

"사실상 불가능하다는 것은, 이론적으로는 가능하다는 뜻입니까?"

강혜윤이 정직하게 답했다.

"루트 권한을 가진 사람이 충분한 시간과 기술이 있다면, 이론적으로 가능합니다. 그러나 이 사건에서 Wazuh 서버의 루트 접근 로그를 확인한 결과, 해당 시점에 루트 접근은 없었습니다. 또한 OpsClaw와 AWS CloudTrail이라는 두 개의 독립적인 시스템도 동일한 사실을 기록하고 있습니다."

---

검사가 다음 화면을 띄웠다.

"이 알림들을 시간순으로 보여주십시오."

강혜윤이 준비한 타임라인을 펼쳤다.

```
[Wazuh 알림 타임라인 — 서문기 관련]

2024-03-14 23:41 — Rule 100201 (Level 10)
  "Unauthorized access to /admin/config from non-whitelisted IP"
  srcuser: svc_admin_smg, srcip: 10.20.5.47

2024-03-15 00:12 — Rule 100205 (Level 11)
  "Security audit log rotation manually triggered"
  srcuser: svc_admin_smg
  → 보안 감사 로그를 수동으로 순환(삭제) 시도

2024-06-14 02:17 — Rule 100210 (Level 12)
  "Database admin performed bulk DELETE on customer_data table"
  srcuser: svc_admin_smg, rows: 47,293

2024-06-14 02:34 — Rule 100211 (Level 12)
  "Audit trail modification attempt detected"
  srcuser: svc_admin_smg
  → 감사 로그 수정 시도 탐지

2024-09-22 01:08 — Rule 100215 (Level 13)
  "SIEM configuration change: alert suppression rule added"
  srcuser: svc_admin_smg
  → Wazuh 알림 억제 룰 추가 시도
```

강혜윤이 설명했다.

"서문기 상무는 2024년 3월부터 9월까지 체계적으로 증거를 인멸하려 했습니다. 데이터 삭제, 감사 로그 수정 시도, 심지어 SIEM 알림 자체를 억제하려는 시도까지 했습니다."

"하지만 Wazuh는 그 모든 시도를 기록했습니다. 알림을 억제하려는 시도조차 알림으로 기록되었습니다."

방청석에서 탄성이 흘러나왔다.

---

변호인이 물었다.

"이 Wazuh라는 시스템을 관리한 것도 넥서스AI 직원 아닙니까? 내부자가 관리하는 시스템을 어떻게 신뢰할 수 있습니까?"

강혜윤이 답했다.

"좋은 질문입니다. Wazuh 서버는 SOC팀이 관리했고, 서문기 상무는 SOC팀에 대한 직접적 권한이 없었습니다. Wazuh 서버의 접근 로그를 확인한 결과, 서문기 상무의 계정으로 Wazuh 서버에 접근한 기록은 없습니다."

"또한, Wazuh는 에이전트-서버 구조입니다."

```
[Wazuh 아키텍처와 독립성]

에이전트 (각 서버에 설치)
    │
    │ TLS 암호화 채널
    │
    ▼
Wazuh 서버 (SOC팀 관리)
    │
    ├── 로컬 로그 저장
    └── Wazuh Indexer (OpenSearch)

핵심: 에이전트가 수집한 데이터는 서버로 전송되는 즉시
      에이전트 측에서 삭제해도 서버에는 이미 복사본이 존재.
      서버는 SOC팀 관리 → CTO(서문기)의 권한 밖.
```

"에이전트가 데이터를 수집하여 서버로 보내는 순간, 그 데이터의 사본은 서문기 상무의 손이 닿지 않는 곳에 존재합니다. 이것이 SIEM의 구조적 독립성입니다."

---

법정을 나온 후, 강혜윤과 가람이 법원 앞 벤치에 앉았다.

가람이 물었다.

"Wazuh가 없었으면 어떻게 됐을까요?"

강혜윤이 잠시 생각했다.

"증거가 훨씬 약했겠지. 서문기가 DB 레코드를 삭제하고, 감사 로그를 수정했으니까—Wazuh가 아니었으면 '삭제된 사실' 자체를 증명하기 어려웠을 거야."

"SIEM이 회사를 구한 거네요."

"정확히는, SIEM을 운영한 사람들이 회사를 구한 거야. 시스템은 도구일 뿐이야. 그 도구를 제대로 설치하고, 룰을 작성하고, 알림을 확인한 사람들이—"

강혜윤이 말을 멈추었다.

"추박하 선배도 포함이에요?"

가람이 조심스럽게 물었다.

강혜윤이 한참을 침묵했다.

"... 추박하가 Wazuh 초기 설정을 했어. Rule ID 100201부터 100220까지, 그 탐지 룰을 처음 작성한 건 추박하야."

"서문기의 행동을 탐지한 룰을 추박하 선배가 작성했다는 거예요?"

"그래. 아이러니하지."

```
[가람의 노트]
Wazuh — SIEM이 지킨 것

1. 독립적 로그 수집 → 삭제해도 사본이 존재
2. FIM → 로그 변조 시도 자체를 탐지
3. 구조적 독립성 → 관리자가 다르므로 한 사람이 모든 증거를 지울 수 없음

그리고—
탐지 룰을 작성한 것은 추박하 선배였다.
나중에 공범이 된 사람이, 탐지 도구를 만든 사람이기도 했다.
사람은 변하지만, 코드는 변하지 않는다.
추박하 선배가 작성한 룰이, 서문기를 잡았다.

이것이 기록의 힘이다.
```

가람은 노트북을 닫고 하늘을 올려다보았다. 구름 사이로 햇살이 내리쬐고 있었다.

---

*— 10권 7장 끝 —*
