# 10권 제4장: 주주총회

## 강의 연계: C4W13(심화) — SOC 2/HIPAA

---

넥서스AI 본사 대강당.

긴급 주주총회 당일 아침, 건물 앞에는 기자 삼십여 명과 시위대가 모여 있었다. 플래카드에는 "데이터 은폐 책임져라"는 문구가 적혀 있었다.

김영훈은 SOC팀을 대표하여 대강당 뒤편에 앉아 있었다. 옆에는 이주원이 있었고, 반대편에는 법무팀장이 서류를 정리하고 있었다.

"김 팀장, 컴플라이언스 관련 질문이 나오면 내가 받을 테니, 기술적인 질문은 김 팀장이 받아줘."

이주원이 말했다.

김영훈이 고개를 끄덕였다. 양복이 불편했다. 평소에는 후드티를 입는 사람이었다.

---

주주 대표가 마이크를 잡았다.

"첫 번째 안건. 넥서스AI는 현재 SOC 2 Type II 인증을 보유하고 있습니까?"

법무팀장이 답했다.

"2024년에 갱신된 SOC 2 Type II 인증을 보유하고 있습니다."

"그 인증이 유효한 상태에서 데이터 유출이 발생하고 은폐된 것입니까?"

법무팀장이 머뭇거렸다. 이주원이 마이크를 가져왔다.

"말씀드리겠습니다. SOC 2 인증은 보안 통제가 '존재하고 운영되고 있는지'를 검증합니다. 그러나 인증은 경영진이 의도적으로 통제를 우회하는 상황까지 탐지하지는 못합니다."

```
[SOC 2 Type II 인증 개요]

SOC 2(Service Organization Control 2)는 AICPA가 제정한
서비스 조직의 보안 통제 기준입니다.

Trust Service Criteria:
1. Security (보안) — 무단 접근 방지
2. Availability (가용성) — 서비스 지속성
3. Processing Integrity (처리 무결성) — 데이터 정확성
4. Confidentiality (기밀성) — 민감 데이터 보호
5. Privacy (개인정보보호) — 개인정보 수집/사용/보관

Type I: 특정 시점의 통제 설계 평가
Type II: 일정 기간(보통 6-12개월)의 통제 운영 효과성 평가
```

주주가 날카롭게 물었다.

"그러면 SOC 2 인증이 있어도 이런 일이 벌어질 수 있다는 겁니까? 그 인증이 무슨 의미가 있습니까?"

이주원이 답했다.

"인증은 '최소한의 기준'입니다. 자동차에 비유하면 안전벨트와 에어백이 있다는 것이지, 운전자가 음주운전을 안 한다는 보증은 아닙니다. 넥서스AI의 문제는 안전장치가 없었던 것이 아니라, 안전장치를 의도적으로 끄고 운전한 것입니다."

대강당이 웅성거렸다.

---

두 번째 질문이 나왔다.

"넥서스AI는 HIPAA 대상 기관입니까? 의료 데이터가 포함되어 있었습니까?"

이주원이 이 질문을 예상하고 있었다.

"넥서스AI는 직접적인 HIPAA 적용 대상(Covered Entity)은 아닙니다. 그러나 의료 기관에 서비스를 제공하는 Business Associate로서, BAA(Business Associate Agreement)를 체결한 고객이 있었습니다."

```
[HIPAA 관련 넥서스AI 현황]

HIPAA (Health Insurance Portability and Accountability Act):
- Covered Entity: 의료기관, 보험사, 의료정보교환소
- Business Associate: CE에게 서비스를 제공하며 PHI에 접근하는 기관

넥서스AI의 위치:
- 3개 의료기관과 BAA 체결
- PHI(Protected Health Information) 처리 서비스 제공
- BAA 조건: 유출 시 60일 이내 통지 의무

위반 사항:
- PHI 유출 인지 후 통지 미이행
- BAA 계약 위반 → 개별 소송 가능성
```

"즉, HIPAA의 직접 적용은 받지 않지만, 계약상 의무를 위반했습니다. 이로 인해 의료기관 고객들이 별도 소송을 준비하고 있는 것으로 알고 있습니다."

주주들 사이에서 한숨이 흘러나왔다.

---

세 번째 질문. 한 기관투자자가 일어섰다.

"재무적 영향은 어느 정도입니까? 솔직하게 말해 주십시오."

CFO가 답했다.

"현재 파악된 법적 리스크는 다음과 같습니다."

```
[넥서스AI 법적/재무적 리스크 — 요약]

1. 개인정보보호법 과징금: 최대 매출의 3% → 약 450억 원
2. GDPR 과징금: 최대 매출의 4% 또는 2천만 유로 → 약 300억 원
3. 집단 소송 (국내): 약 2만 명 고객 → 예상 배상금 200-500억 원
4. 의료기관 BAA 위반 소송: 3건 → 예상 50-100억 원
5. 주가 하락에 따른 주주 소송: 예측 불가
6. 형사 벌금 (서문기 개인): 별도

합계 추산: 1,000-1,350억 원 (최악의 경우)
```

대강당이 얼어붙었다.

기관투자자가 물었다.

"회사가 이걸 감당할 수 있습니까?"

CFO가 말했다.

"현금성 자산과 보험으로 일부 커버 가능하나, 전액 감당은 어렵습니다. 구조조정이 불가피할 수 있습니다."

---

김영훈에게 질문이 돌아온 것은 오후였다.

"보안 운영팀(SOC) 입장에서, 향후 재발 방지 계획은 무엇입니까?"

김영훈이 마이크를 잡았다. 손이 떨렸지만, 목소리는 떨리지 않았다.

"세 가지입니다."

"첫째, CISO 독립성 보장. CISO는 이사회에 직접 보고하는 구조로 변경합니다. CTO와의 겸임을 금지합니다."

"둘째, OpsClaw 증거 체인의 외부 감사. 내부 시스템의 무결성을 외부 감사인이 정기적으로 검증합니다."

"셋째, SOC 2 Type II 심사를 반기별로 전환합니다. 기존 연 1회에서 반기 1회로 강화하여, 통제 우회가 장기간 지속되는 것을 방지합니다."

```
[재발 방지 계획 — SOC팀 제안]

1. 거버넌스 개선
   - CISO 독립성: 이사회 직속
   - 보안 위원회 신설: 분기별 경영진 + 외부 전문가 합동 심의

2. 기술적 통제 강화
   - OpsClaw evidence_pack 외부 감사
   - SIEM 로그 변조 방지: WORM 스토리지 도입
   - 접근 통제 강화: 최소 권한 원칙 재적용

3. 컴플라이언스 강화
   - SOC 2 Type II 반기별 심사
   - GDPR/개인정보보호법 컴플라이언스 전담 조직 신설
   - 내부 제보 채널(익명) 운영
```

한 주주가 물었다.

"그 계획을 제안한 건 누구입니까?"

"SOC팀입니다. 우리 팀이 이 사건을 처음 탐지했고, 증거를 수집했고, 보고서를 작성했습니다. 재발 방지도 우리가 설계합니다."

김영훈의 목소리에 힘이 실려 있었다.

---

주주총회가 끝난 후, 김영훈은 대강당 밖 복도에서 벽에 기대어 서 있었다.

가람이 다가왔다.

"팀장님, 잘하셨어요."

"잘한 건지 모르겠다. 회사가 살아남을 수 있을지도 모르겠고."

"그래도 우리가 할 수 있는 건 했잖아요."

김영훈이 쓰게 웃었다.

"가람아, SOC 2가 뭔지 아는 사람이 주주총회에서 발표하는 날이 올 줄은 몰랐다."

가람이 노트북을 꺼내 적었다.

```
[가람의 노트]
컴플라이언스는 체크리스트가 아니다.
SOC 2 인증이 있어도, HIPAA 계약이 있어도,
사람이 의도적으로 우회하면 무력화된다.

진짜 컴플라이언스는:
1. 독립적 감시자가 존재하고
2. 기록이 변조 불가능하고
3. 내부 제보가 보호받는 것

인증서가 회사를 지키는 게 아니라,
인증서를 진지하게 받아들이는 문화가 회사를 지킨다.
```

밖에서 기자들의 카메라 셔터 소리가 들렸다. 넥서스AI의 오늘은 끝나지 않았다. 하지만 SOC팀은 여전히 자리를 지키고 있었다.

---

*— 10권 4장 끝 —*
