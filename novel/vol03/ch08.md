# Chapter 8: 공유 계정

---

일요일. 시큐리온 팀이 **네트워크 전체 감사**를 수행했다.

그 과정에서 secu 서버의 이상한 점이 발견됐다.

"secu 서버에 **admin**이라는 사용자가 있어요." 위협 헌팅 전문가가 말했다.

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"grep admin /etc/passwd 2>/dev/null","subagent_url":"http://10.20.30.1:8002"}'
```

```
admin:x:1001:1001:System Admin:/home/admin:/bin/bash
```

"이건 우리가 만든 게 아닌데." 정규가 말했다. "secu의 일반 사용자는 'secu'야."

"'admin'은 누가 만든 거야?"

```bash
# admin 계정 생성 시점 확인
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"echo 1 | sudo -S stat /home/admin 2>/dev/null","subagent_url":"http://10.20.30.1:8002"}'
```

```
  Birth: 2024-06-15 09:22:33
```

"**2024년 6월**에 만들어졌어." 정규가 말했다. "2년 전이야. 이번 공격과는 무관해."

"그럼 뭐야?"

---

정태식 본부장에게 확인했다. 인프라 운영본부장.

정태식이 관제실에 내려왔다. 큰 체격에 거친 말투.

"admin 계정? 아, 그거 **점프서버** 접속용이야. 서버 관리자들이 공용으로 쓰는 계정이지."

"공용이요?" 김영훈이 물었다.

"응. admin 계정 비밀번호를 팀원들끼리 공유해서, 긴급할 때 아무나 들어가서 작업할 수 있게."

김영훈의 표정이 변했다. "비밀번호를... **공유**요?"

"왜? 문제 있어?"

"본부장님." 김영훈이 목소리를 낮추었다. "공유 계정은 **누가 접속했는지 추적이 안 됩니다**. 문제가 생겨도 '누가 한 거야?'를 알 수 없어요."

"그건 관리 편의를 위해서—"

"**ISO 27001 A.8.5 안전한 인증**에 위반됩니다. 각 사용자에게 **개인 계정을 부여**해야 합니다."

정태식의 얼굴이 붉어졌다. "김 과장, 지금 나한테 **보안 강의**를 하는 건가?"

"강의가 아니라 **사실**을 말씀드리는 겁니다."

"보안이 중요한 건 알아. 하지만 현실적으로, 새벽에 서버 장애가 나면 담당자가 안 잡힐 때가 있어. 그때 **누구나** 접속해서 고칠 수 있어야 해."

"그래서 **누구나 접속할 수 있는 계정**이, 공격자에게도 **누구나 접속할 수 있는 문**이 되는 겁니다."

---

추박하가 끼어들었다.

"태식아."

정태식이 추박하를 바라보았다. 두 사람은 오래 알고 지낸 사이였다.

"admin 계정의 비밀번호가 뭐야?"

정태식이 잠시 머뭇거렸다. "... admin123."

관제실에 한숨이 퍼졌다. 민재가 이마를 짚었다.

"admin/admin123." 추박하가 되뇌었다. "2년 동안?"

"바꾸려고 했는데—"

"**'나중에 하겠다'**?" 김영훈이 말했다. 이전에 가람에게 했던 그 말이었다.

---

"이 공유 계정이 공격에 사용되었는지 확인해야 해." 시큐리온 전문가가 말했다.

```bash
# secu 서버에서 admin 계정 로그인 기록
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"grep admin /var/log/auth.log 2>/dev/null | grep Accepted | tail -10","subagent_url":"http://10.20.30.1:8002"}'
```

```
Mar 15 14:22:11 secu sshd: Accepted password for admin from 192.168.10.33
Mar 22 09:15:44 secu sshd: Accepted password for admin from 192.168.10.33
Mar 28 03:45:22 secu sshd: Accepted password for admin from 10.20.30.80
Apr 01 02:33:15 secu sshd: Accepted password for admin from 10.20.30.80
```

"**3월 28일과 4월 1일.** web 서버(10.20.30.80)에서 admin 계정으로 secu에 접속." 정규가 말했다. "이건 공격자야."

"admin/admin123으로 secu까지 접근한 거야." 김영훈이 말했다.

"하지만 auth.log에는 **'admin'으로 접속**이라고만 나와." 시큐리온 전문가가 말했다. "공유 계정이니까 **공격자인지 정상 관리자인지 구분이 안 돼**."

"그게 바로 공유 계정의 문제야." 추박하가 말했다. "**누가 한 건지 모른다.** 증거로서 가치가 없어."

정태식의 얼굴이 창백해졌다.

---

가람은 수첩에 적었다.

> `공유 계정 admin/admin123 — 2년간 변경 없음`
> `공격자가 이 계정으로 secu 서버에 접근 (03/28, 04/01)`
> `공유 계정 → "누가 했는지 모른다" → 증거 가치 없음`
> `정태식 본부장: "관리 편의" vs 김영훈: "보안 위반"`
> `"공유 계정은 누구에게나 열려 있는 문이다."`
