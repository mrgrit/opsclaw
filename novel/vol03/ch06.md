# Chapter 6: 사라진 사람

---

금요일 오전 9시. AI 연구소에서 전화가 왔다.

"이주원 선임이 오늘 출근하지 않았습니다. 연락이 안 됩니다."

김영훈이 전화를 받았다. 표정이 굳었다.

"언제부터?"

"어제도 반차를 냈는데, 오늘은 아예 연락이 안 돼요."

---

30분 후. 진수현 실장(인사실)이 SOC에 내려왔다.

"이주원이 어제 오후부터 연락이 되지 않아요." 진수현이 말했다. "사내 메일, 메신저, 휴대폰 전부."

"퇴근은 했나요?" 김영훈이 물었다.

"어제 14시에 반차 신청 후 나간 것으로 기록되어 있어요."

"출입 기록은?"

진수현이 서류를 확인했다. "14:03 퇴문 기록. 이후 재입장 없음."

---

"집에는 확인했나요?"

"비상 연락망으로 부모님께 연락했는데, '아직 연락이 안 됐다'고 하세요. 원래 연락을 자주 안 하는 편이라 걱정은 아직 안 하고 계시지만..."

추박하가 끼어들었다. "이주원의 **사내 PC 상태**를 확인해."

---

김영훈과 지후가 이주원의 사내 PC(192.168.10.45)에 원격 접속을 시도했다.

```bash
ping -c 1 192.168.10.45
```

```
PING 192.168.10.45: 56 data bytes
Request timeout for icmp_seq 0
```

"PC가 **꺼져 있거나 네트워크에서 분리**되어 있어." 지후가 말했다.

"어제 14시에 퇴근했으면 PC를 꺼고 갔을 수도 있지." 민재가 말했다.

"하지만 넥서스AI는 **원격 접속을 위해 PC를 항상 켜두는 문화**야." 지후가 고개를 저었다. "특히 연구원들은."

---

"비상 절차." 김영훈이 말했다. "이주원의 PC를 물리적으로 확보해야 합니다."

진수현이 고개를 끄덕였다. "AI 연구소 관리자에게 요청할게요. 하지만 서문기 상무의 승인이—"

"서문기 상무는 **제가** 처리하겠습니다." 추박하가 일어섰다. 만화책을 책상에 놓았다.

"팀장님—" 김영훈이 불렀다.

추박하가 돌아보았다. 평소와 다른 눈빛이었다. 날카롭고, 차가웠다.

"영훈아. 이주원이 도망친 거야."

관제실이 고요해졌다.

"도망이요?" 도윤이 물었다.

"VPN 로그, 프롬프트 인젝션 시도, 이메일의 외부 링크. 우리가 조사하고 있다는 걸 **눈치챘을** 가능성이 있어." 추박하가 말했다. "그래서 증거를 지우고 사라진 거야."

"하지만 확실한 건—"

"확실한 건 없어. 그래서 **PC를 확보**해야 해. 거기 남은 데이터가 마지막 증거야."

---

오후 2시. 추박하가 서문기 상무와 30분간 회의한 후, 이주원의 PC 접근 승인을 받아왔다.

김영훈은 물었다. "어떻게 설득하신 겁니까?"

추박하가 답했다. "이주원이 **사라졌다**고 하니까, 서문기도 색이 변하더군. 이주원이 도망친 건 서문기한테도 문제거든. **자기 부하 직원이 산업 스파이**였다면, 상무인 자기 감독 책임도 문제가 되니까."

---

오후 4시. 지후와 시큐리온 포렌식 전문가가 이주원의 PC를 분석하기 시작했다.

PC는 꺼져 있었다. 전원을 넣기 전에 **디스크를 분리하여 이미징**했다. 포렌식의 기본 — 원본을 건드리지 않는다.

```bash
# 디스크 이미지 생성
dd if=/dev/sda of=/evidence/case-juwon/disk.img bs=4M status=progress
sha256sum /evidence/case-juwon/disk.img > /evidence/case-juwon/hash.txt
```

"해시 확보 완료." 지후가 보고했다.

이미지를 마운트한 후, 가장 먼저 확인한 것은 **최근 삭제된 파일**.

```bash
# 삭제된 파일 복구 시도
photorec /evidence/case-juwon/disk.img
```

결과가 나오기까지 시간이 걸렸다. 그 사이 가람은 관제실에서 기다리며 수첩을 펼쳤다.

> `이주원 실종. 4/09 14:03 퇴문 후 연락 두절.`
> `PC 꺼져 있음 (비정상).`
> `추팀장: "도망친 거야" — 증거를 지우고 사라진 것.`
> `서문기도 감독 책임 문제 → PC 접근 승인.`
> `포렌식 진행 중.`

가람은 수첩을 닫고 생각했다.

*이주원은 정말 도망친 걸까? 아니면...*

*누군가에 의해 사라진 건 아닐까?*

---

오후 6시. 지후가 첫 번째 분석 결과를 가져왔다.

"디스크에 **대규모 삭제 흔적**이 있어." 지후의 표정이 심각했다. "sdelete 같은 보안 삭제 도구가 사용됐어. 일반 삭제가 아니라 **덮어쓰기 삭제**. 복구가 불가능한 수준."

"전부?" 김영훈이 물었다.

"Documents, Downloads, Desktop 폴더가 **통째로** 보안 삭제됐어. 하지만—" 지후가 USB 드라이브를 들었다. "**레지스트리**와 **Prefetch**는 남아 있어."

```
[Prefetch 분석 — 최근 실행 프로그램]
SDELETE.EXE-ABCD1234.pf  → 보안 삭제 도구 실행 (04/09 13:45)
CHROME.EXE-12345678.pf   → 브라우저 (04/09 13:30)
MEGA.EXE-87654321.pf     → Mega 클라우드 클라이언트 (04/09 13:35)
PUTTY.EXE-11223344.pf    → PuTTY SSH 클라이언트 (04/09 13:40)
```

"타임라인이 보여." 지후가 정리했다.

```
13:30 — Chrome (뭔가 다운로드/업로드?)
13:35 — Mega (클라우드에 파일 업로드?)
13:40 — PuTTY (어딘가에 SSH 접속?)
13:45 — sdelete (증거 삭제)
14:03 — 퇴문 (퇴근)
```

"퇴근 전 30분 동안 **파일을 외부로 빼돌리고, 증거를 삭제하고, 나간** 거야." 김영훈이 말했다.

"Mega에 뭘 올렸는지 알 수 있어?" 진호가 물었다.

"로컬 파일은 삭제됐지만, **Mega 계정 정보**가 레지스트리에 남아 있을 수 있어." 지후가 레지스트리를 뒤졌다.

```
HKEY_CURRENT_USER\Software\Mega Limited\MEGAsync
  Email: jw.research.2025@gmail.com
```

관제실이 고요해졌다.

**jw.research.2025@gmail.com.**

이주원의 이메일에서 반복 등장했던 그 Gmail 주소. 동시에 **Mega 계정 이메일**.

"이주원 = jw.research.2025." 김영훈이 확인했다. "이주원이 **자신의 외부 계정으로** 파일을 업로드한 거야."

"뭘 올렸는지는 모르지만..." 진호가 말했다.

"**Nexus-1 관련 데이터**일 가능성이 높아." 추박하가 만화책 뒤에서 말했다. "모델 가중치, 학습 데이터, 아키텍처 문서."

김영훈이 주먹을 쥐었다.

"경찰에 신고해야 합니다. 산업기밀 유출이에요."

추박하가 고개를 끄덕였다. "강혜윤 실장에게 먼저 가. 법무 검토 후에 신고 절차를 밟아야 해."

---

가람은 관제실의 공기가 바뀐 것을 느꼈다.

지금까지는 "보안 사고 대응"이었다. 하지만 이제는 **"범죄 수사"**가 시작되려 하고 있었다.

가람은 수첩에 적었다.

> `이주원의 PC: Documents/Downloads 보안 삭제 (sdelete).`
> `퇴근 전 30분: Chrome → Mega 업로드 → PuTTY SSH → sdelete → 퇴문`
> `Mega 계정 = jw.research.2025@gmail.com (이전 이메일의 외부 연락처와 동일!)`
> `이주원이 Nexus-1 데이터를 외부로 빼돌렸을 가능성.`
> `추팀장: "도망친 거야." — 맞았다.`
> `하지만 어디로? 그리고 왜?`

---

*— 3권 6장 끝 —*
