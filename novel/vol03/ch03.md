# Chapter 3: 사다리

---

외부 보안 업체가 도착하기 전, 김영훈은 공격자가 **얼마나 높이 올라갔는지** 확인해야 했다.

"권한 상승. 공격자가 root까지 갔는지 확인한다."

---

web 서버에서의 권한 상승 경로는 이미 알고 있었다.

```
web 사용자 → sudo NOPASSWD:ALL → root
```

"하지만 공격자가 **다른 서버에도** 접근했을 가능성을 확인해야 해." 김영훈이 말했다.

정규가 4대 서버의 auth.log를 교차 분석했다.

```bash
# 4대 서버에서 web 서버(10.20.30.80)에서 온 SSH 접속 기록 확인
curl -s -X POST "http://localhost:8000/projects/$PID/execute-plan" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "tasks": [
      {"order":1,"title":"secu auth.log","instruction_prompt":"grep \"10.20.30.80\" /var/log/auth.log 2>/dev/null | grep Accepted | tail -5","risk_level":"low","subagent_url":"http://10.20.30.1:8002"},
      {"order":2,"title":"siem auth.log","instruction_prompt":"grep \"10.20.30.80\" /var/log/auth.log 2>/dev/null | grep Accepted | tail -5","risk_level":"low","subagent_url":"http://10.20.30.100:8002"},
      {"order":3,"title":"opsclaw auth.log","instruction_prompt":"grep \"10.20.30.80\" /var/log/auth.log 2>/dev/null | grep Accepted | tail -5","risk_level":"low","subagent_url":"http://localhost:8002"}
    ],
    "subagent_url":"http://localhost:8002",
    "parallel":true
  }'
```

결과:

```
[secu] (없음)
[siem] Mar 28 03:44:15 siem sshd: Accepted password for siem from 10.20.30.80 port 54321
       Apr 03 02:15:22 siem sshd: Accepted password for siem from 10.20.30.80 port 54322
[opsclaw] (없음)
```

"**siem 서버**에 접속한 기록이 있어." 정규의 목소리가 긴장되었다. "3월 28일과 4월 3일. 둘 다 **web 서버에서** SSH로."

"3월 28일은 최초 침해 직후야." 김영훈이 말했다. "공격자가 web를 장악한 뒤, **siem 서버에 측면 이동(Lateral Movement)**한 거야."

"siem 서버가 뚫렸으면..." 도윤이 말을 흐렸다.

"**SIEM 자체가 공격자 손에 있다**는 뜻이야." 김영훈이 이를 악물었다. "경보를 조작하거나 삭제할 수 있어. **감시자를 감시하는** 격이지."

---

"siem 서버에서 뭘 했는지 확인해." 김영훈이 지시했다.

```bash
# siem 서버에서 web(10.20.30.80) IP로 로그인한 세션의 활동
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"echo 1 | sudo -S grep -A5 \"from 10.20.30.80\" /var/log/auth.log 2>/dev/null | tail -20","subagent_url":"http://10.20.30.100:8002"}'
```

```
Mar 28 03:44:15 siem sshd: Accepted password for siem from 10.20.30.80
Mar 28 03:44:18 siem sudo: siem : TTY=pts/0 ; PWD=/home/siem ; USER=root ; COMMAND=/bin/cat /var/ossec/etc/rules/local_rules.xml
Mar 28 03:44:22 siem sudo: siem : TTY=pts/0 ; PWD=/home/siem ; USER=root ; COMMAND=/bin/cat /var/ossec/etc/ossec.conf
```

"Wazuh의 탐지 룰(local_rules.xml)과 설정(ossec.conf)을 **열람**했어." 정규가 말했다.

"탐지 룰을 보면 **어떤 공격이 탐지되고 어떤 게 안 되는지** 알 수 있어." 김영훈이 설명했다. "공격자가 방어 체계의 **약점**을 파악한 거야."

"4월 3일 접속에서는?" 진호가 물었다.

```
Apr 03 02:15:22 siem sshd: Accepted password for siem from 10.20.30.80
Apr 03 02:15:30 siem sudo: siem : COMMAND=/bin/systemctl stop wazuh-agent
Apr 03 02:15:35 siem sudo: siem : COMMAND=/bin/iptables -A OUTPUT -p tcp --dport 1514 -j DROP
```

관제실이 얼어붙었다.

"**Wazuh Agent를 정지시키고, 1514 포트(Wazuh 통신)를 차단**했어." 정규의 목소리가 떨렸다. "공격자가 **SIEM을 무력화**한 거야."

"그래서 4월 4일 웹 공격 때 탐지가 늦었던 거야?" 도윤이 물었다.

"아니." 정규가 고개를 저었다. "Wazuh Manager는 siem 서버에서 동작하고, Agent는 각 서버에서 동작해. 공격자가 정지한 건 **siem 서버 자체의 Agent**야. web 서버의 Agent는 별개."

"하지만 iptables로 1514를 차단하면, siem 서버에서 발생하는 이벤트가 **Manager에게 전달되지 않아**." 김영훈이 말했다. "siem 서버 **자체의** 침해 흔적이 로그에 안 남게 되는 거야."

"**자기 흔적을 지우기 위해 SIEM을 끈** 거네요." 가람이 말했다.

모두가 가람을 바라보았다. 가람은 자기도 모르게 핵심을 찔렀다.

"맞아." 추박하가 — 이미 돌아와 있었다 — 만화책 뒤에서 말했다. "이건 MITRE ATT&CK T1562.001(Disable or Modify Tools)이야. **방어 도구를 무력화**하는 거지."

---

김영훈이 타임라인을 업데이트했다.

```
[공격 타임라인 — 측면 이동 + SIEM 무력화]

03/28 03:43 — web 침투 (SSH 브루트포스)
03/28 03:44 — siem에 측면 이동 (SSH, 비밀번호 '1')
            → Wazuh 탐지 룰/설정 열람 (방어 체계 파악)
04/03 02:15 — siem 재접속
            → Wazuh Agent 정지 (systemctl stop)
            → iptables로 1514 차단 (로그 전송 방해)
            → 이후 siem 서버의 이벤트 SIEM에 미전달
04/04 02:48 — 대규모 웹 공격 (탐지 체계 일부 무력화된 상태에서)
```

"공격자는 **우리의 방어 체계를 알고, 약점을 찾고, 무력화**했어." 김영훈이 말했다. "이건 스크립트 키디가 아니야. **우리 시스템을 아는 사람**이야."

다시 한번, 그 결론이 떠올랐다.

*내부자.*

---

가람은 수첩에 적었다.

> `공격자의 사다리: web(SSH) → siem(SSH, 비밀번호 '1')`
> `siem 비밀번호도 '1'이었다. 모든 서버가 같은 비밀번호.`
> `Wazuh 룰 열람 → 방어 약점 파악 → Agent 정지 + 통신 차단`
> `SIEM을 무력화하면 "감시자가 눈을 감는다"`
> `이 모든 것은 내부 정보 없이는 불가능하다.`

가람은 수첩을 닫고 창밖을 바라보았다. 지하 2층에는 창이 없었다. 대신 대형 모니터의 빛만이 있었다.

*이 관제실은 회사의 눈이다. 하지만 그 눈을 가리려는 손이 있다.*
