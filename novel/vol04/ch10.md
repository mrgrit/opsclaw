# Chapter 10: 추박하의 결심

---

목요일 새벽 1시. 넥서스AI SOC 관제실.

관제실에는 야간 당번 도윤과, 왜인지 모르게 아직 남아 있는 가람, 그리고 — 추박하가 있었다.

추박하가 이 시간에 관제실에 있다는 것은 **비정상**이었다. 추박하는 보통 오후 4시면 만화책을 덮고 퇴근했다. 야근은 "젊은이들의 몫"이라며 절대 하지 않았다. 김영훈이 밤새 일할 때도 추박하는 "영훈아, 내일 봐"라고 하고 유유히 사라졌다.

그런 추박하가 새벽 1시에 앉아 있었다. 만화책도 없이. 모니터 앞에서.

"팀장님, 안 가세요?" 도윤이 조심스럽게 물었다.

"... 가야지." 추박하가 대답했다. 하지만 움직이지 않았다.

가람은 추박하의 화면을 볼 수 없었다. 추박하의 모니터는 늘 각도가 조절되어 있어서, 본인 외에는 볼 수 없었다. 하지만 키보드 소리는 들렸다. 빠르지 않지만 꾸준한 타이핑.

---

새벽 2시. 도윤이 잠깐 화장실에 간 사이, 가람은 추박하에게 다가갔다.

"팀장님."

"응."

"혹시... 도와드릴 거 있으세요?"

추박하가 가람을 바라보았다. 길고 조용한 시선. 가람은 그 시선에서 무언가를 느꼈다 — 피로? 아니. 무게. **결정의 무게.**

"가람아."

"네."

"넌 여기 온 지 얼마나 됐지?"

"3개월입니다."

"3개월이면... 이 팀이 좋은 팀인지 나쁜 팀인지 알겠어?"

가람은 잠시 생각했다.

"좋은 팀인 것 같아요. 다들 실력도 있고, 서로 챙기고."

"그래." 추박하가 고개를 끄덕였다. "좋은 팀이야. 내가 만든 건 아니지만, 내가 **지킨** 팀이긴 해."

"..."

"가람아. 한 가지만 물어볼게."

"네."

"**좋은 사람이 나쁜 일을 하면**, 그 사람은 좋은 사람이야, 나쁜 사람이야?"

가람은 대답하지 못했다.

추박하가 웃었다. 쓸쓸한 웃음이었다.

"어려운 질문이지. 나도 20년째 답을 못 찾고 있어."

추박하가 일어섰다. 코트를 집으며 말했다.

"가람아. 내일 아침에 영훈이한테 메시지 하나 전해줘."

"뭐라고요?"

"**genesis를 열 시간이 됐다.**"

### 오후: 분산 지식 아키텍처

오후 2시. 관제실 전원 집합.

김영훈, 정규, 민재, 진호, 지후, 도윤, 가람. 그리고 추박하.

추박하는 평소와 달리 만화책이 없었다. 의자에 앉아 팔짱을 끼고 있었다.

"오늘 안건은 OpsClaw의 **분산 지식 아키텍처**입니다." 김영훈이 시작했다. "민재, 설명해줘."

민재가 화이트보드 앞에 섰다.

"OpsClaw의 Experience Layer는 지금까지 **중앙 집중형**이었어요. 모든 경험이 opsclaw 서버의 DB에 저장되고, 다른 SubAgent는 이 중앙 DB를 참조하는 구조였죠."

```
[기존: 중앙 집중형]

  secu SubAgent → ─────┐
  web  SubAgent → ─────┼─→ opsclaw DB (중앙) → Experience
  siem SubAgent → ─────┘
```

"하지만 이 구조에는 **문제**가 있어요."

"어떤 문제?" 진호가 물었다.

"**opsclaw 서버가 죽으면 모든 지식이 사라져.**" 민재가 말했다. "그리고 각 SubAgent가 **자기 환경에서만 유효한 지식**을 중앙에 저장하면, 다른 환경에서는 쓸모없는 경험이 섞여."

"분산하자는 거야?" 정규가 물었다.

"네. **각 SubAgent가 자체적으로 Experience를 보유**하고, 필요할 때만 다른 SubAgent의 경험을 **교환**하는 구조."

```
[신규: 분산 지식형]

  secu SubAgent ←→ 자체 Experience DB
       ↕ (지식 교환 API)
  web  SubAgent ←→ 자체 Experience DB
       ↕
  siem SubAgent ←→ 자체 Experience DB
       ↕
  opsclaw Manager ←→ 중앙 Index (메타데이터만)
```

"각 SubAgent가 **자기 경험은 자기가 보관**하고, Manager는 **메타데이터 인덱스만** 갖고 있어. 'secu에 SSH 관련 경험이 있다'는 것만 알고, 실제 내용은 secu에 요청해서 가져오는 거야."

"블록체인처럼 **분산 원장**이네요." 가람이 말했다.

"비슷해. 하지만 합의 메커니즘은 없어. 각 노드가 **자기 데이터의 주인**이고, 필요할 때만 공유하는 **느슨한 연합**이야."

---

추박하가 입을 열었다. 관제실이 조용해졌다. 추박하가 기술 이야기를 직접 하는 것은 **극히 드문** 일이었다.

"분산 지식이 필요한 이유가 하나 더 있어."

"뭔가요?" 민재가 물었다.

"**한 곳에 모든 기록이 있으면, 한 번에 지울 수 있어.**"

관제실의 공기가 변했다.

"분산되어 있으면, 한 곳을 지워도 **다른 곳에 사본이 남아.** 그리고 각 SubAgent의 PoW 체인은 독립적이니까, 하나의 체인을 위조해도 **다른 체인과 교차 검증**하면 위조를 발견할 수 있어."

"팀장님." 정규가 말했다. "혹시 누군가가 OpsClaw의 기록을 **지우려고 할 것**을 대비하는 건가요?"

추박하가 정규를 바라보았다. 미소가 없는 눈.

"대비는 항상 하는 거야."

---

민재가 분산 지식 구현을 설명하는 동안, 가람은 추박하를 관찰했다.

*추팀장은 분산 지식을 **방어 수단**으로 보고 있다.*
*중앙의 genesis가 삭제되더라도, 각 SubAgent에 분산된 경험이 남는다.*
*그리고 PoW 체인의 교차 검증으로 위조를 발견할 수 있다.*

*이건 기술 설계가 아니다. 이건 **보험**이다.*

가람은 수첩에 적었다.

> `[분산 지식 아키텍처]`
> `  - 각 SubAgent가 자체 Experience DB 보유`
> `  - Manager는 메타데이터 인덱스만 관리`
> `  - 필요 시 지식 교환 API로 상호 참조`
> `  - PoW 체인 교차 검증 → 위조 탐지`
>
> `[추팀장의 의도?]`
> `  - "한 곳에 모든 기록이 있으면 한 번에 지울 수 있어"`
> `  - genesis의 기록이 분산 저장되면 삭제 불가능`
> `  - 서문기가 admin@nexusai로 삭제를 시도한 것처럼,`
> `    OpsClaw DB를 삭제하려는 시도가 올 수 있다`
> `  - 분산 = 방어 = 보험`

### RL Steering: 보상으로 행동을 조종하다

회의 후반, 정규가 새 기능을 소개했다.

"RL Steering에 대해 설명하겠습니다."

```
[RL Steering 개념]

기존 RL: "이 상황에서 어떤 리스크 수준이 최적인가?" → 추천

RL Steering: "에이전트의 행동을 보상으로 **유도**한다"
  → 보상 가중치를 조절하여 에이전트의 행동 패턴을 변경
  → 예: "보안을 더 중시하라" → risk_penalty를 높임 → 에이전트가 low risk 선호
  → 예: "속도를 더 중시하라" → speed_bonus를 높임 → 에이전트가 빠른 실행 선호
```

"보상 설계가 에이전트의 **행동**을 결정한다는 거죠." 진호가 말했다.

"맞아. 보상 함수를 바꾸면, 같은 에이전트라도 **완전히 다른 행동**을 해." 정규가 말했다.

```bash
# 현재 RL 정책 확인
curl -s -H "X-API-Key: opsclaw-api-key-2026" \
  "http://localhost:8000/rl/policy"
```

```json
{
  "algorithm": "Q-learning",
  "exploration": "UCB1",
  "states": 48,
  "actions": 4,
  "total_episodes": 89,
  "coverage": "62.5%",
  "reward_weights": {
    "base_score": 1.0,
    "speed_bonus": 0.3,
    "risk_penalty": 0.5
  }
}
```

"현재 **risk_penalty가 0.5**야." 정규가 설명했다. "이것을 **1.0**으로 올리면, 에이전트가 높은 리스크 작업을 더 꺼리게 돼. 반대로 0으로 낮추면 리스크를 무시하고 빠르게 실행해."

"그러면 **보안이 중요한 환경**에서는 risk_penalty를 높이고, **속도가 중요한 환경**에서는 낮추면 되겠네요." 가람이 말했다.

"정확해. 환경에 따라 에이전트의 성향을 **튜닝**할 수 있어." 정규가 말했다.

추박하가 중얼거렸다. "보상이 행동을 만든다. 사람이든 에이전트든."

### 밤: 추박하의 터미널

밤 9시. 대부분이 퇴근한 후.

추박하는 다시 관제실에 남아 있었다. 이번에는 가람만 함께.

추박하가 터미널을 열었다. 가람은 이번에는 **화면을 볼 수 있었다.** 추박하가 모니터 각도를 바꾸지 않았기 때문이다. 의도적인지 실수인지는 모른다.

화면에는 이런 것이 보였다.

```bash
curl -s -H "X-API-Key: opsclaw-api-key-2026" \
  "http://localhost:8000/projects/prj_000000000000/evidence/summary"
```

**prj_000000000000.** genesis.

결과가 화면에 나타났다.

```json
{
  "project_id": "prj_000000000000",
  "project_name": "genesis",
  "total_evidence": 40132,
  "success_count": 38901,
  "failure_count": 1231,
  "success_rate": "96.9%",
  "earliest": "2006-03-15T09:23:00",
  "latest": "2026-04-23T21:15:00"
}
```

가람의 심장이 빠르게 뛰었다.

*40,132건의 evidence. 2006년 3월부터 2026년 4월까지. **20년.**
*성공률 96.9%. 실패 1,231건.*

추박하가 다음 명령을 입력했다.

```bash
curl -s -H "X-API-Key: opsclaw-api-key-2026" \
  "http://localhost:8000/projects/prj_000000000000/replay" \
  | python3 -c "
import sys, json
data = json.load(sys.stdin)
timeline = data.get('timeline', [])
print(f'Total events: {len(timeline)}')
print(f'First: {timeline[0][\"ts\"]} — {timeline[0][\"action\"]}')
print(f'Last:  {timeline[-1][\"ts\"]} — {timeline[-1][\"action\"]}')
# 2006년 데이터 샘플
early = [t for t in timeline if t['ts'].startswith('2006')]
print(f'2006 events: {len(early)}')
if early:
    print(f'  First 2006: {early[0][\"action\"][:60]}')
"
```

```
Total events: 40132
First: 2006-03-15T09:23:00 — [GENESIS] 프로젝트 초기화: 시스템 보안 모니터링
Last:  2026-04-23T21:15:00 — [GENESIS] 분산 지식 아키텍처 백업 완료
2006 events: 347
  First 2006: [GENESIS] AlphaAI 내부망 접근 — 회장 승인 사본 #AUTH-001
```

가람은 숨을 멈추었다.

*[GENESIS] AlphaAI 내부망 접근 — 회장 승인 사본 #AUTH-001.*

*2006년. 넥서스AI가 보안 솔루션 회사였던 시절.*
*AlphaAI에 접근한 기록. **회장의 승인 사본.**
*추팀장은 20년 전부터 **회사의 지시로** 경쟁사를 해킹했고, 그 증거를 genesis에 기록했다.*

추박하가 화면을 닫았다. 가람을 돌아보았다.

"봤어?"

"... 네."

"어떻게 생각해?"

가람은 손이 떨렸다. "추팀장님이... 20년간 이걸..."

"기록했어." 추박하가 말했다. "회사가 시킨 일을 했고, 했던 일을 기록했어. 회장의 승인도, 서문기의 지시도, 정태식의 묵인도."

"왜요?"

추박하가 잠시 침묵했다.

"처음에는 **습관**이었어. 보안 엔지니어니까 기록하는 거지. 그 다음에는 **보호**였어. 나중에 '네가 했잖아'라고 할 때 '시키셔서 했습니다, 여기 증거 있습니다'라고 할 수 있으니까."

"그리고 지금은요?"

추박하가 가람을 바라보았다.

"지금은 **책임**이야. 40,132건의 기록은 **누군가** 세상에 내놓아야 해. 20년간 기록한 사람의 책임이야."

---

가람이 물었다. "그런데 팀장님도 **공범** 아닌가요? 회사의 지시를 실행한 사람이잖아요."

추박하가 눈을 감았다.

"맞아. 나도 공범이야."

"그러면 이 기록을 세상에 내놓으면, 팀장님도..."

"처벌받아." 추박하가 눈을 떴다. "하지만 기록을 묻으면, **더 큰 범죄가 계속돼.** Nexus-1 학습 데이터. 경쟁사 해킹. 고객 데이터 남용. 이건 **20년간 계속된 패턴**이야, 가람아. 내가 묻으면, 10년 후에도 같은 일이 반복돼."

가람은 아무 말도 할 수 없었다.

추박하가 일어섰다.

"내일 아침에 영훈이한테 genesis 접근 키를 줄 거야."

"김 과장님에게요?"

"응. 영훈이는 올바른 사람이야. 나와 다르게." 추박하가 코트를 집었다. "그리고 가람아."

"네."

"오늘 본 것은 **수첩에 적지 마.** 아직 때가 아니야."

추박하가 나갔다.

---

가람은 수첩을 펴려다 멈추었다. 적지 말라고 했다.

하지만 머릿속에서는 지울 수 없었다.

*genesis. 40,132건. 20년.*
*AlphaAI 내부망 접근. 회장 승인.*
*추팀장은 공범이면서 증인이다.*
*OpsClaw는 무기이면서 증거다.*
*기록은 방패이면서 폭탄이다.*

*그리고 내일, genesis의 열쇠가 김영훈 과장에게 전해진다.*

*이제 정말 — 돌아갈 수 없다.*
