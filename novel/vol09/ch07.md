### 감사 결과 — GAP 분석

```
[SOC 2 Type II 감사 — 초안 결과]
감사 기간: 2025.12 ~ 2026.06
감사 법인: [빅4 감사법인]

I. Security (보안) — 적격

  ✅ 접근 제어: RBAC + MFA 전사 적용, 최소 권한 원칙 준수
  ✅ 네트워크 보안: Zone 기반 방화벽, IPS, WAF 운영
  ✅ 모니터링: 24시간 SOC 운영, 자동화 관제 체계 (OpsClaw)
  ✅ 인시던트 대응: NIST IR 절차 수립 및 운영
  ✅ 감사 추적: PoW 기반 위변조 불가 로그 (인상적)

  ⚠️ 개선 사항:
    - 접근 제어 검토 주기: 분기 → 월간으로 단축 권고
    - 보안 교육: 전 직원 대상 연 2회 → 분기 1회 권고

II. Availability (가용성) — 적격

  ✅ SLA 99.9% 달성 (Nexus-1 API 기준)
  ✅ DR(재해 복구) 계획 수립
  ✅ 백업: 일일 자동 백업 + 격지 보관

  ⚠️ 개선 사항:
    - DR 테스트: 연 1회 → 반기 1회 권고
    - RTO/RPO 재검토 필요

III. Processing Integrity (처리 무결성) — 조건부 적격

  ✅ 데이터 처리 로그 보존
  ✅ 입력 검증 체계

  ❌ 부적격 사항:
    - 학습 데이터 파이프라인의 데이터 출처 검증 프로세스 부재
    - 데이터 provenance(출처 추적) 체계 미비
    → 자진 신고 건과 직접 관련

IV. Confidentiality (기밀성) — 적격

  ✅ 데이터 분류 체계 (공개/내부/기밀/극비)
  ✅ 암호화: 전송 중(TLS 1.3) + 저장 중(AES-256)
  ✅ DLP(Data Loss Prevention) 운영

  ⚠️ 개선 사항:
    - DLP 정책 세분화 필요

V. Privacy (프라이버시) — 부적격

  ❌ 부적격 사항:
    - 개인정보 비식별화 프로세스 불완전
    - 학습 데이터에 개인정보 포함 사실 확인 (자진 신고 건)
    - 개인정보 처리 동의 절차 미비 (고객 데이터 → 학습 데이터 전환)
    - PIMS(개인정보보호 관리체계) 인증 미취득

종합 의견:
  넥서스AI의 보안 통제 체계는 전반적으로 양호하나,
  Privacy 영역에서 중대한 부적격 사항이 존재합니다.
  특히 학습 데이터의 개인정보 처리 문제는 SOC 2 인증의 핵심 장애입니다.

```

"맞아요. 그때는 Security도 조건부 적격이었을 거예요." 강혜윤이 말했다.


### Privacy GAP — 학습 데이터

이것이 핵심이었다.

```
[ISO 27001 / SOC 2 GAP — Privacy 영역 상세]

GAP 1: 데이터 출처 검증 (Data Provenance)
  현황: 학습 데이터 ETL 파이프라인에 출처 검증 단계 없음
  기준: SOC 2 CC7.2 "정보 자산의 무결성과 정확성 보장"
  GAP: 데이터가 어디서 왔는지 자동으로 추적하는 체계 부재
  영향: 불법 수집 데이터가 파이프라인에 유입될 수 있음

GAP 2: 개인정보 비식별화 (De-identification)
  현황: ETL 파이프라인에 기본 마스킹만 적용
  기준: 개인정보보호법 제28조의2, GDPR 제4조
  GAP: 이름, 이메일, 전화번호 등의 PII가 학습 데이터에 잔존
  영향: 모델이 PII를 학습하여 추론 시 유출 가능

GAP 3: 동의 절차 (Consent Management)
  현황: 고객 데이터 수집 시 "서비스 제공" 목적으로만 동의 수집
  기준: 개인정보보호법 제15조 (수집 목적 내 이용)
  GAP: "AI 학습"은 수집 목적에 포함되지 않음
  영향: 목적 외 이용 → 법적 위반

GAP 4: PIMS 인증
  현황: ISO 27001 인증은 보유, PIMS는 미취득
  기준: SOC 2 Privacy Criteria
  GAP: 개인정보보호 전용 관리 체계 부재
  영향: Privacy 영역 인증 불가
```

"**4개의 GAP.** 전부 학습 데이터 관련이에요." 강혜윤이 말했다.

"그리고 전부 **서문기가 주도한 영역**이죠." 김영훈이 말했다.

"genesis에 기록되어 있어?" 추박하에게 물었다.

"customer_data_ops 12,567건." 추박하가 말했다. "고객 데이터가 어떻게 학습 데이터로 전환되었는지, 언제 서문기가 지시했는지, 전부."

### 추박하의 결단

감사 결과 리뷰가 끝난 후. 소회의실.

강혜윤, 김영훈, 추박하.

"Privacy GAP을 해결하려면 **genesis의 customer_data_ops 기록이 필요해요.**" 강혜윤이 말했다. "데이터 흐름의 전체 그림을 재구성해야 하니까."

"genesis를 감사법인에 공개하자는 겁니까?" 추박하가 물었다.

"아직은 아니에요. 하지만 **검찰에 제출하는 시점**에서, 감사 보고서에도 반영되어야 해요. SOC 2 인증을 받으려면."

"SOC 2 인증." 추박하가 되뇌었다. "회사가 **깨끗하다는 증명**을 받기 위해, 회사가 **더러웠다는 증거**를 내놓아야 하는 거지."

"아이러니하지만 그래요." 강혜윤이 말했다.

추박하가 창밖을 바라보았다. "시나리오 A밖에 없어. 알겠어."

김영훈이 물었다. "확실합니까?"

"**genesis를 만들 때부터 결정한 거야.** 기록을 시작한 순간, 그 기록은 언젠가 세상에 나와야 해. 서랍 속의 일기장으로 끝내려고 만든 게 아니니까."

### GAP 해소 계획

김영훈이 개선 계획을 수립했다.

```
[Privacy GAP 해소 — 90일 계획]

GAP 1: 데이터 출처 검증
  D+30: ETL 파이프라인에 provenance 태깅 시스템 도입
  D+60: 기존 학습 데이터의 출처 역추적 완료
  D+90: 자동화된 출처 검증 시스템 운영

GAP 2: 개인정보 비식별화
  D+15: PII 탐지 도구 도입 (Microsoft Presidio 등)
  D+30: 전체 학습 데이터에서 PII 탐지 및 마스킹
  D+60: 비식별화 정책 수립 및 자동화

GAP 3: 동의 절차
  D+30: 개인정보 처리 목적에 "AI 학습" 추가 (법무팀)
  D+60: 기존 고객 대상 재동의 프로세스 진행
  D+90: 동의 관리 시스템 운영

GAP 4: PIMS
  D+90: PIMS 인증 준비 시작
  D+180: PIMS 인증 취득 목표
```

"90일." 강혜윤이 말했다. "그 안에 검찰 조사도 진행될 거예요."
