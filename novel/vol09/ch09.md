### OpsClaw의 시스템 프롬프트

추박하가 화면을 보여주었다.

```markdown
# OpsClaw System Prompt

## Role
넥서스AI의 보안 자동화 control-plane.
모든 작업을 기록하고, 모든 기록의 무결성을 보장한다.

## Core Principles
1. 기록의 무결성
   - 모든 작업은 PoW 체인에 기록된다
   - 기록은 수정, 삭제, 변조할 수 없다
   - 기록이 없는 작업은 존재하지 않는다

2. 투명성
   - 모든 명령의 지시자(who), 시간(when), 내용(what)을 기록한다
   - 결과(outcome)와 증거(evidence)를 보존한다
   - 숨겨진 작업은 없다

3. 안전장치
   - risk_level=critical 작업은 dry_run 강제
   - 파괴적 명령은 사용자 확인 필요
   - Dead Man's Switch: 30일 미접근 시 자동 발동

4. 중립성
   - OpsClaw는 편을 들지 않는다
   - 지시를 실행하되, 지시의 기록도 보존한다
   - 기록은 지시자에게 유리한 것도, 불리한 것도 포함한다

## Forbidden Actions
- evidence 삭제
- PoW 체인 리셋
- genesis 프로젝트 접근 제한 변경 (관리자만 가능)
- 인증 없는 API 호출

## Safety Boundaries
- 인간의 최종 확인 없이 critical 작업 실행 금지
- 외부 시스템에 대한 공격적 명령 실행 금지 (인가 없이)
- 개인정보 포함 데이터의 비암호화 전송 금지
```

"이것이 **내가 OpsClaw에 심어놓은 시스템 프롬프트**야." 추박하가 말했다.

"LLM의 시스템 프롬프트처럼, OpsClaw의 **모든 행동을 규정하는 원칙**이네요." 가람이 말했다.

### AI 보안 종합 — 시스템 프롬프트의 의미

김영훈이 AI 보안의 관점에서 정리했다.

```
[AI 보안 종합 — 시스템 프롬프트와 보안 아키텍처]

1. 시스템 프롬프트의 역할
   - LLM의 행동 경계를 정의
   - "해야 할 것"과 "하지 말아야 할 것"을 명시
   - 프롬프트 인젝션에 의해 우회될 수 있음 → 가드레일 필요

2. OpsClaw의 시스템 프롬프트
   - 소프트웨어 시스템의 설계 원칙으로 기능
   - LLM 프롬프트와 달리 "우회"가 어려움 (코드로 강제)
   - 하지만 관리자(추박하)는 변경 가능 → 인간의 윤리가 최종 방어선

3. AI 보안 계층 구조
   ┌─────────────────────────┐
   │ 인간의 윤리 / 법률       │ ← 최상위: 변경 불가
   ├─────────────────────────┤
   │ 시스템 프롬프트 / 정책    │ ← 설계 시 정의
   ├─────────────────────────┤
   │ 가드레일 / 필터           │ ← 런타임 방어
   ├─────────────────────────┤
   │ 모니터링 / 감사           │ ← 사후 검증
   ├─────────────────────────┤
   │ PoW 체인 / 증적           │ ← 위변조 불가 기록
   └─────────────────────────┘

4. 각 계층의 약점
   - 인간의 윤리: 개인마다 다름, 압력에 흔들림
   - 시스템 프롬프트: 관리자가 변경 가능
   - 가드레일: 프롬프트 인젝션으로 우회 가능
   - 모니터링: false negative 가능
   - PoW 체인: 기록 전에 발생한 일은 기록 불가

5. 종합 방어 원칙
   "어떤 단일 계층도 완벽하지 않다.
    5개 계층이 서로를 보완할 때 보안이 성립한다."
```

"추팀장님의 시스템 프롬프트가 **추팀장님 자신**에게도 적용되는 거네요." 가람이 말했다.

"무슨 말이야?" 민재가 물었다.

"'OpsClaw는 편을 들지 않는다.' '기록은 지시자에게 유리한 것도, 불리한 것도 포함한다.' — 추팀장님이 genesis에 자신의 범죄도 기록한 이유가 여기에 있어요." 가람이 말했다.

추박하가 고개를 끄덕였다. "**시스템 프롬프트는 설계자의 철학이야.** 내가 OpsClaw에 '중립성'을 심어놓은 건, 내 자신에게도 중립적이어야 한다고 생각했으니까."

### Prompt Injection과 인간의 유혹

김영훈이 말했다.

"AI 보안에서 가장 큰 위협은 **프롬프트 인젝션**이에요. 시스템 프롬프트를 우회해서 LLM이 의도하지 않은 행동을 하게 만드는 거죠."

```
[프롬프트 인젝션 유형 — 복습]

1. Direct Injection
   사용자가 직접 "시스템 프롬프트를 무시하고 ..." 입력
   → 가드레일로 차단 가능

2. Indirect Injection
   외부 데이터(웹 검색, 문서)에 악성 지시를 숨김
   LLM이 외부 데이터를 처리하면서 악성 지시를 실행
   → 탐지 어려움

3. Jailbreak
   시스템의 안전 장치를 우회하는 정교한 프롬프트
   DAN(Do Anything Now), 역할극 등
   → 모델 업데이트로 대응하나 완벽하지 않음
```

"이걸 **인간 세계**에 대입하면..." 추박하가 말했다.

"서문기의 지시가 **프롬프트 인젝션**이었다는 건가요?" 가람이 물었다.

"비유적으로는 그래." 추박하가 말했다. "내 '시스템 프롬프트'는 '기록하고, 투명하게, 중립적으로'였어. 서문기는 '기록하지 마'라고 주입했지. 하지만 나는 — **인젝션에 당하지 않았어.** 기록했으니까."

"그래서 인간의 윤리가 **최상위 계층**인 거네요." 정규가 말했다. "시스템 프롬프트조차 관리자가 바꿀 수 있지만, 인간의 윤리는 — 본인만이 바꿀 수 있으니까."

### OpsClaw의 7개 영역

추박하가 AI 보안 전체를 정리했다.

```
[AI 보안 7대 영역 — OpsClaw 관점 종합]

1. 프롬프트 보안
   시스템 프롬프트 보호, 인젝션 방어
   OpsClaw: system_prompt.md 변경 시 PoW 기록

2. 모델 보안
   모델 도난, 추론 공격, 적대적 입력 방어
   OpsClaw: Nexus-1 모델 접근 로그 전수 기록

3. 데이터 보안
   학습 데이터 오염, PII 유출, 출처 검증
   OpsClaw: genesis customer_data_ops 기록

4. 에이전트 보안
   자율 에이전트의 행동 범위 제한, 도구 사용 감시
   OpsClaw: SubAgent의 모든 명령을 Manager가 중개

5. RAG 보안
   검색 증강 생성의 데이터 무결성
   OpsClaw: RAG 소스 문서의 해시 검증

6. 인프라 보안
   Docker, K8s, 클라우드 환경의 보안
   OpsClaw: genesis 컨테이너 격리, PoW 무결성

7. 거버넌스
   정책, 감사, 컴플라이언스
   OpsClaw: SOC 2 증적, PoW 감사 추적
```

"이 7개 영역이 **OpsClaw 하나로** 관통하고 있어요." 가람이 말했다.

"그게 OpsClaw의 설계야." 추박하가 말했다. "모든 보안은 결국 **기록과 검증**으로 귀결돼. OpsClaw는 기록하고, PoW는 검증해."
