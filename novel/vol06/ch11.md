### 통합 보안 아키텍처 재설계

정규가 새 아키텍처를 설계했다.

```
[넥서스AI 보안 아키텍처 — 목표 (After)]

                    ┌── Internet ──┐
                    │              │
                    └──── DMZ ─────┘
                          │
                    ┌─── secu ───┐   ← 방화벽 + IPS
                    │  nftables  │
                    │  Suricata  │
                    └─────┬──────┘
                          │
            ┌─────────────┼─────────────┐
            │             │             │
     [Zone A: Web]  [Zone B: AI]  [Zone C: Mgmt]
     ┌──────────┐  ┌──────────┐  ┌──────────┐
     │ Apache   │  │ Nexus-1  │  │ OpsClaw  │
     │ WAF      │  │ API      │  │ Manager  │
     │          │  │ (인증필수)│  │ SubAgent │
     └──────────┘  └──────────┘  └──────────┘
            │             │             │
     [Zone D: Test]       │      [Zone E: SIEM]
     ┌──────────┐         │      ┌──────────┐
     │ JuiceShop│         │      │ Wazuh    │
     │ (격리)   │         │      │ OpenCTI  │
     └──────────┘         │      └──────────┘
                          │
                   [Zone F: Data]
                   ┌──────────┐
                   │ PostgreSQL│
                   │ 학습 데이터│
                   └──────────┘

Zone 간 트래픽 규칙:
  A→B: 차단 (웹 서버가 AI 서버에 직접 접근 불가)
  A→C: 차단 (웹 서버가 관리 플랫폼에 접근 불가)
  D→*: 전부 차단 (테스트 환경은 완전 격리)
  C→B: 허용 (OpsClaw만 Nexus-1에 접근 가능)
  C→E: 허용 (OpsClaw만 SIEM에 접근 가능)
  E→*: 제한적 (SIEM은 로그 수집만)
```

"**6개 존(Zone)으로 분리.** 각 존 사이에 **명시적 트래픽 규칙.**" 정규가 설명했다.

"Zone D(테스트)가 **완전 격리**라면, JuiceShop에서 내부로 피벗하는 건 불가능해져요." 가람이 말했다. "6권 4장의 웹셸 → SubAgent 접근 같은 공격이 원천 차단돼요."

"맞아. **테스트 환경과 프로덕션 환경은 반드시 분리**해야 해." 김영훈이 말했다.

### 구현: nftables Zone 기반 정책

정규가 nftables로 존 기반 정책을 구현했다.

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/execute-plan" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "tasks": [
      {"order":1,"title":"secu: Zone 기반 nftables 정책","instruction_prompt":"echo 1 | sudo -S nft flush ruleset 2>/dev/null && echo 1 | sudo -S nft -f /etc/nftables-zones.conf 2>/dev/null && echo \"Zone-based nftables applied\" && echo 1 | sudo -S nft list ruleset 2>/dev/null | wc -l","risk_level":"critical","subagent_url":"http://10.20.30.1:8002","confirmed":true},
      {"order":2,"title":"검증: web→SubAgent 접근 차단 확인","instruction_prompt":"curl -s --connect-timeout 3 http://10.20.30.1:8002/a2a/health 2>/dev/null || echo \"BLOCKED (expected)\"","risk_level":"low","subagent_url":"http://10.20.30.80:8002"},
      {"order":3,"title":"검증: opsclaw→SubAgent 접근 허용 확인","instruction_prompt":"curl -s http://10.20.30.1:8002/a2a/health","risk_level":"low","subagent_url":"http://localhost:8002"}
    ],
    "subagent_url":"http://localhost:8002",
    "parallel": false
  }'
```

결과:

```
Zone-based nftables applied ✅ (312줄)
web→secu SubAgent: BLOCKED (expected) ✅
opsclaw→secu SubAgent: {"status":"ok"} ✅
```

"**187줄에서 312줄로 증가.** 존 기반 정책이 적용됐어." 정규가 말했다.

"Zone 간 트래픽이 **의도대로** 제한되는지 전수 테스트합니다."

```bash
# 전수 테스트: 6개 존 × 6개 존 = 36 경로
curl -s -X POST "http://localhost:8000/projects/$PID/execute-plan" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "tasks": [
      {"order":1,"title":"web→siem 차단 확인","instruction_prompt":"curl -s --connect-timeout 3 http://10.20.30.100:55000 2>/dev/null || echo \"BLOCKED\"","risk_level":"low","subagent_url":"http://10.20.30.80:8002"},
      {"order":2,"title":"web→opsclaw 차단 확인","instruction_prompt":"curl -s --connect-timeout 3 http://10.20.30.201:8000 2>/dev/null || echo \"BLOCKED\"","risk_level":"low","subagent_url":"http://10.20.30.80:8002"},
      {"order":3,"title":"siem→web 차단 확인","instruction_prompt":"curl -s --connect-timeout 3 http://10.20.30.80:80 2>/dev/null || echo \"BLOCKED\"","risk_level":"low","subagent_url":"http://10.20.30.100:8002"},
      {"order":4,"title":"opsclaw→all 접근 확인","instruction_prompt":"curl -s http://10.20.30.1:8002/a2a/health && curl -s http://10.20.30.80:8002/a2a/health && curl -s http://10.20.30.100:8002/a2a/health && echo \"All SubAgents accessible\"","risk_level":"low","subagent_url":"http://localhost:8002"}
    ],
    "subagent_url":"http://localhost:8002",
    "parallel": true
  }'
```

```
web→siem: BLOCKED ✅
web→opsclaw: BLOCKED ✅
siem→web: BLOCKED ✅
opsclaw→all: All SubAgents accessible ✅
```

"**완벽.** 의도한 대로 존 간 트래픽이 제한되고, opsclaw만 모든 SubAgent에 접근 가능해."

가람은 수첩에 적었다.

> `[통합 보안 아키텍처 — 교육 정리]`
>
> `네트워크 세그먼테이션: Zone 기반 분리`
> `  DMZ / Web / AI / Management / Test / SIEM / Data`
> `  각 Zone 간 트래픽 규칙 명시 (화이트리스트)`
> `  → "허용하지 않은 것은 차단한다" (Default Deny)`
>
> `Defense in Depth (심층 방어):`
> `  Layer 1: 네트워크 (nftables, Zone 분리)`
> `  Layer 2: IPS (Suricata, 커스텀 룰)`
> `  Layer 3: WAF (ModSecurity)`
> `  Layer 4: SIEM (Wazuh, FIM)`
> `  Layer 5: CTI (OpenCTI, IOC 매칭)`
> `  Layer 6: AI (Blue Agent, LLM 분석)`
> `  Layer 7: 사람 (SOC팀, 24시간 교대)`
> `  → "한 겹이 뚫려도 다음 겹이 막는다"`
>
> `[교훈]`
> `"보안 아키텍처는 한 번에 완성되지 않는다.`
> ` 공격을 받으며 강화하고, 강화하며 발전한다.`
> ` 하지만 기본은 변하지 않는다:`
> ` 분리하고, 제한하고, 감시하고, 기록한다."`
