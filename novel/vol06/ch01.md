# Chapter 1: 선전포고

---

월요일 오전 9시. 넥서스AI 대회의실.

서문기 상무가 직접 소집한 회의. 참석자: SOC팀 전원, SecureEdge 한승우 팀, 인프라 운영본부 정태식, 법무실 강혜윤, 인사실 진수현.

가람은 이 회의실에 이렇게 많은 사람이 모인 것을 처음 봤다. 보안팀의 전쟁이 **회사 전체의 전쟁**으로 확대되고 있다는 느낌이었다.

서문기가 먼저 말했다.

"Nexus-1 릴리즈가 **다음 주**입니다. SOC 2 감사는 **6월 첫째 주**. 이 두 가지를 모두 성공시키기 위해, 오늘부터 **2주간 종합 모의해킹(Red Team Exercise)**을 실시합니다."

화면에 슬라이드가 떴다.

```
[Operation Shield — 종합 침투 테스트]

목적: Nexus-1 릴리즈 전 보안 최종 검증 + SOC 2 감사 대비
기간: 5월 12일(월) ~ 5월 23일(금), 2주간
주관: SecureEdge (한승우 대표)

Red Team: SecureEdge 공격팀 (4명)
  → 넥서스AI 인프라 전체 대상 침투 테스트
  → 교전 규칙(Rules of Engagement) 적용

Blue Team: 넥서스AI SOC팀 (김영훈 과장)
  → 실시간 탐지, 대응, 보고

판정: 양측 보고서 + 독립 심사 (강혜윤 실장)
```

"**교전 규칙**은 한승우 대표가 설명합니다." 서문기가 한승우에게 마이크를 넘겼다.

한승우가 일어섰다. 차분한 목소리.

"교전 규칙, 즉 **RoE(Rules of Engagement)**입니다."

```
[교전 규칙 — Rules of Engagement]

허용:
  ✅ 외부 네트워크에서의 스캐닝 및 열거
  ✅ 공개된 취약점 익스플로잇
  ✅ 소셜 엔지니어링 (피싱 메일, 전화 등)
  ✅ 물리적 접근 시도 (배지 복제 등) — 사전 통보 후
  ✅ 내부 네트워크 피벗 (외부 진입 후)

금지:
  ❌ 서비스 거부(DoS/DDoS) 공격
  ❌ 프로덕션 데이터 삭제 또는 변조
  ❌ 개인 기기(BYOD) 대상 공격
  ❌ 물리적 시설 파손
  ❌ 실제 고객 데이터 접근 (학습 데이터 포함)

통보 의무:
  - Critical 취약점 발견 시 즉시 Blue Team에 통보
  - 프로덕션 서비스 영향 우려 시 사전 협의

기록:
  - Red Team의 모든 행동은 SecureEdge 측에서 기록
  - Blue Team의 탐지/대응은 OpsClaw에 기록
  - 양측 기록을 교차 검증하여 최종 보고서 작성
```

"소셜 엔지니어링도 허용이요?" 민재가 놀랐다. "피싱 메일을 보낸다고?"

"실제 공격자는 피싱을 써요." 한승우가 말했다. "방어 능력을 테스트하려면 **실전과 같은 조건**이 필요합니다."

"물리적 접근도?" 도윤이 물었다.

"배지 복제나 테일게이팅(따라 들어가기) 시도를 할 수 있습니다. 단, 사전에 보안팀에 **날짜만** 통보합니다. 구체적 방법은 알리지 않습니다."

김영훈이 팔짱을 끼고 들었다. 표정은 단단했다.

"교전 규칙은 수용합니다." 김영훈이 말했다. "다만 한 가지 추가 조건."

"뭡니까?" 한승우가 물었다.

"Red Team의 **모든 공격 트래픽**을 사후에 공개해주세요. 우리가 놓친 것이 뭔지 확인하려면, Red Team의 행동 로그가 필요합니다."

한승우가 잠시 생각했다. "합리적입니다. 훈련 종료 후 공개하겠습니다."

서문기의 표정이 미묘하게 변했다. 김영훈이 조건을 건 것이 마음에 안 드는 것 같았다.

---

회의가 끝나고 관제실로 돌아왔다.

"자, 전쟁 준비다." 김영훈이 팀을 모았다.

"2주간 **실시간 탐지와 대응**을 해야 해. Red Team이 공격하는 동안, 우리는 **실시간으로** 탐지하고 차단하고 기록한다."

"24시간이요?" 도윤이 물었다.

"Red Team도 사람이야. 밤에는 쉬겠지." 민재가 말했다.

"안 쉴 수도 있어." 정규가 말했다. "**실제 APT**는 타겟의 시간대를 노려서 야간에 공격하잖아. Red Team이 그걸 시뮬레이션할 수 있어."

"그래서 **교대 근무**를 짠다." 김영훈이 화이트보드에 적었다.

```
[Blue Team 교대 근무 — 2주간]

주간 (09~18): 김영훈 + 정규 + 가람
야간 (18~03): 도윤 + 진호
새벽 (03~09): 민재 + 지후

핵심:
  - 모든 경보는 OpsClaw에 기록
  - Suricata + Wazuh + OpsClaw LLM 분석 파이프라인 가동
  - Red Team 트래픽이 의심되면 즉시 Slack 알림
  - 차단은 "관찰 모드"에서 시작 → 확인 후 차단
    (Red Team 활동을 너무 빨리 차단하면 학습 기회를 잃음)
```

"**관찰 모드**가 핵심이에요." 가람이 말했다. "Red Team의 공격을 바로 차단하면, 그들이 **어디까지 갈 수 있는지** 파악 못 해요."

"맞아. Red Team Exercise의 목적은 **차단**이 아니라 **발견**이야." 김영훈이 고개를 끄덕였다. "우리가 몰랐던 약점을 Red Team이 찾아주는 거니까."

추박하가 만화책을 넘기며 말했다. "영훈아. 한 가지 더."

"뭡니까?"

"Red Team만 경계하지 마. **진짜 공격자**도 이 기간에 움직일 수 있어."

"무슨 뜻이요?"

"모의해킹이 진행되면, 네트워크에 **비정상 트래픽이 원래 많아져.** 진짜 공격자가 이때 끼어들면, Red Team의 노이즈에 **숨을** 수 있어."

관제실이 조용해졌다.

"**은폐 효과.**" 정규가 말했다. "축제 때 도둑이 들기 쉬운 것처럼."

"그래서 Red Team 트래픽의 **출발지 IP를 사전에 받아둬야** 해." 김영훈이 말했다. "Red Team IP가 아닌 출발지에서 같은 패턴의 공격이 오면, 그건 **진짜**야."

김영훈이 즉시 한승우에게 이메일을 보냈다.

```
수신: seungwoo.han@secureedge.co.kr
제목: Red Team 출발지 IP 목록 요청

한 대표님,
교전 규칙 합의에 따라, Red Team이 사용할 출발지 IP 주소 목록을
사전에 공유해주시기 바랍니다.
Blue Team의 정확한 탐지/대응을 위해 필요합니다.
(Red Team의 기법이나 일정은 공유하지 않으셔도 됩니다.)
```

30분 후 답장:

```
발신: seungwoo.han@secureedge.co.kr

김 과장님,
아래 3개 IP에서 공격 트래픽이 발생합니다.
  198.51.100.10
  198.51.100.11
  198.51.100.12
이 외의 IP에서 공격 트래픽이 관측되면, 그것은 저희가 아닙니다.
```

"Red Team IP **3개 확보.**" 김영훈이 말했다. "이 3개가 아닌 IP에서 공격이 오면 — 그건 **진짜 적**이야."

---

오후. 가람이 OpsClaw에 Red Team Exercise 프로젝트를 생성했다.

```bash
PID=$(curl -s -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"name":"operation-shield-blue","request_text":"Operation Shield — Blue Team 대응 기록","master_mode":"external"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['project']['id'])")

curl -s -X POST "http://localhost:8000/projects/$PID/plan" -H "X-API-Key: opsclaw-api-key-2026" > /dev/null
curl -s -X POST "http://localhost:8000/projects/$PID/execute" -H "X-API-Key: opsclaw-api-key-2026" > /dev/null
```

"2주간의 모든 탐지와 대응을 이 프로젝트에 기록합니다." 가람이 말했다.

"**Operation Shield.** 이름 잘 지었네." 도윤이 말했다.

"시작은 내일 아침 9시." 김영훈이 말했다. "오늘은 **준비**하자. Suricata 룰 업데이트, Wazuh 경보 임계값 조정, nftables 기본 정책 확인."

관제실이 움직이기 시작했다. 전쟁 전야의 긴장감.
