### AI 에이전트 아키텍처 설계

정규가 화이트보드에 아키텍처를 그렸다.

```
[자율 보안 에이전트 아키텍처]

Layer 1: 감지 (Detect)
  └─ Suricata IPS → 네트워크 경보
  └─ Wazuh SIEM → 시스템/앱 경보
  └─ OpsClaw Watcher → 상태 모니터링

Layer 2: 분석 (Analyze)
  └─ LLM 경보 분석 (5권에서 구현)
  └─ ATT&CK 자동 매핑
  └─ Context 참조 (Experience + CTI)

Layer 3: 판단 (Decide)
  └─ Risk Level 자동 분류 (RL 추천 활용)
  └─ Playbook 매칭 (유사 인시던트 → 자동 대응)
  └─ Escalation 규칙 (임계값 초과 시 사람에게 알림)

Layer 4: 실행 (Act)
  └─ 자동 대응 (low risk: 즉시 실행)
  └─ 승인 후 대응 (high/critical: 사람 승인 대기)
  └─ 기록 (모든 판단과 행동을 evidence로)
```

"이건 OpsClaw의 **기존 기능**을 조합한 거야." 정규가 말했다.

"Watcher가 감지, LLM이 분석, RL이 판단, execute-plan이 실행, evidence가 기록." 민재가 이어받았다. "새로 만드는 게 아니라, **기존 기능을 연결**하는 거야."

"**SOAR(Security Orchestration, Automation and Response)**의 LLM 버전이네요." 가람이 말했다.

"정확해." 김영훈이 말했다. "SOAR는 **룰 기반**으로 자동 대응하지만, 우리 건 **LLM이 분석하고 RL이 최적화**해. 더 유연하고, 경험이 쌓일수록 똑똑해져."

### 구현: Autonomous Blue Agent

민재가 구현을 이끌었다.

```bash
# 자율 에이전트 스케줄 등록 — 5분마다 실행
curl -s -X POST "http://localhost:8000/schedules" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{
    "name": "autonomous-blue-agent",
    "cron": "*/5 * * * *",
    "project_template": {
      "name": "auto-blue",
      "request_text": "자율 Blue Agent — 경보 수집+분석+대응"
    },
    "enabled": true
  }'
```

"5분마다 Blue Agent가 실행돼서:
1. Wazuh/Suricata 최신 경보를 수집하고
2. LLM으로 분석하고
3. true_positive이면 자동으로 인시던트를 생성하고
4. low risk이면 자동 대응(IP 차단 등), high 이상이면 Slack으로 사람에게 알림."

"**사람이 자도 AI가 감시한다.**" 도윤이 말했다. "이러면 새벽 교대가 편해지겠다."

"편해지는 게 아니라 **보조**받는 거야." 김영훈이 강조했다. "AI가 1차 필터를 하고, 사람은 AI가 올린 것만 확인하면 돼."

### 자율 Red Agent 개념

추박하가 만화책을 덮으며 말했다.

"Blue Agent만 있으면 아쉬워."

"Red Agent도 만들자는 건가요?" 민재가 물었다.

"OpsClaw의 SubAgent에 **자율 미션** 기능이 있잖아." 추박하가 말했다. "Red SubAgent가 **자동으로 공격을 시도**하고, Blue SubAgent가 **자동으로 방어**하면?"

```
[자율 Purple Team 구상]

Red SubAgent (공격):
  → Ollama gemma3:12b 모델
  → 미션: "web 서버에 취약점을 찾아 침투하라"
  → 자동으로 스캔 → 취약점 발견 → 익스플로잇 시도
  → 모든 행동을 OpsClaw evidence로 기록

Blue SubAgent (방어):
  → Ollama llama3.1:8b 모델
  → 미션: "경보를 분석하고 위협에 대응하라"
  → 자동으로 경보 수집 → 분석 → 차단/알림
  → 모든 대응을 OpsClaw evidence로 기록

Purple = Red + Blue 동시 실행:
  → 공격과 방어가 자동으로 반복
  → 경험(Experience)이 축적되면서 양쪽 모두 진화
  → RL 보상으로 성능 최적화
```

"**AI가 AI를 공격하고, AI가 AI를 방어한다.**" 가람이 말했다. "그리고 양쪽 다 경험이 쌓이면서 진화해."

"이건 **자기강화형 보안 시스템**이야." 정규가 말했다. "시간이 지날수록 공격도 강해지고, 방어도 강해지니까."

"구현은 다음에 하자." 김영훈이 말했다. "지금은 Blue Agent만으로 충분해. Red Team Exercise가 끝나면 Purple Team을 구축하자."

추박하가 만화책을 펴며 웃었다. "좋아. 기다릴게."
