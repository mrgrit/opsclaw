모든 사람이 퇴근한 후. 야간 교대인 도윤도 잠시 화장실에 갔다.

추박하는 터미널을 열었다.

```bash
curl -s http://192.168.0.105:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-oss:120b",
    "messages": [
      {"role": "system", "content": "당신은 보안 분석 전문가입니다. 주어진 증거를 종합하여 사건의 가능성 있는 시나리오를 추론하세요."},
      {"role": "user", "content": "증거: 1) SSH 브루트포스 공격자가 서버 사용자명을 미리 알고 있었음. 2) Nexus-1 API 엔드포인트가 다크웹에서 판매됨. 3) AI 연구소 직원(이주원)이 내부에서 프롬프트 인젝션을 시도. 4) 퇴직자 노트북에서 경쟁사 보고서 발견. 5) Apache 리버스 프록시가 보안팀 미통보로 설정됨. 이 증거들이 가리키는 가능성 있는 시나리오 3개를 제시하세요."}
    ],
    "temperature": 0.3,
    "max_tokens": 2000
  }' | python3 -c "import sys,json; print(json.load(sys.stdin)['choices'][0]['message']['content'])"
```

Ollama LLM의 응답이 화면에 나타났다.

```
시나리오 1: 내부자-외부 공모
이주원이 의도적으로 내부 정보(사용자명, API 경로)를 외부에 제공하고,
외부 공격자가 이를 활용하여 공격. 퇴직자 노트북의 경쟁사 보고서는
이주원과 경쟁사 간의 연결 고리일 가능성.
확률: 중간~높음

시나리오 2: 조직적 산업 스파이
경쟁사(AlphaAI)가 체계적으로 넥서스AI에 내부자를 심거나 매수.
IAB를 통한 접근 정보 확보, 프롬프트 인젝션으로 모델 기술 탈취.
퇴직자가 첫 번째 에이전트, 이주원이 두 번째.
확률: 중간

시나리오 3: 무관한 사건의 우연한 겹침
SSH 브루트포스는 일반적인 봇 공격, Nexus-1 API 노출은 단순 설정 실수,
이주원의 인젝션은 정상적인 Red Teaming, 퇴직자 보고서는 공개 자료.
모든 것이 별개 사건.
확률: 낮음
```

추박하는 화면을 읽고 고개를 끄덕였다.

*시나리오 1과 2가 유력하다.* 하지만 LLM의 분석만으로는 부족했다. 실제 데이터와 대조해야 한다.

추박하는 OpsClaw를 통해 교차 검증을 시작했다.

```bash
# 외부 공격 시간대와 내부 접근 시간대 상관관계 확인
curl -s -X POST "http://localhost:8000/projects/prj_00000000/execute-plan" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: ████████████" \
  -d '{
    "tasks": [
      {"order":1,"title":"외부 SSH 브루트포스 시간대","instruction_prompt":"grep \"Failed password\" /var/log/auth.log | awk \"{print substr(\\$1,1,10), \\$2}\" | sort | uniq -c | sort -rn | head -5","risk_level":"low","subagent_url":"http://10.20.30.1:8002"},
      {"order":2,"title":"이주원 VPN 세션 기록","instruction_prompt":"grep \"juwon\" /var/log/openvpn/status.log 2>/dev/null | tail -20","risk_level":"low","subagent_url":"http://10.20.30.1:8002"},
      {"order":3,"title":"Nexus-1 API 비정상 접근","instruction_prompt":"grep -E \"injection|prompt|bypass\" /var/log/nexus-api/access.log 2>/dev/null | awk \"{print \\$1, \\$4}\" | head -10","risk_level":"low","subagent_url":"http://10.20.30.80:8002"}
    ],
    "subagent_url":"http://localhost:8002",
    "parallel":true
  }'
```

결과가 돌아왔다. 추박하는 세 개의 결과를 나란히 놓고 시간대를 비교했다.

*외부 공격 피크: 02:48~03:15. 이주원 VPN 해제: 02:30. 차이: 18분.*

18분. 우연이라기엔 너무 정확했다.

추박하는 다시 Ollama를 호출했다. 이번에는 모델 선택에 대해 잠시 고민했다.

DGX-Spark에는 여러 모델이 올라가 있었다. 가벼운 분석이라면 gemma3:12b로 충분하다. 하지만 지금은 다중 증거 간의 인과관계를 추론해야 한다. 시간순 배열, 동기 분석, 가능성 평가 — 이런 복합적 추론에는 파라미터가 큰 모델이 유리하다.

*gpt-oss:120b.* 120억이 아니라 1,200억 파라미터. DGX-Spark의 128GB 메모리를 거의 전부 쓰는 모델이다. 응답은 느리지만, 복잡한 추론 품질은 확실히 다르다.

Ollama는 로컬에서 돌아가는 LLM 서버다. OpenAI 호환 API를 제공하기 때문에, curl 한 줄이면 보안 분석을 요청할 수 있다. 클라우드 AI 서비스와 달리 **데이터가 외부로 나가지 않는다** — 보안 조사에서 이 점이 결정적이다. 증거를 외부 서버에 보내는 순간, 그 자체가 정보 유출이 된다.

temperature 0.3은 창의적 응답보다 일관된 분석을 원한다는 의미다. 보안 분석에서는 상상력보다 정확성이 중요하다.

```bash
curl -s http://192.168.0.105:11434/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-oss:120b",
    "messages": [
      {"role": "system", "content": "보안 포렌식 전문가로서, 시간대 상관관계를 분석하세요."},
      {"role": "user", "content": "외부 SSH 브루트포스 시작: 02:48. 이주원 VPN 해제: 02:30. 시간차 18분. 이 상관관계의 가능한 해석과, 우연의 확률을 분석하세요."}
    ],
    "temperature": 0.3,
    "max_tokens": 1000
  }' | python3 -c "import sys,json; print(json.load(sys.stdin)['choices'][0]['message']['content'])"
```

LLM의 답변이 화면에 떠올랐다.

```
18분 시간차 분석:
1. 의도적 시간차 (가장 유력): 내부자가 VPN 작업 완료 후 연결을 끊고,
   일정 시간 후 외부 공격이 시작되도록 설계. 자신의 VPN 세션과
   공격 시간이 겹치지 않게 하려는 의도적 분리.
2. 자동화 가능성: VPN 해제가 외부 공격의 트리거일 수 있음.
   스크립트로 "VPN 해제 → 15분 대기 → 공격 시작" 구성 가능.
3. 우연의 확률: 하루 1,440분 중 특정 18분 창에서 두 이벤트가
   겹칠 확률 ≈ 1.25%. 단독으로는 유의미하지 않으나,
   다른 정황(사용자명 사전 파악, API 노출)과 결합 시 확률은 급감.
```

추박하는 마지막으로 RL 추천을 확인했다. OpsClaw의 강화학습 엔진은 과거 작업 이력을 기반으로 최적의 행동을 추천해준다.

```bash
curl -s "http://localhost:8000/rl/recommend?agent_id=http://localhost:8002&risk_level=medium" \
  -H "X-API-Key: ████████████"
```

```json
{
  "recommended_action": "escalate_investigation",
  "confidence": 0.82,
  "reason": "medium-risk 정황 증거 축적 패턴 — 과거 유사 사례에서 조기 에스컬레이션이 최적 보상"
}
```

*RL도 에스컬레이션을 권한다.* 하지만 추박하는 아직 움직이지 않았다. 기계의 추천과 자신의 직감이 같은 방향을 가리키고 있었지만, 아직 **결정적 한 방**이 부족했다.

그는 LLM의 분석과 RL 추천, 그리고 자신의 판단을 genesis에 기록했다.

```bash
curl -s -X POST http://localhost:8000/projects/prj_00000000/dispatch \
  -H "X-API-Key: ████████████" \
  -d '{"command":"echo $(date +%Y%m%d_%H%M%S) LLM_ANALYSIS_3SCENARIOS_INSIDER_INDUSTRIAL_COINCIDENCE >> /var/log/opsclaw/genesis.log"}'
```

그는 LLM의 분석을 참고하되, **자신만의 결론**은 아직 내리지 않았다.

하지만 한 가지는 확신했다.

*시나리오 3(우연)은 아니다. 이 정도의 겹침은 우연이 아니다.*

> `Ollama — 로컬 LLM 서버. 외부 API와 달리 데이터 유출 위험 없음.`
> `DGX-Spark (192.168.0.105) — GPU 서버에서 gpt-oss:120b 모델 구동.`
> `120b = 1,200억 파라미터. 복잡한 추론에 큰 모델이 필요한 이유:`
> `  - 다중 증거 간 인과관계 분석`
> `  - 시간순 상관관계 추론`
> `  - 단순 분류가 아닌 "왜?"에 대한 답`
> `AI 보안 분석의 핵심: 기계가 답을 주는 게 아니라, 사람이 질문을 잘 해야 한다.`
> `temperature 낮음 = 일관된 분석. 높음 = 창의적 응답.`
> `추팀장은 LLM을 "도구"로 쓴다. 결론은 자신이 내린다.`
