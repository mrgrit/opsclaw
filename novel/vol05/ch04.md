# Chapter 4: 도둑맞은 모델

---

목요일 오전 8시. 넥서스AI 로비.

가람이 출근하자마자 이상한 광경을 보았다. 로비에 **양복 차림의 남녀 4명**이 서 있었다. 방문자 배지를 달고 있었지만, 분위기가 달랐다. 일반 방문객의 어색함이 없었다. 마치 자기 건물인 것처럼 주변을 훑고 있었다.

"저 사람들 누구예요?" 가람이 로비의 경비원에게 물었다.

"SecureEdge라는 보안 감사 회사래요. 서문기 상무님이 부르셨다고."

*서문기가 외부 보안 감사 회사를 불렀다?*

가람은 관제실로 빠르게 올라갔다.

---

관제실에서는 이미 긴장감이 감돌고 있었다.

"SecureEdge?" 김영훈이 이름을 되뇌었다. "들어본 적 없는데."

"국내 보안 컨설팅 Top 5에는 없어요." 진호가 빠르게 검색했다. "2020년 설립. 대표 한승우. 주 고객층은 중견 IT 기업. 직원 약 30명."

"작은 회사네." 민재가 말했다.

"서문기가 왜 **갑자기** 외부 감사 회사를 불렀을까?" 김영훈이 생각에 잠겼다.

추박하가 만화책 뒤에서 말했다. "반격이야."

"반격이요?" 도윤이 물었다.

"우리가 Nexus-1 보안 검증을 하고 있잖아. 서문기는 이걸 알고 있어. 강혜윤이 법무팀 권한으로 접근을 확보한 것도 알 거고. 그래서 **자기 사람**을 넣은 거야. 외부 감사라는 이름으로."

"자기 사람이라면..." 김영훈이 말했다.

"SecureEdge가 '넥서스AI의 보안은 문제없습니다'라는 보고서를 내면, 서문기는 이걸 **우리 보고서에 대한 반박**으로 쓸 수 있어."

관제실이 무거워졌다.

---

오전 11시. 서문기 상무가 SecureEdge 대표를 데리고 SOC에 왔다.

한승우. 40대 중반. 짧은 머리, 날카로운 눈, 단정한 정장. 말투는 부드럽지만, 눈은 차가웠다.

"보안팀 여러분, 안녕하세요. SecureEdge 대표 한승우입니다." 한승우가 명함을 돌렸다. "서문기 상무님의 의뢰로 넥서스AI의 **전반적 보안 상태를 독립 평가**하게 되었습니다."

"독립 평가." 김영훈이 말을 되씹었다.

"네. 목적은 SOC 2 감사 대비 **사전 진단**입니다. 보안팀과 협력하여 진행하고 싶습니다."

서문기가 뒤에서 지켜보고 있었다. 표정은 온화했지만, 눈은 김영훈을 향해 있었다.

"협력은 환영합니다." 김영훈이 말했다. "다만 접근 범위와 방법론은 사전에 **문서화**해주셔야 합니다."

"당연하죠." 한승우가 웃었다. "모든 것을 문서화하는 건 보안의 기본이니까요."

서문기가 만족한 표정으로 나갔다.

---

한승우가 나간 후, 관제실은 즉시 긴급 회의에 들어갔다.

"서문기의 **트로이 목마**야." 민재가 말했다.

"섣불리 판단하지 마." 추박하가 말했다. "한승우가 서문기의 사람인지, 아니면 진짜 독립 감사인지는 아직 몰라."

"팀장님, 한승우라는 사람을 아세요?" 김영훈이 물었다.

추박하가 잠시 침묵했다.

"... 이름은 알아."

"어디서요?"

추박하가 만화책을 넘겼다. "군 사이버사령부 시절에."

관제실이 얼어붙었다. 추박하의 **군 시절** 이야기가 나온 것은 처음이었다.

"한승우도 사이버사령부 출신이었어요?" 가람이 물었다.

"아니. 한승우는 **민간 정보보안 전문가**야. 하지만 사이버사령부가 민간 전문가를 **자문**으로 쓸 때 이름이 올라왔어."

"그러면 실력은 있는 사람인가요?"

"실력은 있어." 추박하가 말했다. "문제는 **누구를 위해** 실력을 쓰느냐야."

### 오후: 모델 도난 위협 분석

오전의 긴장을 뒤로하고, 오후에는 계획대로 **모델 도난(Model Theft)** 위협을 분석했다.

"Nexus-1의 모델 가중치 파일은 어디에 보관되어 있어?" 김영훈이 도현에게 물었다.

도현이 아키텍처를 설명했다.

```
[Nexus-1 모델 가중치 보관 구조]

1. 학습 서버 (GPU 클러스터):
   /data/nexus1/weights/v3.2/
   → 용량: 약 400GB (70B 파라미터 × FP16)
   → 접근: AI 연구소 직원 (SSH 키 인증)

2. 배포 서버 (API 서빙):
   /opt/nexus1/model/
   → 용량: 동일
   → 접근: 서비스 계정 (자동 배포)

3. 백업:
   AWS S3 (ap-northeast-2)
   → s3://nexusai-model-vault/nexus1/v3.2/
   → 접근: IAM 역할 기반

4. 이주원이 다운로드한 파일:
   nexus1_weights_v3.2.tar.gz (Google Drive에서 발견)
   → 전체 가중치의 일부? 전체?
```

"이주원이 가져간 **nexus1_weights_v3.2.tar.gz**가 전체 가중치인지 일부인지가 중요해." 김영훈이 말했다.

"파일 크기로 추정할 수 있어요." 도현이 말했다. "전체 가중치가 400GB인데, 이주원의 Drive에 있던 파일이 몇 GB였죠?"

가람이 포렌식 기록을 확인했다.

```bash
curl -s -H "X-API-Key: opsclaw-api-key-2026" \
  "http://localhost:8000/projects/juwon-pc-forensic/evidence/summary" \
  | python3 -c "
import sys, json
data = json.load(sys.stdin)
for e in data.get('evidence', []):
    if 'weights' in str(e.get('detail','')).lower():
        print(e['detail'][:150])
"
```

"**nexus1_weights_v3.2.tar.gz — 12.8GB.** 전체 400GB의 일부예요." 가람이 말했다.

"12.8GB면 **양자화(Quantized) 버전**일 수 있어요." 도현이 말했다. "INT4 양자화하면 400GB가 약 12~15GB로 줄어요. 정확도는 약간 떨어지지만, 동작은 해요."

"그러면 이주원이 가져간 건 **양자화된 전체 모델**이라는 거야?" 김영훈이 물었다.

"가능성이 높아요."

---

김영훈이 화이트보드에 모델 도난의 위협을 정리했다.

```
[모델 도난(Model Theft) 위협 분석]

위협 1: 내부자에 의한 직접 복제
  - 이주원이 모델 가중치를 Google Drive로 유출
  - 양자화 버전이면 일반 GPU에서도 실행 가능
  - 경쟁사나 제3자에게 전달 가능
  → 피해: 수백억 원의 학습 비용 + 경쟁 우위 상실

위협 2: API를 통한 모델 추출 (Model Extraction Attack)
  - 대량의 입출력 쌍을 수집하여 "그림자 모델" 학습
  - API가 공개되면 누구나 시도 가능
  - 방어: rate limiting, 출력 제한, API 인증
  → 피해: 모델의 "지식"이 복제됨

위협 3: 추론 공격 (Inference Attack)
  - 모델의 출력 패턴으로 학습 데이터 추론
  - 어제/오늘의 탈옥 + 적대적 입력이 바로 이것
  - 방어: Differential Privacy, Output Perturbation
  → 피해: 학습 데이터(개인정보, 기밀 포함)가 추론됨

위협 4: 가중치 역공학 (Weight Reverse Engineering)
  - 가중치에서 학습 데이터의 패턴을 직접 추출
  - 연구 단계이지만, 가능성은 있음
  → 피해: 학습 데이터의 직접적 재구성
```

"우리가 어제/오늘 한 것은 **위협 3**이야." 김영훈이 말했다. "그리고 이주원이 한 것은 **위협 1**이야. 모델 자체를 가져간 거지."

"위협 2도 주의해야 해요." 정규가 말했다. "Nexus-1 API가 공개되면, 누구든 대량 요청으로 그림자 모델을 만들 수 있어요."

"2권에서 외부 공격자가 17건의 completions를 호출했잖아요." 가람이 말했다. "그것도 **모델 추출**의 시작이었을 수 있어요."

"연결됐어." 김영훈이 고개를 끄덕였다. "2권의 API 공격 → 모델 추출 시도. 이주원의 다운로드 → 직접 복제. 전부 Nexus-1의 **지적 재산 유출**이야."

### 저녁: 한승우와의 대화

저녁 7시. 관제실에서 대부분이 퇴근한 후.

가람이 자판기에서 커피를 뽑으려는데 한승우가 서 있었다.

"안녕하세요. 오 가람 사원이죠?" 한승우가 웃으며 말했다.

"네. 안녕하세요."

"자판기 커피가 그나마 나은 편이더라고요. 여기 커피는 좀..." 한승우가 웃었다.

"민재 선배 커피를 드셨군요."

"누군지는 모르겠지만, **용감한 커피**였어요."

가람이 웃었다. 한승우는 불쾌한 사람은 아니었다.

"가람 사원, 보안팀에 온 지 얼마나 됐어요?"

"3개월요."

"3개월 만에 이런 사건을 겪다니. 대단해요."

"대단한 건 아니고, 그냥... 배우는 중이에요."

한승우가 커피를 한 모금 마셨다.

"나도 비슷했어요. 처음 보안 업계에 들어왔을 때, 큰 사건 한가운데 있었거든요."

"어떤 사건이요?"

한승우가 잠시 생각했다. "옛날 얘기야. 중요한 건—" 한승우가 가람을 바라보았다. "이 사건에서 **진실이 뭔지**는 나도 모릅니다. 서문기 상무가 부른 건 맞지만, 내 보고서는 **내가 본 것만** 씁니다."

"... 그래요?"

"네가 본 것과 내가 본 것이 다를 수 있어요. 하지만 **둘 다 진실**일 수 있어요."

가람은 이 사람이 단순한 서문기의 꼭두각시가 아닐 수도 있다고 느꼈다.

"한 대표님." 가람이 물었다. "추박하 팀장님을 아세요?"

한승우의 표정이 미묘하게 변했다. 아주 잠깐.

"... 이름은 들어봤어요."

그리고 돌아섰다.

가람은 수첩에 적었다.

> `한승우 (SecureEdge 대표)`
> `  - 서문기가 선임한 외부 감사 회사`
> `  - 추박하와 군 사이버사령부 시절 접점 (추팀장 언급)`
> `  - 한승우도 추박하를 "이름은 들어봤다" → 알고 있다`
> `  - "내 보고서는 내가 본 것만 쓴다" → 독립적?`
> `  - 표정 변화: 추박하 이름에 반응 → 단순 관계 아님`
>
> `[모델 도난 위협 교육 정리]`
> `  내부자 복제: 가중치 파일 직접 유출 (이주원)`
> `  API 추출: 대량 요청으로 그림자 모델 학습`
> `  추론 공격: 출력 패턴으로 학습 데이터 추론`
> `  역공학: 가중치에서 데이터 직접 추출 (연구 단계)`
>
> `방어:`
> `  접근 통제 + 최소 권한 원칙`
> `  API rate limiting + 출력 제한`
> `  Differential Privacy`
> `  DLP(Data Loss Prevention) + 이상 행동 탐지`
>
> `[풀리지 않는 질문]`
> `  한승우는 서문기의 사람인가, 독립적인가?`
> `  추박하와 한승우의 관계는?`
> `  SecureEdge의 감사 결과가 우리 보고서와 충돌하면?`
