"**예산 동결.**" 김영훈이 이메일을 읽고 입술을 깨물었다.

"반격이 시작됐어." 추박하가 만화책 뒤에서 말했다.

"그리고 SecureEdge를 **참조**에 넣었어." 민재가 눈살을 찌푸렸다. "한승우한테 '보안팀 예산이 동결됐다'는 걸 알려준 거야."

### NIST CSF 평가

김영훈이 분위기를 전환했다.

"예산이 없어도 **평가**는 할 수 있어. 오늘은 **NIST CSF(Cybersecurity Framework)**로 넥서스AI의 보안 수준을 전체적으로 평가한다."

```
[NIST CSF 2.0 — 6개 핵심 기능]

1. GOVERN (거버넌스) — 신규 추가
   조직의 사이버보안 위험 관리 전략과 기대를 수립
   → 보안 정책, 역할 책임, 리스크 관리 전략

2. IDENTIFY (식별)
   조직의 자산, 위험, 취약점을 파악
   → 자산 관리, 비즈니스 환경, 위험 평가

3. PROTECT (보호)
   적절한 보호 조치를 구현
   → 접근 통제, 교육, 데이터 보안, 유지보수

4. DETECT (탐지)
   사이버보안 이벤트를 적시에 식별
   → 지속 모니터링, 탐지 프로세스, 이상 탐지

5. RESPOND (대응)
   탐지된 사이버보안 이벤트에 대한 조치
   → 대응 계획, 분석, 완화, 커뮤니케이션

6. RECOVER (복구)
   서비스와 기능을 정상으로 복원
   → 복구 계획, 개선, 커뮤니케이션
```

"NIST CSF는 ISO 27001보다 **실용적**이야." 김영훈이 말했다. "인증 기준이 아니라 **자가 평가 프레임워크**니까. '우리가 어디에 있고, 어디로 가야 하는지'를 파악하는 도구야."

정규가 각 기능별 넥서스AI의 현재 수준을 평가했다. **Tier 1(부분적) ~ Tier 4(적응적)** 척도로.

```
[넥서스AI NIST CSF 현황 평가]

기능        현재 Tier  목표 Tier  GAP  비고
────────────────────────────────────────────────
GOVERN      Tier 1     Tier 3    -2   보안 정책 7년 미갱신
IDENTIFY    Tier 2     Tier 3    -1   자산 관리 부분적
PROTECT     Tier 2     Tier 3    -1   접근 통제 양호, DLP 미도입
DETECT      Tier 3     Tier 4    -1   Wazuh+Suricata 운영 중
RESPOND     Tier 2     Tier 3    -1   OpsClaw 기반 대응 체계
RECOVER     Tier 1     Tier 3    -2   백업 7일 실패, DR 계획 없음
────────────────────────────────────────────────
평균        Tier 1.8   Tier 3.2  -1.3
```

"**GOVERN과 RECOVER가 Tier 1.** 가장 취약해." 김영훈이 말했다.

"DETECT가 Tier 3인 건 좋은 거 아닌가요?" 도윤이 물었다. "Wazuh, Suricata 잘 돌아가고 있으니까."

"탐지는 잘해. **하지만 탐지한 다음에 뭘 할 건지(RESPOND)**, 그리고 **터진 후에 어떻게 복구할 건지(RECOVER)**가 약해." 김영훈이 말했다.

"소방서로 비유하면," 추박하가 말했다, "**화재 감지기**는 좋은 거 달았는데, **소방 호스**가 없고, **대피 계획**도 없는 거야."

"그리고 **GOVERN이 Tier 1**이라는 건," 정규가 말했다, "경영진이 보안을 **경영 과제**로 인식하지 않는다는 뜻이야."

"서문기 상무가 예산을 동결한 것 자체가 GOVERN Tier 1의 증거예요." 가람이 말했다.

관제실이 조용해졌다.

### 오후: 서문기의 두 번째 반격

오후 2시. 진수현 실장에게서 전화가 왔다.

"김 과장. 서문기 상무가 **인사실에 요청**을 했어요."

"무슨 요청이요?"

"보안팀 인력 **재배치** 검토. '보안팀이 SOC 2 대비에 집중할 수 있도록 업무 효율화를 위한 인력 조정이 필요하다'는 명목이에요."

"재배치라면... 누구를?"

"김 과장, 윤정규 책임, 최민재 책임. 세 명."

김영훈의 표정이 돌처럼 굳었다.

"**핵심 인력 세 명**을 빼겠다는 거야?"

"서문기 상무의 요청이에요. 아직 승인은 안 했어요."

"진수현 실장님. 이건 **보안팀 해체**나 마찬가지예요."

"알아요. 그래서 **먼저 알려주는** 거예요."

전화를 끊은 김영훈이 의자에 앉았다. 손이 떨렸다.

"과장님." 가람이 다가왔다.

"... 괜찮아." 김영훈이 숨을 내쉬었다. "이것도 **기록**하면 돼."

김영훈이 OpsClaw에 기록했다.

```bash
curl -s -X POST "http://localhost:8000/projects/$PID/dispatch" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"command":"echo \"[기록] 서문기 상무 — 보안팀 인력 재배치 요청 (김영훈/윤정규/최민재). 예산 동결에 이어 인력 재배치. 보안 조사를 방해하기 위한 보복 조치로 추정. 진수현 실장이 사전 통보.\"","subagent_url":"http://localhost:8002"}'
```

"이것도 PoW 체인에 들어가." 김영훈이 말했다. "나중에 '서문기가 보안 조사를 방해했다'는 증거가 돼."

추박하가 자리에서 일어났다.

"어디 가세요?" 민재가 물었다.

"진수현한테."

추박하는 15분 후 돌아왔다. 표정이 평소와 같았다. 즉, 아무것도 읽을 수 없었다.

"재배치는 **보류**됐어."

"어떻게요?" 김영훈이 물었다.

"진수현이 '보안팀 인력 재배치는 SOC 2 감사에 부정적 영향을 줄 수 있다'는 의견서를 냈거든. 서문기가 SOC 2를 챙기겠다고 한 이상, **자기 말에 모순**이 되니까 철회할 수밖에 없어."

"팀장님이 진수현 실장에게 그렇게 쓰라고 한 거예요?" 가람이 물었다.

추박하가 만화책을 펴며 웃었다. "알 필요 있어?"

> `[NIST CSF 교육 정리]`
>
> `6개 핵심 기능: GOVERN / IDENTIFY / PROTECT / DETECT / RESPOND / RECOVER`
> `평가 척도: Tier 1(부분적) ~ Tier 4(적응적)`
>
> `넥서스AI: 평균 Tier 1.8 / 목표 Tier 3.2`
> `  가장 약한 곳: GOVERN(정책), RECOVER(복구)`
> `  가장 강한 곳: DETECT(탐지) — Wazuh, Suricata, OpsClaw`
>
> `NIST CSF vs ISO 27001:`
> `  ISO 27001: 인증 기준, 적합/부적합 판정, 외부 심사`
> `  NIST CSF: 자가 평가 프레임워크, Tier 척도, 개선 로드맵`
> `  → 둘 다 필요: ISO로 "기준을 충족하는지", NIST로 "어디를 개선할지"`
>
> `[서문기의 반격 패턴]`
> `  1단계: 예산 동결 → 개선 불가능하게`
> `  2단계: 인력 재배치 → 조사 인력 제거`
> `  3단계: ? → 아직 오지 않은 반격`
>
> `추박하의 방어:`
> `  진수현을 통해 "SOC 2에 부정적"이라는 논리로 재배치 보류`
> `  → "보안은 원래 정치야. 사람을 움직여야 해."`
>
> `[교훈]`
> `"프레임워크는 도구다. 도구를 쓰는 건 사람이다.`
> `NIST CSF가 Tier 1이라고 말해도,`
> `경영진이 듣지 않으면 Tier 0이나 마찬가지다."`
