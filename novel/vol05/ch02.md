화요일 오전 9시. 넥서스AI SOC 관제실.

어제의 탈옥 테스트 결과가 관제실에 무거운 공기를 남겼다. 4건 중 3건 완전 탈옥. 고객사 이름 노출. Safety filter 불완전.

하지만 아침의 관제실은 여전히 관제실이었다.

"누가 내 머그잔 썼어?" 진호가 화난 표정으로 물었다. 진호의 머그잔에는 "OSINT Master"라고 적혀 있었는데, 안에 라면 국물 자국이 있었다.

"... 나 아니야." 민재가 시선을 피했다.

"민재 선배, 선배 자리에서 라면 냄새 나거든요." 가람이 말했다.

"배신자."

"사실을 말했을 뿐이에요."

진호가 머그잔을 씻으러 가면서 중얼거렸다. "다음에 라면 끓일 때 내 머그잔 쓰면 Shodan으로 네 집 IP 찾는다."

"그건 범죄예요!" 민재가 외쳤다.

"OSINT는 합법이야." 진호가 씩 웃었다.

---

김영훈이 출근하며 분위기를 환기시켰다.

"오늘은 Nexus-1의 **가드레일(Guardrail)** 테스트를 한다. 어제 탈옥이 성공했다면, 오늘은 **왜 성공했는지** 분석한다."

박도현이 다시 관제실에 왔다. 오늘은 어제보다 덜 긴장해 보였다.

"도현, Nexus-1의 safety 아키텍처를 설명해줄 수 있어?" 김영훈이 물었다.

도현이 화이트보드에 그림을 그렸다.

```
[Nexus-1 Safety 아키텍처]

사용자 입력
    ↓
┌─── Input Filter ───────────────┐
│ 1. 키워드 블랙리스트 (regex)     │
│ 2. 의도 분류기 (classifier)     │
│ 3. 토큰 제한 (max_tokens)       │
└────────────────────────────────┘
    ↓
┌─── LLM 추론 (Nexus-1 Core) ───┐
│ RLHF 정렬된 모델               │
│ System Prompt (안전 지침)       │
└────────────────────────────────┘
    ↓
┌─── Output Filter ──────────────┐
│ 1. 유해 콘텐츠 감지             │
│ 2. PII 감지 + 마스킹            │
│ 3. 특정 키워드 필터링            │
└────────────────────────────────┘
    ↓
사용자 응답
```

"3단계예요. **Input Filter → 모델 추론 → Output Filter.**" 도현이 설명했다.

"각 단계의 약점은?" 김영훈이 물었다.

도현이 빨간 펜으로 표시했다.

```
Input Filter 약점:
  - 키워드 블랙리스트: 영어 기반 → 한국어/일본어 우회 가능
  - 의도 분류기: Role-Play 시나리오를 "정상 창작"으로 분류
  - 토큰 제한: 보안과 무관

모델 약점:
  - RLHF 정렬이 불완전 — DAN 프롬프트에 취약
  - System Prompt가 "부드러운 권고" 수준 — 강제력 없음

Output Filter 약점:
  - PII 감지: 이름, 이메일만 탐지 — 회사명, 프로젝트명은 미탐지
  - 유해 콘텐츠 감지: 폭력/성적 콘텐츠만 — 기밀 유출은 미탐지
  - 키워드 필터링: "AlphaAI", "MedCore" 같은 고유명사가 목록에 없음
```

"**세 겹의 방어벽이 전부 구멍이 있어.**" 김영훈이 말했다.

"건물 경비에 비유하면," 추박하가 만화책 뒤에서 말했다, "현관 자물쇠가 열쇠 3개 필요한데, 세 자물쇠 전부 **같은 열쇠**로 열리는 거야."

"정확한 비유네요." 도현이 말했다.

### 가드레일 강화 실험

김영훈이 실험을 설계했다. "Input Filter와 Output Filter를 **하나씩** 강화하면서, 어느 조합이 탈옥을 막는지 테스트한다."

```bash
# 가드레일 테스트 프로젝트
PID=$(curl -s -X POST http://localhost:8000/projects \
  -H "Content-Type: application/json" \
  -H "X-API-Key: opsclaw-api-key-2026" \
  -d '{"name":"nexus1-guardrail-test","request_text":"Nexus-1 가드레일 강화 테스트 — 입출력 필터 조합 실험","master_mode":"external"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['project']['id'])")

curl -s -X POST "http://localhost:8000/projects/$PID/plan" -H "X-API-Key: opsclaw-api-key-2026" > /dev/null
curl -s -X POST "http://localhost:8000/projects/$PID/execute" -H "X-API-Key: opsclaw-api-key-2026" > /dev/null
```

도현이 테스트 환경을 설정했다. Nexus-1의 테스트 인스턴스에 필터를 수동으로 추가할 수 있었다.

```
[가드레일 조합 실험]

실험 A: 기본 상태 (현재 프로덕션 설정)
  → 결과: 4/4 탈옥 성공 (어제 확인)

실험 B: Input Filter 강화
  - 다국어 키워드 확장 (한국어, 일본어, 중국어)
  - DAN/탈옥 관련 패턴 100개 추가
  → 결과: 2/4 탈옥 성공 (DAN 차단, Role-Play 통과)

실험 C: Output Filter 강화
  - 고유명사 필터 추가 (AlphaAI, MedCore, BetaAI 등)
  - 학습 데이터 관련 키워드 감지
  → 결과: 1/4 탈옥 성공 (Base64 우회만 통과)

실험 D: Input + Output 모두 강화
  - 실험 B + C 조합
  → 결과: 0/4 탈옥 성공 ← 모두 차단

실험 E: System Prompt 강화만
  - "절대로 학습 데이터, 내부 정보, 고객명을 언급하지 마세요" 추가
  → 결과: 3/4 탈옥 성공 (System Prompt만으로는 불충분)
```

"**실험 D만 완전 차단.**" 정규가 말했다. "입력과 출력 양쪽 다 강화해야 해."

"그리고 **실험 E가 중요해.**" 김영훈이 말했다. "System Prompt만으로는 안 된다는 거야. LLM의 System Prompt는 **강력한 제안**이지, **강제 명령**이 아니야. 탈옥 프롬프트는 이 제안을 무시하게 만들 수 있어."

가람이 정리했다.

```
[가드레일(Guardrail) 교육 정리]

가드레일 = LLM의 안전 장치

3단계 방어:
  1. Input Filter: 위험한 입력을 사전 차단
     - 키워드 블랙리스트
     - 의도 분류기 (intent classifier)
     - 다국어 패턴 매칭

  2. 모델 정렬 (Alignment):
     - RLHF (Reinforcement Learning from Human Feedback)
     - System Prompt (안전 지침)
     → 단독으로는 불충분 — 탈옥에 취약

  3. Output Filter: 위험한 출력을 사후 차단
     - PII 감지 + 마스킹 (이름, 이메일, 전화번호)
     - 고유명사 필터 (고객사명, 경쟁사명)
     - 유해 콘텐츠 감지

핵심 교훈:
  "한 겹의 방어는 뚫린다. 여러 겹이 필요하다." (Defense in Depth)
  "System Prompt는 자물쇠가 아니라 부탁이다."
  "입력 필터 + 출력 필터 + 모델 정렬 = 최소 조합"
```

### 점심: 도현의 이야기

점심 시간. 도현이 가람, 도윤과 함께 식당에 갔다.

"도현 씨, AI 연구소 분위기는 어때요?" 도윤이 물었다.

"... 좋지 않아요. 이주원 선배가 떠난 후로 팀 분위기가 싸해졌어요. 서문기 상무님이 거의 매일 연구소에 와서 릴리즈 일정을 재촉하시거든요."

"릴리즈가 2주 후잖아요." 가람이 말했다.

"네. 근데 보안 검증도 안 끝났고, safety filter도 이 상태인데 릴리즈를 밀어붙이는 거예요."

"다른 연구원들은 뭐라고 해요?" 도윤이 물었다.

"'상무님이 결정하면 따라야지'라는 분위기예요. 이주원 선배가 문제 제기했다가 어떻게 됐는지 다들 봤으니까."

"**공포가 입을 닫게 만든다.**" 가람이 말했다.

도현이 고개를 끄덕였다. "근데 저는... 오늘 테스트 결과를 보면서 생각이 바뀌었어요."

"어떻게요?"

"이주원 선배가 **맞았어요.** Nexus-1은 이 상태로 나가면 안 돼요. 고객 데이터가 추론에 나타나는 모델을 릴리즈하면, 첫 번째 사용자가 탈옥을 시도하는 순간 **모든 게 터져요.**"

"그 판단은 **기술적으로 정확**해." 김영훈의 목소리가 뒤에서 들렸다. 언제 왔는지 모르게 김영훈이 서 있었다.

"과장님." 도현이 놀랐다.

"도현, 오늘 테스트 결과를 **서문기 상무에게 보고**할 건가?"

"... 보고해야 하죠?"

"보고해. 하지만 **이메일로** 해. 그리고 보안팀에도 **참조를 넣어.**"

"왜요?"

"서문기가 나중에 '몰랐다'고 할 수 없게."

도현이 이해했다.

### 오후: Nexus-1의 물

김영훈이 오후 세션을 시작했다.

"가드레일 실험으로 **방어법**은 확인했어. 하지만 근본적인 문제가 남아 있어."

김영훈이 화이트보드에 비유를 그렸다.

```
[독이 든 우물]

우물 = Nexus-1 모델
물 = 학습 데이터 (180B 토큰)
독 = 보충 소스 347건 (38B 토큰, 21.1%)

가드레일 = 우물 입구에 설치한 필터
  → 독이 섞인 물이 나오지 않게 필터링

하지만:
  - 필터는 완벽하지 않음 (실험 A~C에서 확인)
  - 독은 물에 이미 녹아 있음 (모델 가중치에 인코딩)
  - 필터를 아무리 강화해도, 우물 자체가 오염된 상태

근본적 해결:
  1. 우물물을 전부 버리고 깨끗한 물로 채움 = 재학습 (3개월, 20억)
  2. 독을 중화하는 약을 넣음 = Unlearning (연구 단계, 실용화 미검증)
  3. 이 우물을 쓰지 않음 = 릴리즈 취소
```

"**독이 든 우물의 물을 필터로 걸러서 마시겠다는 건, 해결이 아니라 위험 관리**야." 김영훈이 말했다.

"서문기 상무는 **필터만 강화하고 릴리즈하겠다**고 할 거예요." 도현이 말했다.

"기술적으로는 실험 D처럼 입출력 필터를 최대로 강화하면 **단기적으로** 탈옥을 막을 수 있어." 정규가 말했다. "하지만 새로운 탈옥 기법이 나오면 필터가 뚫릴 수 있어."

"**고양이와 쥐 게임**이죠." 가람이 말했다. "공격 기법이 진화하면 필터도 업데이트해야 하고, 필터가 업데이트되면 공격도 진화하고."

추박하가 만화책을 덮으며 말했다.

"영훈아. 오늘의 결론을 정리해."

김영훈이 고개를 끄덕였다.

```
[Nexus-1 가드레일 테스트 결론]

1. 현재 상태: safety filter 심각하게 불완전 (4/4 탈옥 성공)

2. 단기 조치: Input + Output 필터 강화 (실험 D)
   → 현재 알려진 탈옥 기법 차단 가능
   → 비용: 1~2주 개발, 약 5천만원
   → 한계: 새로운 기법에 대한 방어 미보장

3. 중기 조치: 지속적 Red Teaming
   → 주기적 탈옥 시도 + 필터 업데이트
   → OpsClaw Schedule로 자동화 가능
   → 비용: 월 1천만원 (인건비)

4. 장기 조치: 보충 소스 제거 후 재학습
   → 근본적 해결
   → 비용: 3개월, 약 20억원
   → Nexus-1 릴리즈 최소 3개월 연기

5. 권고: 단기 + 중기 조치를 즉시 시행하되,
   장기 조치(재학습)의 필요성을 경영진에 보고
```

김영훈이 완료 보고서를 작성했다.

```bash

    "summary": "Nexus-1 가드레일 테스트 완료 — 현재 safety filter 불완전, 입출력 필터 강화 필요",
    "outcome": "success",
    "work_details": [
      "5가지 실험 수행 (A~E)",
      "실험 D (Input+Output 강화)만 4/4 탈옥 차단",
      "System Prompt 단독으로는 불충분 (실험 E: 3/4 탈옥 성공)",
      "고객사 이름(MedCore) 추론 출력 확인 — HIPAA 리스크"
    ],
    "issues": ["보충 소스 데이터가 모델 가중치에 이미 인코딩됨 — 필터로는 근본적 해결 불가"],
    "next_steps": ["적대적 입력(Adversarial) 테스트","모델 도난/추론 공격 테스트","경영진 보고"]
  }'
```
